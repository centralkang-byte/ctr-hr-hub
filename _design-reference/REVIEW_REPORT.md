# CTR HR Hub — 전체 페이지 빡센 리뷰

> 총 30+ 페이지 통합 리뷰. 발견된 이슈를 우선순위별로 정리.

---

## 🔴 P0 — 즉시 수정 필요 (시각 일관성 깨짐)

### 1. 사이드바 통합 페이지의 자식 헤더 중복
- `LeaveAbsenceWrapper`, `PerfGrowthWrapper`의 `InlineNoPageH` CSS가 자식 페이지의 page-h를 숨기지만, 자식 페이지가 자체 `wd-stat-strip`을 그리면 부모와 시각적으로 단절됨.
- **해결:** 자식 페이지의 통계도 같이 숨기거나, wrapper에 통합 KPI 표시.

### 2. 내 프로필 vs 직원 상세 — 동일 구조지만 일부 데이터 불일치
- 같은 컴포넌트지만 액션 라벨이 다름 ("수정 요청" vs "편집").
- **해결:** 권한 prop으로 통합. `isSelf` flag로 액션 그룹만 다르게.

### 3. 위저드 페이지 헤더 충돌
- 위저드는 자체 sticky 헤더가 있는데 토픽바도 같이 보임 → 두 헤더가 겹침.
- **해결:** 위저드 활성 시 토픽바 숨기거나 위저드 헤더에 토픽 정보 통합.

## 🟡 P1 — 일관성 점검 필요

### 4. KPI 사용 일관성 모호
- 통계 스트립 사용: 휴가/근태/온보딩/팀허브/채용/인사이트 등
- 인라인 칩 사용: 직원관리/조직도/팀근태/위임
- **기준이 명확치 않음.** 어떤 페이지가 어떤 패턴을 써야 하는지 디자인 규칙 보완 필요.

**제안 규칙:**
- 4개 의미있는 숫자 + 데이터 풍부 → 통계 스트립
- 2-4개 단순 카운트 → 인라인 칩
- 1-2 핵심 + 서술 → 요약 문장형

### 5. 카피 톤 일관성
- "친근 톤" 적용된 페이지: ~예요/돼요
- 일부 페이지 잔존: "관리합니다", "확인합니다" (격식체)
- 특히 **인사이트·설정·컴플라이언스**에 격식체 잔존.

### 6. 빈 상태(empty state) 변형 多
- 종류 1: `.empty` 단순 (EmptyBox + em-title)
- 종류 2: `.empty` 풀 (아이콘 + 제목 + 부제 + CTA)
- 종류 3: 카드 안에 직접 `padding: var(--space-10)`
- **통일 필요.**

### 7. 버튼 라벨 길이 불균형
- "신규 직원 등록" (6자) vs "신규" (2자) — 같은 액션인데 페이지마다 다름.

## 🟢 P2 — 콘텐츠 풍부도 점검

### 8. 인사이트 8개 페이지 콘텐츠 풍부도 격차
- Executive Summary, 인력, 급여, 성과, 이직, AI: 풍부 ✓
- 근태 분석, 팀 헬스: 잘됐지만 다른 페이지 대비 차트 적음.

### 9. Placeholder 잔존
- 설정 카테고리 5개 (조직·근태·급여·성과·채용) — 좌측 탭만 있고 우측은 placeholder.
- 시스템 카테고리 3개 탭 — 알림 규칙, 언어/타임존, 데이터 보존.

### 10. 나의 공간 일부 콘텐츠 얇음
- 칭찬/인정: 보낸/받은 탭 placeholder
- 문서/증명서: 빠른 신청 4개 카드만, 이력 빈약

## 🔵 P3 — 인터랙션 완성도

### 11. ⌘K 글로벌 검색 미구현
- 탑바 검색이 작동 안 함 (placeholder만).
- **Workday 시그니처 기능.** 우선 구현 권장.

### 12. 테이블 행 클릭 → 상세 슬라이드 없음
- 직원 목록·결재 큐 등에서 페이지 이동 대신 inspector 패널 (이전에 논의).

### 13. 테이블 가로 스크롤만으로 모바일 대응
- `.tbl-as-cards` 클래스 정의됐지만 실제 사용된 곳 없음.

### 14. 호버 시 직원 미니카드 — 미구현
- 이름에 호버 시 작은 프로필 카드.

### 15. 벌크 액션
- `.wd-bulk-bar` CSS 있으나 실제 사용 없음.
