## 상세 Gap 분석

### G1: 직급 체계 불일치 🔴

**규정 (제3조, 제3조의2)**:
- 관리직 직급: L1(매니저), L2(책임매니저) — 2단계
- 호칭(Title): 직급과 별도 운영 가능
- 임원: 전문리더, 경영리더 (별도 체계)

**현재 시스템** (`prisma/seeds/37-job-grades.ts`):
- 7단계: G-CHAIR(회장) / G-ML(경영리더) / G-EL(전문리더) / G-SM(책임매니저) / G-MGR(매니저) / G-SE(책임연구원) / G-ENG(연구원)
- 직급과 호칭이 혼용됨

**CEO 확인**:
- 회장/부회장은 호칭, 실제 직급은 L1(매니저), L2(책임매니저) + 전문리더, 경영리더
- **향후**: L1~L5로 확장 + 임원(전문리더/경영리더) + 호칭(Title) 분리

**수정 방향**: JobGrade 리팩터링 — 별도 Phase에서 진행
- `JobGrade.gradeType`: 'STAFF' | 'EXECUTIVE' | 'TITLE' 구분
- 또는 별도 `EmployeeTitle` 모델

---

### G2: 관리직/생산직/전문직 구분 없음 🟡

**규정 (제3조)**:
- 종업원 = 관리직 + 생산직 + 전문직
- 전문직: 단순업무/보조, 별도 직급 가능
- 계약직: 별도 호칭 부여 가능

**현재**: `EmploymentType` = FULL_TIME | CONTRACT | DISPATCH | INTERN
- 관리직/생산직/전문직 구분 없음

**수정 방향**: `employeeCategory` 또는 `jobTrack` 필드 추가
- 'MANAGEMENT' | 'PRODUCTION' | 'SPECIALIST' | 'CONTRACT'

---

### G3: 승진년한 시스템 미구현 🔴

**규정 (제24조)**:
- L1→L2 표준 소요년한: 4년 (대졸 기준)
- 전문학사: +1년 (5년)
- 석사: -1년 (3년)
- 경력직: 인정 경력년수 차감
- 특별승진: 탁월한 성과 시 년한 미달 가능

**현재**: JobGradesTab에 `minYears` 표시 있으나, 승진 시 자동 검증 없음

**수정 방향**:
- JobGrade에 `minPromotionYears` 필드 (Settings에서 설정)
- 승진 실행 시 경고/차단: "최소 년한 미달 (3/4년)"
- 특별승진 플래그로 우회 가능

---

### G4: 수습 전환 팀장 평가 알림 + ApprovalFlow 미연결 🟡

**규정 (제18~19조)**:
- 수습기간: 3개월
- 적격 여부 판단 → 전환 또는 채용 취소

**현재**: `probationEndDate` 필드 있으나 만료 시 알림/워크플로우 없음

**수정 방향** (플로우 확정):
```
[D-14] Cron → 팀장에게 "수습 평가 필요" 알림
  → 팀장: 수습 평가서 작성 (전환 추천 / 연장 / 취소)
  → ApprovalFlow 실행 (Settings에서 설정)
  → 승인 → EmployeeAssignment.employmentType 전환
  → 거절 → 수습 연장 또는 채용 취소
```

---

### G5: 계약직→정규직 전환 알림 미구현 🟡

**규정 (제15조)**:
- 2년 이상 계속 근무 + 우수 실적 → 정규직 전환 검토 가능

**현재**: 자동 알림/워크플로우 없음

**수정 방향** (플로우 확정):
```
[D-30] Cron → 팀장에게 "계약 만료 평가 필요" 알림
  → 팀장: 전환 평가서 (정규직 전환 / 계약 연장 / 종료)
  → ApprovalFlow 실행
  → 승인 → CONTRACT → FULL_TIME 전환
  → 계약 연장 → endDate 갱신
  → 종료 → 자연면직 (제29조 7항)
```

### G6: 업무인수인계서 미구현 🟠

**규정 (제22조)**: 전보 시 업무인수인계서 작성 필수

**현재**: 전보 API에 인수인계 문서 관리 기능 없음

**수정 방향**: 별도 Phase — HandoverDocument 모델 또는 체크리스트

---

### G7: 면직 5유형 구분 없음 🟡

**규정 (제26~31조)**: 의원면직, 정년면직, 자연면직, 징계면직, 회사형편면직

**현재**: `EmployeeStatus` = RESIGNED | TERMINATED (2가지만)

**수정 방향**: `separationType` 필드 추가
- 'VOLUNTARY' | 'RETIREMENT' | 'NATURAL' | 'DISCIPLINARY' | 'REDUNDANCY'

---

### G8: 정년 자동 알림 미구현 🟡

**규정 (제28조)**: 만 60세, 당해 년말까지

**현재**: 생년월일 필드 있으나 정년 알림 없음

**수정 방향**: Nudge 엔진에 정년 규칙 추가
- D-180, D-90, D-30 알림 (HR + 본인 + 팀장)

---

### G9: 사직 본부장→대표 승인 미반영 🟡

**규정 (제32조)**: 사직원 → 소속 본부장 결재 → 인사부서 → 대표이사 승인

**현재**: HR_ADMIN이 직접 오프보딩 처리 (결재 없음)

**수정 방향**: ApprovalFlow 모듈에 'offboarding' 추가 → 결재 플로우 연결

---

### G10: 발령 최고경영자 승인 미반영 🟡

**규정 (제33조)**: 채용/전보/승진 → 최고경영자 승인 → 발령

**현재**: BulkMovement/Transfer API에 결재 체크 없음

**수정 방향**: ApprovalFlow 모듈에 'personnel_order' (인사발령) 추가

---

### G11: 인사 문서 영구보존 정책 🟠

**규정 (제34~40조)**:
- 합격자 지원서류: 영구보존
- 근로자 명부: ERP 등록, 영구보존
- 발령원부: 영구보존
- 인사 품의서: 10년 이상 보존

**현재**: 데이터 보존 정책 미정의 (soft delete만 사용)

**수정 방향**: Settings > 데이터 보존 탭에서 문서 유형별 보존 기간 설정
