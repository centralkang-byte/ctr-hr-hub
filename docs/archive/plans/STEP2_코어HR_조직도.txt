### 챗봇 API

**POST /api/v1/hr-chatbot/ask**
```
Request:
  session_id: string (optional, 없으면 신규 세션)
  question: string
  company_id: string

Response:
  session_id: string
  answer: string
  sources: [{ document_title, doc_type, chunk_content_preview, similarity_score }]
  confidence_score: number (0~1)
  needs_escalation: boolean
```

**처리 플로우**
1. 질문 텍스트 임베딩 생성
2. hr_document_chunks에서 cosine 유사도 검색 (WHERE document.company_id = ? AND document.is_active = true)
3. top_k=5 청크 가져오기 (threshold >= 0.7)
4. Claude 호출: hrChatbotAnswer (prompt_version: "hr-chatbot-v1.0")
   - system: "당신은 CTR 그룹의 HR 규정 전문 상담사입니다. 제공된 규정 문서 맥락만 기반으로 답변하세요. 규정에 없는 내용은 '해당 규정을 찾지 못했습니다'라고 답하세요. 반드시 출처(규정명+조항)를 명시하세요."
   - context: top_k 청크 내용
   - conversation_history: 최근 5턴
5. confidence_score < 0.7 → needs_escalation=true
6. hr_chat_messages INSERT

**POST /api/v1/hr-chatbot/escalate**
- session_id → 해당 세션의 최근 대화를 HR_ADMIN에게 알림
- hr_chat_messages.escalated = true
- notifications INSERT (HR_ADMIN 대상, "챗봇 에스컬레이션: [질문 요약]")

**GET /api/v1/hr-chatbot/sessions**
- 내 대화 세션 목록 (최근 20개)

**GET /api/v1/hr-chatbot/sessions/:id/messages**
- 세션 내 메시지 목록

- 고용정보: 사번, 법인, 부서, 직급, 직군, 고용형태, 입사일, 상태
- HR_ADMIN: 인라인 편집 모드 (연필 아이콘 클릭 → 폼 전환 → 저장)
- 수정 시 audit_logs 기록

**탭 2: 인사이력**
- employee_histories 타임라인 (최신순)
- 각 항목: 아이콘 + 변경 유형 + 변경 내용 + 일자 + 승인자
- 아이콘: HIRE=🟢, TRANSFER=🔄, PROMOTION=⬆️, DEMOTION=⬇️, RESIGN=🔴, TRANSFER_CROSS_COMPANY=🌐
- 법인 간 전입: "CTR → CTR America" 형태 표시

**탭 3: 문서**
- employee_documents 목록 (DataTable: 유형/제목/업로드일/크기)
- 업로드 버튼 (S3 Presigned URL)
- 다운로드: 권한 재검증 후 Presigned URL 발급
- 민감 문서(CONTRACT, ID_CARD): HR_ADMIN만 다운로드 가능
- 삭제: Soft Delete (deleted_at)

**탭 4: 징계·상벌**
- 징계 이력 타임라인 (유형별 아이콘 + 색상)
  VERBAL_WARNING=🟡, WRITTEN_WARNING=🟠, REPRIMAND=🔴, SUSPENSION=⛔
- 포상 이력 타임라인 (🏆 아이콘)
- EMPLOYEE: 본인 것만 조회 가능 / 타인 → 403

**탭 5: 연봉이력**
- compensation_history 타임라인
- 각 항목: 변경 유형 + 이전 금액 → 변경 금액 + 인상률 + compa-ratio
- HR_ADMIN만 조회 가능 / EMPLOYEE는 이 탭 자체 비표시

### 3. 직원 등록 (/employees/new) — HR_ADMIN

**다단계 폼 (Step Wizard)**

Step 1: 기본정보
- 이름(필수), 영문명, 생년월일, 성별, 국적, 이메일(필수, UK 검증),
  전화, 비상연락처 이름+전화

Step 2: 고용정보
- 사번(자동 생성 또는 수동, UK 검증)
- 고용형태: FULL_TIME / CONTRACT / DISPATCH / INTERN
- 입사일

Step 3: 배정
- 법인 선택 (CompanySelector에서 선택된 법인 기본값)
- 부서 선택 (법인별 부서 드롭다운)
- 직급 선택
- 직군 선택 (OFFICE / PRODUCTION / R_AND_D / MANAGEMENT)
  ★ v3.1: job_category_id FK로 저장
- 매니저 선택 (직원 검색)

Step 4: 확인
- 입력 내용 요약 + 최종 확인 + 등록 버튼

**등록 처리 ($transaction)**
1. employees INSERT
2. employee_histories INSERT (change_type: HIRE)
3. employee_auth INSERT (초기 비밀번호 해시 또는 null=SSO 전용)
4. 알림: 매니저에게 "새 팀원 입사" 알림
5. 온보딩 자동 시작 체크박스 → 체크 시 employee_onboarding 자동 생성

**법인 간 전입 처리**
- 기존 법인에서 RESIGN 처리 없이 TRANSFER_CROSS_COMPANY 이력 기록
- employee_histories: from_company_id + to_company_id
- employees.company_id 업데이트
- employee_roles: 이전 법인 역할 종료(end_date) + 새 법인 역할 시작

### 5. 조직 개편 관리 (/settings/org-changes) — HR_ADMIN

**조직개편 이력 목록**
- DataTable: 일자 / 변경유형 / 대상부서 / 변경내용 / 승인자
- 변경유형 필터: CREATE/MERGE/SPLIT/RENAME/CLOSE/RESTRUCTURE

**변경 전/후 비교 UI**
- from_data / to_data JSON diff 시각화
- 변경된 필드 하이라이트

**조직개편 실행**

단건 변경:
- 부서 생성: 부서명, 코드, 상위부서, 레벨 입력 → departments INSERT + org_change_history(CREATE)
- 부서 통합: 대상 부서 2개+ 선택 → 통합 부서 선택 → 소속 직원 일괄 이동 + org_change_history(MERGE)
- 부서 분리: 대상 부서 선택 → 새 부서 정보 입력 → 직원 배분 → org_change_history(SPLIT)
- 부서 폐지: 대상 부서 선택 → 소속 직원 이동처 지정 → departments.is_active=false + org_change_history(CLOSE)
- 부서 명칭 변경: 대상 부서 선택 → 새 이름 입력 → org_change_history(RENAME)

일괄 변경:
- POST /api/v1/org/restructure
- 여러 부서 동시 변경 → $transaction
- 각 변경마다 org_change_history 기록

**감사 로그**: 모든 조직개편 → audit_logs 필수
