### 1. 급여 밴드 관리 (/settings/salary-bands) — HR_ADMIN

**급여 밴드 CRUD**
- DataTable: 법인 / 직급 / 직군 / 최소 / 중간값 / 최대 / 통화
- 필터: 법인(CompanySelector) / 직급 / 직군
- 생성/수정 폼:
  - 법인 선택
  - 직급 선택 (job_grades)
  - 직군 선택 (job_categories JOIN)
  - 최소 금액 (min_salary)
  - 중간값 (mid_salary)
  - 최대 금액 (max_salary)
  - 통화 코드 (KRW/USD/CNY/INR/PLN/TRY/MXN)
- salary_bands INSERT/UPDATE

**Compa-Ratio 개념**
- compa_ratio = 직원 현재 연봉 / 해당 밴드 mid_salary
- 1.0 = 중간값 수준
- < 0.8 = 저보상 🔴
- 0.8~0.95 = 시장 대비 낮음 🟡
- 0.95~1.05 = 적정 🟢
- 1.05~1.20 = 높음 🔵
- > 1.20 = 과보상 🟣

### 2. 연봉 조정 매트릭스 (/settings/salary-matrix) — HR_ADMIN

**인상률 매트릭스 설정**
- salary_adjustment_matrix CRUD
- 3×3 매트릭스 (EMS 블록 기반):
  - 행: 성과 등급 (Low/Mid/High → EMS 블록 그룹)
  - 열: compa-ratio 구간 (Below/At/Above)
  - 셀: 인상률 범위 (min% ~ max%)

예시 매트릭스:
```
                 Compa < 0.9   0.9~1.1    Compa > 1.1
 ───────────────────────────────────────────────────────
 성과 High        8~12%        5~8%        2~4%
 (블록 6,8,9)
 ───────────────────────────────────────────────────────
 성과 Mid         5~8%         3~5%        1~3%
 (블록 2,5,7)
 ───────────────────────────────────────────────────────
 성과 Low         0~3%         0~2%        0%
 (블록 1,3,4)
 ───────────────────────────────────────────────────────
```

- 법인별 + 연도별 설정
- 매트릭스 복사 기능 (전년도 → 올해)

### 3. 연봉 조정 프로세스 (/compensation)

#### 3-1. 연봉 조정 시뮬레이션 — HR_ADMIN

**시뮬레이션 뷰 (/compensation/simulation)**
- 대상: 해당 법인의 ACTIVE 직원 전체 (또는 부서별 필터)
- DataTable:
  직원명 / 부서 / 직급 / 직군 / EMS 블록 / 현재 연봉 / Compa-Ratio /
  매트릭스 권장 인상률 / 시뮬레이션 인상률 / 시뮬레이션 신규 연봉 / 신규 Compa-Ratio

- **매트릭스 자동 추천**:
  - 직원의 EMS 블록 → 성과 등급 매핑
  - 직원의 현재 compa-ratio → 구간 매핑
  - 매트릭스에서 인상률 범위 추출 → 중간값을 기본 추천

- **수동 조정**:
  - HR_ADMIN이 개별 직원의 인상률 수정 가능 (인라인 편집)
  - 매트릭스 범위 벗어남 → 🟡 경고 아이콘 (저장은 허용)

- **총 예산 요약**:
  - 총 인상 금액 합계
  - 평균 인상률
  - 부서별 인상률 분포

**AI 연봉 조정 보조**

#### 3-2. 연봉 조정 확정 — HR_ADMIN

**확정 프로세스**
1. 시뮬레이션 → "확정" 버튼
2. 확정 확인 모달: "총 {{N}}명의 연봉이 조정됩니다. 진행하시겠습니까?"
3. $transaction:
   - 대상 직원별 compensation_history INSERT:
     - change_type: ANNUAL_RAISE / PROMOTION_RAISE / MARKET_ADJUSTMENT / OTHER
     - previous_salary, new_salary, change_percentage
     - compa_ratio (신규 기준)
     - effective_date
     - approved_by
   - employees.current_salary 업데이트 (Phase 1: employees에 current_salary 컬럼 없으면 최신 compensation_history에서 조회)
4. audit_logs (COMPENSATION_ADJUSTMENT) 기록
5. 알림: 각 직원에게 "연봉이 조정되었습니다" (금액은 알림에 미포함, 상세는 프로필에서 확인)

### 8. Attrition Risk 통합 모듈

#### 8-1. 위험 점수 산출 로직

**6요인 모델 (0~100 가중 합산)**

| 요인 | 가중치 | 데이터 소스 | 산출 방식 |
|------|--------|-----------|----------|
| 근속 위험 | 15% | employees.hire_date | 1~2년 = 높음, 3~5년 = 중간, 5년+ = 낮음 |
| 보상 불만족 | 25% | compa-ratio | < 0.85 = 높음, 0.85~0.95 = 중간, 0.95+ = 낮음 |
| 성과 불일치 | 20% | EMS 블록 + compa | 고성과+저보상 = 높음, 저성과+고보상 = 중간 |
| 매니저 관계 | 15% | 1:1 빈도 + Pulse 매니저 점수 | 30일+ 미실시 = 높음, 낮은 Pulse = 높음 |
| 참여도 | 15% | Pulse 응답 + Recognition | 미응답 연속 = 높음, 이직 고려 YES = 매우 높음 |
| 근태 패턴 | 10% | 지각률 + 결근 + 초과근무 | 지각 증가 추세 = 높음, 번아웃 = 높음 |

**산출 실행**
- pg_cron: 매일 새벽 3시 실행
- SQL 함수 또는 Edge Function:
  1. ACTIVE 직원 전체 순회
  2. 6요인 각각 0~100 산출
  3. 가중 합산 → total_score (0~100)
  4. 리스크 레벨: LOW(0~39) / MEDIUM(40~69) / HIGH(70~100)
  5. attrition_risk_history INSERT (snapshot)

**v3.1 보강: AI 보정 프롬프트**
- total_score 70+ (HIGH) 직원에 대해 AI 보정 실행
- lib/claude.ts:
```ts
attritionRiskAssessment(
  employee_name: string,
  department: string,
  grade: string,
  tenure_months: number,
  factor_scores: {
    tenure_risk: number,
    compensation_dissatisfaction: number,
    performance_mismatch: number,
    manager_relationship: number,
    engagement: number,
    attendance_pattern: number
  },
  total_score: number,
  recent_history_summary: string,   // v3.1: 최근 6개월 인사이력 요약
  discipline_count: number,         // v3.1: 활성 징계 건수
  compa_ratio: number,
  ems_block: number | null
): Promise<{
  adjusted_score: number,           // AI 보정 점수
  adjusted_level: 'LOW' | 'MEDIUM' | 'HIGH',
  risk_drivers: string[],           // 핵심 위험 요인
  contextual_risks: string[],       // v3.1: 맥락적 위험 (강등+급여하락 조합 등)
  retention_actions: string[],      // 권장 리텐션 액션
  confidence: 'HIGH' | 'MEDIUM' | 'LOW'
}>
prompt_version: "attrition-risk-v1.1"   // v3.1 보강 버전
```
- AI 보정 결과 → attrition_risk_history.ai_assessment 저장
- ai_logs (ATTRITION_RISK_ASSESSMENT) 기록

#### 8-2. Attrition Risk 대시보드 (/analytics/attrition) — HR_ADMIN

**전사 위험 현황**
- KPI 카드 3개:
  - HIGH 위험 직원 수 (🔴)
  - MEDIUM 위험 직원 수 (🟡)
  - 전월 대비 증감
- 위험 분포 도넛 차트 (LOW/MEDIUM/HIGH)

**HIGH 위험 직원 목록**
- DataTable: 직원명 / 부서 / 직급 / 점수 / 레벨 / 핵심 요인 / 권장 액션
- 행 클릭 → 상세 패널:
  - 6요인 레이더 차트
  - AI 보정 결과 (contextual_risks 포함)
  - 최근 인사이력 타임라인
  - 권장 리텐션 액션 체크리스트

**부서별 히트맵**
- 부서 × 리스크 레벨 히트맵 (색상 강도)
- 위험 집중 부서 식별

**추이 차트**
- 월별 HIGH/MEDIUM/LOW 인원 추이 (Stacked Area Chart)

#### 8-3. 매니저 뷰

**직원 프로필 → Attrition Risk (HR_ADMIN만)**
- 현재 점수 + 레벨 뱃지
- 6요인 레이더 차트
- AI 인사이트 (있는 경우)
- 이력 추이 (최근 6개월 라인 차트)

**MANAGER**: 직속 부하의 위험 수준만 간접 표시
- 홈 대시보드: "팀 이직 위험 알림" (HIGH 직원 수만, 구체 점수 미표시)
- 상세 점수는 HR_ADMIN만 열람

#### 8-4. 예측 정확도 피드백 (STEP 3 퇴직 연동)

- 퇴직 확정 시 해당 직원의 마지막 attrition_risk_history.total_score 조회
- Analytics에서:
  - 실제 퇴직자 중 HIGH 였던 비율 = True Positive Rate
  - HIGH였으나 미퇴직 비율 = False Positive Rate
  - STEP 7 Analytics 대시보드에 표시
