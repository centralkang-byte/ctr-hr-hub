B-2. 러시아 전자서명 (КЭДО)
================================================================

### 배경
2022년 러시아 연방법 377-ФЗ: 인사 문서 전자화 가능
КЭДО (Кадровый электронный документооборот) = 인사 전자 문서 관리

### 구현 방향 (Phase 3는 인터페이스 정의 + 외부 연동 준비)

**전자서명 수준 3가지:**
1. ПЭП (простая электронная подпись) — 단순 전자서명 (로그인+확인)
2. УНЭП (усиленная неквалифицированная) — 강화 비인증 서명
3. УКЭП (усиленная квалифицированная) — 강화 인증 서명 (공인)

→ CTR Russia: ПЭП (단순) + УНЭП (강화 비인증) 수준으로 시작

### 인터페이스 정의

```ts
// lib/esign/kedo-interface.ts

interface KedoDocument {
  id: string;
  employee_id: string;
  doc_type: KedoDocType;
  title: string;
  content_url: string;            // 문서 파일 URL
  signature_required: SignatureLevel;
  status: KedoStatus;
  signed_at?: string;
  signature_data?: string;        // 서명 데이터 (해시)
}

type KedoDocType =
  | 'EMPLOYMENT_CONTRACT'         // 근로계약서
  | 'SUPPLEMENTARY_AGREEMENT'     // 추가 합의서
  | 'TRANSFER_ORDER'              // 전보 명령
  | 'VACATION_ORDER'              // 휴가 명령
  | 'DISMISSAL_ORDER'             // 해고 명령
  | 'WORK_SCHEDULE'               // 근무 일정 통보
  | 'PAYSLIP'                     // 급여명세서
  | 'OTHER';

type SignatureLevel = 'PEP' | 'UNEP' | 'UKEP';
type KedoStatus = 'DRAFT' | 'PENDING_SIGNATURE' | 'SIGNED' | 'REJECTED' | 'EXPIRED';
```

### 문서 서명 플로우 (ПЭП 단순 전자서명)
1. HR이 문서 생성/업로드
2. 직원에게 서명 요청 알림
3. 직원: 문서 확인 → "서명" 버튼 클릭
4. 2차 인증 (비밀번호 재입력 또는 SMS OTP)
5. 서명 기록: 타임스탬프 + IP + User Agent + 해시값
6. 양측 서명 완료 → 문서 확정 (수정 불가)
7. 보관: 최소 50년 (러시아 법)

### kedo_documents (신규 테이블)
```
id(uuid PK), employee_id(FK employees NOT NULL),
company_id(FK companies NOT NULL),
doc_type(ENUM KedoDocType),
title(String NOT NULL),
file_key(String NOT NULL),
signature_level(ENUM SignatureLevel: PEP/UNEP/UKEP),
status(ENUM KedoStatus: DRAFT/PENDING_SIGNATURE/SIGNED/REJECTED/EXPIRED),
requested_at(DateTime nullable),
signed_at(DateTime nullable),
signer_ip(String nullable),
signer_user_agent(String nullable),
signature_hash(String nullable),
rejected_reason(String nullable),
expires_at(DateTime nullable),
created_by(FK employees),
created_at(DateTime), updated_at(DateTime)

@@index([employee_id, status])
```

### UI — 문서 서명 (/documents/sign)
- EMPLOYEE: 서명 대기 문서 목록 → 문서 미리보기 → 서명 버튼
- HR_ADMIN: 문서 생성 → 대상 직원 선택 → 서명 요청 발송
- 서명 현황 대시보드: 서명완료/대기/거부/만료 통계
