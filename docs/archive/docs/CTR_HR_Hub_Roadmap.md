# CTR HR Hub — 통합 로드맵 v2.0
## 리팩토링 마스터플랜 v2.0 기반 (16세션 / 약 15주)

---

## 1. 전체 구조 요약

기존 24세션(Phase 1~5) 계획을 16세션(Phase A~C)으로 통합 재편.
STEP 0~9 + R1~R9 완료 상태에서, 아키텍처 기반 → 기능 구현 → UX 표준화 순으로 진행.

```
Week 1-2:   A1 → A2          (아키텍처 기반)
Week 3-4:   B1 → B2          (법인 엔진 + Core HR)
Week 5-6:   B3 → B4          (성과 + 채용)
Week 7-8:   B5 → B6          (온보딩 + 근태)
Week 9-10:  B7 → B8          (급여 + 조직도)
Week 11-12: B9 → B10         (복리후생 + 애널리틱스)
Week 13:    B11               (통합 연동)
Week 14-15: C1 → C2 → C3     (UX 표준화)
```

---

## 2. PHASE A: 아키텍처 기반 (2세션)

이후 모든 작업의 토대. 가장 신중하게 진행.

### A1: 사이드바 IA 재설계
- 17개 flat NavGroup → 7개 역할 기반 섹션
- `navigation.ts` config + `useNavigation` 훅 + Sidebar 재구축
- comingSoon 셸 페이지 (8~10개)
- 역할별 메뉴 가시성 매트릭스
- **의존성:** 없음 (첫 번째 세션)
- **상태:** 🔄 프롬프트 작성 완료

### A2: Core HR 데이터 모델 변경
- Effective Dating 패턴 도입 (발효일/종료일)
- Position/Job 분리
- Matrix Organization 지원
- `company_process_settings` 기반 테이블
- **의존성:** A1 완료
- **상태:** 📋 예정

## 3. PHASE B: 기능 구현 (11세션)

A1~A2 아키텍처 위에 실제 기능 구현.

| 세션 | 내용 | 핵심 | 의존성 |
|------|------|------|--------|
| **B1** | 법인별 커스터마이징 엔진 | 평가/승진/보상 설정, 글로벌↔오버라이드 UI | A2 |
| **B2** | Core HR 고도화 UI | 발령 타임라인, 시점 조회, 엑셀 업로드, 프로필 통합 | A2 |
| **B3** | 성과관리 고도화 | Competency Framework, Talent Review+승계, AI 평가 | B1 |
| **B4** | 채용 ATS 고도화 | 채용 요청 워크플로, Internal Mobility, Talent Pool | A2 |
| **B5** | 온보딩/오프보딩 고도화 | Cross-boarding, 법인별 템플릿, 오프보딩 체계화 | B1 |
| **B6** | 근태/휴가 고도화 | 교대근무, 법인별 휴가 정책, 통합 승인함 | B1 |
| **B7** | 급여 해외 + 분석 | 해외 데이터 업로드, 글로벌 급여 분석, 시뮬레이션 | B1 |
| **B8** | 조직도 + People Directory | 인터랙티브 조직도, 전사 검색, 스킬 매트릭스, 조직 시뮬레이션 | A2 |
| **B9** | 복리후생 모듈 | 6개 테이블, 22개 시드, 카탈로그+동적폼+승인 워크플로 | B1 |
| **B10** | HR 애널리틱스 + CFR | 번아웃 감지, 이직 예측 v2, Feedback Badges, KPI 커스텀 | B5 |
| **B11** | 기존 모듈 통합 연동 | 법인설정 반영, 알림 이벤트 추가, i18n 키 확장 | B1~B10 |

### 세션 간 의존성 맵
```
A1 ──→ A2 ──→ B1 ──→ B2
                │      │
                ├──→ B3 (성과: B1 평가설정)
                ├──→ B5 (온보딩: B1 법인설정)
                ├──→ B6 (근태: B1 법인설정)
                ├──→ B7 (급여: B1 보상설정)
                └──→ B9 (복리후생: B1 법인설정)
         A2 ──→ B4 (채용: Position 모델)
         A2 ──→ B8 (조직도: Matrix Org)
                B5 ──→ B10 (애널리틱스: 퇴직면담 데이터)
    B1~B10 전체 ──→ B11 (통합 연동)
```

## 4. PHASE C: UX 리팩토링 (3세션)

PHASE A~B에서 만든 기능의 UI/UX를 표준화.
CLAUDE.md(디자인 토큰) + CTR_UI_PATTERNS.md(패턴) 이원 체계 기준.

| 세션 | 내용 | 산출물 |
|------|------|--------|
| **C1** | 데이터 테이블 + 사이드 패널 표준화 | StandardDataTable, StandardSidePanel, SearchFilterBar, EmptyState |
| **C2** | 대시보드 카드형 레이아웃 + 차트 통일 | DashboardCard, ChartTheme, DashboardGrid |
| **C3** | DnD 인터랙션 + 자동 알림/재촉 | DnD 표준 래퍼, Nudge 엔진, 마이크로 애니메이션 |
