# 위험·결함 등록부 (Risk Register)

> **대상**: 개발팀 + 인프라팀 + CEO
> **목적**: 인계 후 처리해야 할 모든 알려진 위험·결함의 단일 집계 + PR 분배 가능한 spec 수준 정합
> **원본**: `STATUS.md` §10 시스템 fix 트랙 + 12개 매뉴얼 §9 알려진 제약 + 미등록 cron 5건 + E2E 잔존 14 fail + Phase 9 Cluster D

---

## 분류 체계

| 분류 | 정의 | 처리 시점 |
|------|------|----------|
| **P0** Critical | 법규 미준수 = 규제 리스크 + 사고 시 운영 중단 | 인계 후 30일 안 |
| **P1** High | 핵심 업무 silent fail + 외주 수동 처리 의존 | 다음 분기 |
| **P2** Medium | 우회 가능 + 운영 효율 저하 | 6개월 안 |
| **P3** Low | 개선 제안 + 미래 기능 | 백로그 |

## §1. 법규 정합성 (P0 — 인계 후 30일 안)

CTR Group 규제 노출 직접 영향. 즉시 처리 권장.

### #1.1 한국 출산휴가 60/30 분기 + 우선지원기업 분기 미구현

| 항목 | 내용 |
|------|------|
| **현황** | 시드 `MATERNITY`는 단일 `payType: MIXED, payRate: 100`. 60일 사업주 + 30일 고용보험 자동 분리 코드 없음. 고용보험법 §75 우선지원대상기업 90일 전액 분기도 회사별 코드 없음 |
| **법규** | 근로기준법 §74 + 고용보험법 §75 |
| **영향** | 회계 담당이 외부 수동 처리 — 인계 후 인프라팀이 운영 불가 |
| **영향 모듈** | `payroll`, `loa` |
| **현재 우회** | HR 외부 회계사가 수동 분리 처리 |
| **권장 fix** | (1) `LeaveOfAbsenceType` 시드에 `phaseSplit: [{days: 60, payer: COMPANY, rate: 1.0}, {days: 30, payer: EI, rate: 1.0}]` 구조 추가, (2) `Company.priorityEnterprise: Boolean` 필드 추가, (3) PayrollAdjustment 자동 생성 로직 분기 분리 |
| **참조** | `docs/manuals/loa.md` §9 #12, STATUS §10 #1 |

### #1.2 퇴직금 주 15시간 자격 자동 검증 없음

| 항목 | 내용 |
|------|------|
| **현황** | 코드에 주 평균 근로시간 15시간 미만 직원 자동 제외 로직 없음 |
| **법규** | 근로자퇴직급여보장법 §4① — 4주 평균 1주 소정근로시간 15시간 미만 적용 제외 |
| **영향** | 단시간·계약직 퇴직금 오지급 또는 미지급 → 법적 분쟁 위험 |
| **영향 모듈** | `payroll` (퇴직금 계산) |
| **현재 우회** | HR 수동 자격 확인 |
| **권장 fix** | `calculateSeverance()` 진입부에 4주 평균 시간 산출 + 15시간 미만 시 자격 X 반환 + 명시적 audit log |
| **참조** | STATUS §10 #2 |

### #1.3 강등·감액 시 동의서 강제 + 비율 한도 검증 없음

| 항목 | 내용 |
|------|------|
| **현황** | `OffCycleCompRequest` Zod 스키마는 `proposedBaseSalary: z.number().positive()`만 검증. 동의서 첨부 강제 필드 부재, 감액 비율 한도 검증 없음, `reasonCategory` enum에 `DEMOTION` 자체 없음 (현재: PROMOTION/RETENTION/MARKET_ADJUSTMENT/ROLE_CHANGE/EQUITY/OTHER) |
| **법규** | 근로기준법 — 임금 감액 제한 + 취업규칙 변경 동의 |

| **영향 모듈** | `compensation` |
| **현재 우회** | 인사담당자 수동 + 노무사 협의 별도 |
| **권장 fix** | (1) `reasonCategory` enum에 `DEMOTION` 추가, (2) `consentFileUrl: string` 필수 필드 추가 (changeType=DEMOTION_COMP 시), (3) 감액 비율 회사별 한도 검증 (`Company.maxSalaryReductionPct`) |
| **참조** | `docs/manuals/compensation.md` §9 #14, STATUS §10 #3 |
