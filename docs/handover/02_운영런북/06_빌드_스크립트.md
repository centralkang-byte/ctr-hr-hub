# 06. Build & npm scripts 카탈로그

> **대상**: 개발팀 + 인프라팀
> **원본**: `package.json` scripts 섹션
> **갱신**: scripts 변경 시 본 문서 수동 동기화 (또는 자동 생성기 추가 — 향후)

## 개발 워크플로

| 명령 | 용도 |
|------|------|
| `npm run dev` | 로컬 개발 서버 (포트 3002). predev에서 SW 버전 자동 bump. |
| `npm run build` | 프로덕션 빌드. prebuild에서 SW 버전 bump. |
| `npm run start` | 프로덕션 빌드 결과 실행 (Vercel은 직접 호출 안 함) |
| `npm run lint` | ESLint. Pre-commit hook이 자동 실행. |
| `npm run analyze` | `ANALYZE=true next build` — 번들 분석 (`@next/bundle-analyzer`) |

## 타입 체크 + 검증

| 명령 | 용도 |
|------|------|
| `npx tsc --noEmit` | TypeScript 컴파일 체크 (pre-commit hook). 항상 0 errors 유지. |
| `npm run i18n:validate` | i18n 키 사용 검증 — 누락·미사용 키 식별 |
| `npm run i18n:fix` | i18n 검증 + 자동 fix |
| `npm run i18n:detect` | 하드코딩 한국어 탐지 (`detect-hardcoded-korean.ts`) |
| `npm run qa:inventory` | QA 인벤토리 생성 (`scripts/qa/inventory.ts`) |

## 데이터베이스

| 명령 | 용도 |
|------|------|
| `npx prisma generate` | Prisma client 재생성 (postinstall 자동) |
| `npx prisma migrate dev --name <name>` | 로컬 마이그레이션 생성 + 적용 |
| `npx prisma migrate deploy` | 프로덕션 마이그레이션 적용 (build 시 자동 또는 수동) |
| `npx prisma migrate status` | 마이그레이션 적용 상태 확인 |
| `npx prisma studio` | 로컬 DB GUI |

## 시드

| 명령 | 용도 |
|------|------|
| `npx tsx prisma/seed.ts` | 전체 시드 (49개 파일) |
| `npm run seed:dev` | 빠른 dev 시드 |
| `npm run seed:validate` | Phase 6B 정합성 검증 (32 invariants) |

## 테스트

| 명령 | 용도 |
|------|------|
| `npm run test:unit` | Vitest unit 테스트 (~525건) |
| `npm run test:unit:watch` | Vitest watch 모드 |
| `npm run test:e2e` | Playwright E2E 전체 |
| `npm run test:api` | Playwright API 프로젝트 (~1,500건) |
| `npm run test:browser` | Playwright browser 프로젝트 |
| `npm run test:visual` | Playwright visual regression (desktop+tablet+mobile, ~330 baselines) |
| `npm run test:visual:update` | Visual baselines 갱신 (의도된 변경 시) |
| `npm run test:e2e:ui` | Playwright UI 모드 (debug용) |
| `npm run test:e2e:report` | 직전 E2E 리포트 표시 |

## 부하 테스트 (k6)

| 명령 | 용도 |
|------|------|
| `npm run test:load:smoke` | smoke (가벼운 부하) |
| `npm run test:load` | load (정상 부하) |
| `npm run test:load:stress` | stress (한계 부하) |

⚠️ k6 부하 테스트는 **Vercel Deployment Protection 해제 후** staging/prod에서 실행. CI 에서는 실행 X. STATUS.md Phase 8 참조.

## 자동 실행 hooks

| Hook | 실행 시점 | 동작 |
|------|----------|------|
| `predev` | `npm run dev` 직전 | `scripts/bump-sw-version.mjs` — SW 버전 bump |
| `prebuild` | `npm run build` 직전 | 동일 |
| `postinstall` | `npm install` 후 | `prisma generate` + SW 버전 bump |

## 핸드오버 자동 생성 스크립트 (Tier 1)

| 명령 | 용도 |
|------|------|
| `npx tsx scripts/handover/extract-cron-catalog.ts` | Cron 카탈로그 재생성 |
| `npx tsx scripts/handover/extract-env-inventory.ts` | 환경 변수 인벤토리 재생성 |

## 신규 npm script 추가 시

1. `package.json` scripts 섹션에 추가
2. 본 문서에 한 줄 추가 (또는 향후 자동 생성기로 대체)
3. CI 에서 호출하는 경우 `.github/workflows/*.yml` 업데이트
