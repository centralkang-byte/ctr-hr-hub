## 1. 낙관적 UI (SWR mutate) — Priority: HIGH

### 현황
- SWR 2.4.0 설치됨, 코드에서 **미사용** (전부 useEffect + fetch 패턴)
- mutation이 있는 Client 컴포넌트: 24개 (dashboard 11 + components 13)
- 전면 SWR 전환은 범위 과대 → **승인/취소 등 고빈도 액션만 선별 적용**

### 적용 대상 (사용자 체감 최대)
| # | 대상 | 현재 | 개선 |
|---|------|------|------|
| O-1 | 휴가 승인/반려 | fetch → reload | 즉시 상태 뱃지 변경 → 백그라운드 sync |
| O-2 | 출퇴근 기록 (출근/퇴근 버튼) | fetch → setLoading | 버튼 즉시 비활성 + 시간 표시 → sync |
| O-3 | 승인함 벌크 승인 | Promise.allSettled → reload | 진행률 바 유지 + 각 건 즉시 체크 |
| O-4 | 온보딩 태스크 완료 체크 | fetch → 전체 리로드 | 체크박스 즉시 반영 → 진행률 바 업데이트 |

### 구현 방식
- `useSWR` 래퍼 훅 `useApiData(url, options)` 생성
- `mutate()` + `optimisticData` + `rollbackOnError` 패턴
- 기존 useEffect+fetch 컴포넌트는 점진적 마이그레이션 (이번 세션은 O-1~O-4만)
