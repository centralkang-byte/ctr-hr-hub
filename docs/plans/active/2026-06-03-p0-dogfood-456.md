## 결정 (확정)

- **④ workType**: `attendance.work_type` 컬럼은 `WorkType`(NORMAL/OVERTIME/NIGHT/HOLIDAY, 급여 OT분류). 보정 API가 `REGULAR/REMOTE/FIELD/BUSINESS_TRIP`(근무장소)로 검증→DB 거부. → **보정 enum을 DB/급여와 일치(NORMAL/OVERTIME/NIGHT/HOLIDAY)로 교정**. 근무장소는 이미 `EmployeeAssignment.workLocationId`로 모델됨 → attendance 불필요.
- **⑤ positionId**: 입사 시 초기 Assignment에 `positionId`(직위) 누락 → 보고라인(`Position.reportsToPositionId`) 끊김. → **create 경로에 positionId(optional) 추가 + event payload `positionId=jobGradeId` 버그 수정**.
