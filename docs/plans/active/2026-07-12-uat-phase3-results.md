## 🔴 P1 (수정 필요)

1. **직원 정보 수정 불가 (영문이름 없는 직원)** — `src/lib/schemas/employee.ts:24` `nameEn`만 `.nullable()` 누락(다른 optional 필드는 전부 nullable). 클라(`EmployeeDetailClient.tsx:271`)는 `nameEn || null` 전송 → PUT 400. 영문이름 없이 등록(선택 입력)한 직원은 즉시 수정 불가. 인라인 에러는 표시되나 "잘못된 요청 데이터입니다"로 원인 불명. **1줄 수정.**
2. **계약 이력 페이지 무한 fetch 루프** — `employees/[id]/contracts/ContractsClient.tsx:103` 렌더 본문에서 `loadContracts()` 직접 호출(useEffect 아님). 계약 0건 직원이면 "처리 중..." 고착 + API 폭주(관측 1,500회+). 컴포넌트 규칙 위반. + 직원 상세에 "계약" 탭 진입 동선 자체가 없음.
3. **수습/계약 만료 라이프사이클 미노출 (systemic)** — probation/contract 필드가 DB·API에 있으나 UI 0곳 노출: 목록 상태필터에 수습 없음(API도 미지원), 상세에 수습 배지·만료일 없음, 계약 만료 경과(이계약 2026-05-01, 최만기 2026-03-31)에도 무경고·상태 "재직", 대시보드 위젯 없음. 반면 목록 일부 행은 raw `PROBATION` 노출(표시 경로 이원화). 테스트북 4스텝(인사 13~16) 전부 FAIL.
4. **휴가 신청 드로어 잔액 프리뷰 오표시** — `/api/v1/leave/balances`는 remaining 4 정확 반환하나 드로어 배너는 "현재 잔여 41 / 신청 11 / 잔여 31"류 오표시(산술도 불일치). `leave/LeaveClient.tsx:251-270` balancePreview 매칭/계산 결함. 신청자 판단 오도.
