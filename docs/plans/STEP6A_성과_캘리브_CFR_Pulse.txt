================================================================
EMS 9블록 모델 개요
================================================================

CTR 성과 평가 = EMS (Evaluation & Management System)

2축 평가:
- X축: 성과 (Performance) — MBO 달성률 기반 (1~5)
- Y축: 역량 (Competency) — 행동 역량 평가 (1~5)

9블록 매핑:
```
역량 ↑
  5 │ 7-Rising Star  │ 8-Strong       │ 9-Top Talent
    │                │ Contributor    │ (Star)
  ──┼────────────────┼────────────────┼────────────────
    │ 4-Development  │ 5-Core         │ 6-High
    │ Needed         │ Contributor    │ Performer
  ──┼────────────────┼────────────────┼────────────────
  1 │ 1-Under        │ 2-Inconsistent │ 3-Specialist
    │ Performer      │                │
    └────────────────┴────────────────┴────────────────→ 성과
      1                               5
```

블록 번호 1~9, 각 블록에 라벨 + 색상 + 설명 + 권장 액션:
- 1 (Under Performer): 🔴 PIP 검토, 코칭 강화
- 2 (Inconsistent): 🟠 성과 원인 분석, 목표 재설정
- 3 (Specialist): 🟡 역량 개발 프로그램 투입
- 4 (Development Needed): 🟠 역량+성과 코칭
- 5 (Core Contributor): 🟢 유지, 점진적 성장 지원
- 6 (High Performer): 🔵 승진 검토, 도전적 목표 부여
- 7 (Rising Star): 🟡 성과 코칭, 잠재력 활용
- 8 (Strong Contributor): 🔵 핵심 인재풀 편입
- 9 (Top Talent / Star): 🟣 후계자 후보, 리텐션 집중

**ems_block_config (STEP 1 Seed)**
- thresholds: [0, 2.33, 3.67, 5.01]
  → Low: 0 ≤ score < 2.33
  → Mid: 2.33 ≤ score < 3.67
  → High: 3.67 ≤ score < 5.01
- 성과 Low + 역량 Low = 블록 1
- 성과 Mid + 역량 High = 블록 8
- 성과 High + 역량 High = 블록 9
- ... (총 9가지 조합)

### 1. 평가 사이클 관리 (/settings/performance-cycles) — HR_ADMIN

**사이클 CRUD**
- DataTable: 이름 / 연도 / 유형 / 기간 / 상태 / 법인
- 생성 폼:
  - 이름 (예: "2025년 상반기 평가")
  - 연도
  - 유형: ANNUAL / SEMI_ANNUAL / QUARTERLY / PROBATION
  - 기간: 시작일 ~ 종료일
  - MBO 목표 설정 기간: 시작 ~ 마감
  - 자기평가 기간: 시작 ~ 마감
  - 매니저 평가 기간: 시작 ~ 마감
  - 캘리브레이션 기간: 시작 ~ 마감
  - 결과 확정일

**사이클 상태 머신**

DRAFT → GOAL_SETTING → IN_PROGRESS → SELF_EVAL → MANAGER_EVAL → CALIBRATION → FINALIZED

### 2. MBO 목표 관리

#### 2-1. 목표 설정 (/performance/goals) — EMPLOYEE

**내 목표**
- 현재 사이클의 내 MBO 목표 목록
- 목표 추가 (사이클 상태 = GOAL_SETTING 시만):
  - 목표명 (필수)
  - 설명 (Textarea)
  - 카테고리: BUSINESS / DEVELOPMENT / INNOVATION / TEAM
  - 가중치 (%, 전체 합 = 100)
  - 측정 지표 (KPI)
  - 목표값 (숫자 또는 텍스트)
  - 기한
  - CTR 핵심가치 연결 (선택, CtrValue 태그)
- mbo_goals INSERT

**목표 승인 플로우**
1. EMPLOYEE 목표 입력 → status=DRAFT
2. "제출" → status=SUBMITTED → 매니저 알림
3. MANAGER 검토:
   - 승인 → status=APPROVED
   - 수정 요청 → 코멘트 + status=REVISION_REQUESTED → 직원 알림
4. EMPLOYEE 수정 후 재제출 → status=SUBMITTED
5. 가중치 합계 ≠ 100% → 제출 불가 (프론트 + 백엔드 검증)

#### 2-2. 목표 진행률 관리 — EMPLOYEE

**진행률 업데이트**
- 목표별 "진행 기록 추가" 버튼
- 폼: 현재 달성값, 메모, 날짜
- mbo_progress INSERT
- 진행률 자동 계산: (current_value / target_value) × 100
  → mbo_goals.current_value, achievement_rate 업데이트

**진행률 차트**
- 목표별 라인 차트 (X: 날짜, Y: 달성률)
- 마감 기한 점선 표시

#### 2-3. 매니저 목표 관리 (/performance/team-goals) — MANAGER

**팀 목표 현황**
- DataTable: 직원명 / 목표 수 / 평균 달성률 / 제출 상태
- 직원 클릭 → 해당 직원의 목표 상세
- 인라인 승인/수정요청 액션

### 3. 성과 평가

#### 3-1. 자기평가 (/performance/self-eval) — EMPLOYEE

**사이클 상태 = SELF_EVAL 시 진입 가능**

**평가 폼**
- 목표별 섹션:
  - 목표명 + 가중치 표시
  - 자기 달성도 점수 (1~5, 라디오 또는 슬라이더)
  - 자기 평가 코멘트 (Textarea)
  - 증빙/성과물 첨부 (S3 업로드, 선택)

- 역량 평가 섹션:
  - 역량 항목 5개 (회사 공통):
    1. 전문성 (Expertise)
    2. 리더십/협업 (Leadership/Collaboration)
    3. 문제해결 (Problem Solving)
    4. 커뮤니케이션 (Communication)
    5. CTR 핵심가치 실천 (Core Values)
  - 항목별 자기 점수 (1~5) + 코멘트

- 종합 자기평가 코멘트 (Textarea)

**제출**
- "제출" → performance_evaluations INSERT:
  - evaluator_id = self
  - eval_type = SELF
  - 각 목표 점수 + 역량 점수 JSON 저장
  - status = SUBMITTED
- 제출 전 임시저장 가능 (status=DRAFT)

#### 3-2. 매니저 평가 (/performance/manager-eval) — MANAGER

**사이클 상태 = MANAGER_EVAL 시 진입 가능**

**팀원 평가 목록**
- DataTable: 직원명 / 자기평가 완료 여부 / 매니저 평가 상태
- 직원 클릭 → 평가 폼

**평가 폼**
- 좌측: 직원의 자기평가 내용 표시 (읽기 전용)
- 우측: 매니저 평가 입력

- 목표별 섹션:
  - 자기평가 점수 표시 (참고)
  - 매니저 달성도 점수 (1~5)
  - 매니저 코멘트

- 역량 평가:
  - 자기평가 점수 표시 (참고)
  - 매니저 점수 (1~5)
  - 매니저 코멘트

- **성과 종합 점수**: 목표별 (매니저 점수 × 가중치) 합산 → 1~5
- **역량 종합 점수**: 역량 5개 평균 → 1~5
- **EMS 블록 자동 산출**: performance_score + competency_score → 9블록 매핑
  - ems_block_config의 thresholds 적용
  - 예: performance=4.2(High), competency=3.0(Mid) → 블록 6 (High Performer)

- 종합 코멘트 (Textarea)
- EMS 블록 결과 미리보기 (블록 번호 + 라벨 + 색상 + 권장 액션)

**AI 평가 코멘트 보조**
- "AI 코멘트 제안" 버튼 → lib/claude.ts:
```ts
suggestEvalComment(

    title: string,
    weight: number,

    achievement_rate: number
  }>,
  competency_scores: Array<{
    name: string,
    self_score: number,
    manager_score: number
  }>,
  ems_block: number,
  ems_label: string
): Promise<{
  overall_comment: string,      // 종합 평가 코멘트 초안
  strengths: string[],
  development_areas: string[],
  recommended_actions: string[]
}>
prompt_version: "eval-comment-v1.0"
```
- AiGeneratedBadge + HR이 수정 가능
- ai_logs (EVAL_COMMENT_SUGGESTION) 기록

**제출**
- performance_evaluations INSERT:
  - evaluator_id = manager
  - eval_type = MANAGER
  - performance_score, competency_score, ems_block
  - status = SUBMITTED

### 4. 캘리브레이션

#### 4-1. 캘리브레이션 규칙 (/settings/calibration) — HR_ADMIN

**규칙 설정**
- calibration_rules CRUD:
  - 사이클 선택
  - 법인 선택
  - 블록별 비율 가이드라인 (선택):
    - 블록 9: ≤5%
    - 블록 8: ≤10%
    - 블록 6: ≤15%
    - 블록 5: 40~50%
    - 블록 1: ≤5%
    - (나머지는 자율)
  - 강제 배분 여부 (Boolean):
    - true → 비율 초과 시 경고 (저장은 허용, Phase 1)
    - false → 가이드라인만 참고

#### 4-2. 캘리브레이션 세션 (/performance/calibration) — HR_ADMIN + EXECUTIVE

**사이클 상태 = CALIBRATION 시 진입 가능**

**세션 관리**
- calibration_sessions CRUD:
  - 이름, 사이클, 대상 부서(들), 참석자 (HR + 임원)
  - 상태: SCHEDULED / IN_PROGRESS / COMPLETED

**9블록 매트릭스 시각화**
- 3×3 그리드 (인터랙티브):
  - 각 셀: 블록 번호 + 라벨 + 인원 수 + 비율(%)
  - 셀 클릭 → 해당 블록 직원 목록 드롭다운
  - 직원 카드: 이름 + 부서 + 점수 + 현재 블록
- 비율 가이드라인 vs 실제 비교 바 차트 (옆에 표시)
- 초과 블록: 🔴 경고 배지

**조정 (Adjustment)**
- 직원 카드 드래그 → 다른 블록으로 이동
- 또는 직원 클릭 → 조정 모달:
  - 기존 블록 → 변경 블록
  - 조정 사유 (Textarea, 필수)
  - "적용" → calibration_adjustments INSERT:
    - from_block, to_block, reason, adjusted_by
  - performance_evaluations.ems_block 업데이트

**AI 미팅 노트 생성**

★ v3.2 추가 — 다면평가 (360° Feedback)
================================================================

### 개요
기존 성과 관리(자기평가+매니저 평가)에 "다면평가" 레이어 추가.
핵심 혁신: **AI 기반 평가자 자동 추천** — 협업 데이터로 적합한 피어 선정.

### 다면평가 유형
- 90°: 상사 → 부하 (기존 매니저 평가)
- 180°: 상사 ↔ 부하 (역평가)
- 270°: 상사 + 동료
- **360°**: 상사 + 동료 + 부하 + 자기 (CTR 기본값)

### 다면평가 설정 (/settings/peer-review)
**권한**: HR_ADMIN

- 평가 사이클 연결: 기존 performance_cycles에 peer_review_enabled(Boolean) 추가
- 평가자 수 설정:
  - 동료(Peer): 최소 2명, 최대 5명
  - 부하(Direct Report): 부하 전원 또는 최대 N명
  - 상사: 직속 상사 자동 배정
- 익명 설정: 동료/부하 평가는 기본 익명 (anonymous_peer_review=true)
- 평가 질문 템플릿: 관계 유형별 질문 세트

### 평가 질문 템플릿
**동료 (Peer) 기본 질문 8개:**
1. 이 동료와의 협업 경험은 어떠했습니까? (RATING 1~5)
2. 업무 전문성은 어떻습니까? (RATING 1~5)
3. 커뮤니케이션 능력은 어떻습니까? (RATING 1~5)
4. 팀워크와 협조성은 어떻습니까? (RATING 1~5)
5. CTR 핵심가치(도전/신뢰/책임/존중) 중 가장 잘 실천하는 것은? (SELECT)
6. 이 동료의 가장 큰 강점은? (TEXT)
7. 개선이 필요한 영역은? (TEXT)

★ v3.2 핵심 — 평가자 자동 추천 엔진
================================================================

### 2단계 접근: 기본 규칙 + 협업 점수

#### Phase 1A: 기본 규칙 (시스템 초기부터 작동)

**동료(Peer) 후보 풀 생성 규칙:**
1. 같은 법인 (company_id 일치) — 필수
2. 같은 부서 또는 인접 부서 (같은 상위 부서 하위)
3. 비슷한 직급 (±1 등급)
4. 본인 제외, 직속 상사/부하 제외 (이미 별도 평가)
5. ACTIVE 상태

**기본 점수 산출:**
```
base_score = 
  (같은 부서 ? 30 : 같은 상위 부서 ? 15 : 0) +
  (직급 차이 0 ? 20 : 직급 차이 1 ? 10 : 0) +
  (같은 직군 ? 15 : 0) +
  (재직 기간 6개월 이상 ? 10 : 0)
```

#### Phase 1B: 협업 데이터 점수 (데이터 축적 시 활성화)

**collaboration_scores 산출 — pg_cron 매일 새벽 4시**

5가지 협업 신호:
1. **Recognition 주고받기** (가중치 30%)
   - recognitions 테이블에서 지난 12개월 간 sender↔receiver 쌍 카운트
   - 양방향 합산 (A→B + B→A)

2. **1:1 미팅** (가중치 25%)
   - one_on_ones 테이블에서 지난 12개월 간 manager↔employee 쌍 카운트
   - 직속이 아닌 1:1도 포함 (크로스 펑셔널)

3. **같은 부서** (가중치 20%)
   - employees.department_id 일치 → 고정 점수

4. **면접 동석** (가중치 15%)
   - interview_evaluations에서 같은 지원자를 평가한 면접관 쌍 카운트

5. **법인 간 전입 이력 공유** (가중치 10%)
   - employee_histories에서 TRANSFER_CROSS_COMPANY 시 from_company=to_company 관계

**산출 수식:**
```
collaboration_total = 
  (recognition_count / max_recognition * 0.30) +
  (one_on_one_count / max_one_on_one * 0.25) +
  (same_department ? 1 : 0) * 0.20 +
  (interview_panel_count / max_panel * 0.15) +
  (cross_transfer_shared ? 1 : 0) * 0.10

→ 정규화 0~100
```

**Fallback 규칙:**
- Recognition 0건 + 1:1 0건 → Phase 1A 기본 점수만 사용
- 데이터 충분 기준: 해당 직원의 collaboration_scores 레코드 3개 이상

#### 최종 추천 점수
```
final_score = 
  IF 협업데이터 충분:
    base_score * 0.3 + collaboration_total * 0.7
  ELSE:
    base_score * 1.0
```
→ 상위 N명 (설정값, 기본 5명) 추천

### EMS 9블록 확장 (선택적)
기존: performance_score(MBO) × competency_score → 9블록
v3.2 옵션: peer_review_avg를 competency_score에 가중 반영

```
adjusted_competency = 
  competency_score * (1 - peer_weight) + peer_review_avg * peer_weight

기본 peer_weight = 0.3 (설정 가능)
→ 매니저 역량 평가 70% + 동료 평가 30%
```

HR_ADMIN이 사이클별로 peer_weight 설정 (0=미반영, 0.3=기본, 0.5=동등)

### 캘리브레이션 연동
