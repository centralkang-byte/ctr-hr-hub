## 1. 왜 만드는가

### 현재 급여 업무의 문제

CTR 그룹은 6개국 7개 법인에 약 1,000~3,000명의 임직원이 있습니다. 현재 매달 급여 처리 시 각 법인 HR 담당자가 수행하는 업무는 다음과 같습니다:

- 근태 시스템에서 출퇴근 데이터를 엑셀로 추출
- 휴가 시스템에서 무급휴가/병가 데이터를 별도 추출
- 두 데이터를 수작업으로 합치고 검증
- 초과근무 수당, 야간수당, 휴일수당을 수작업 계산
- 급여 시스템(더존/SAP 등)에 하나하나 입력
- 4대보험, 소득세 계산 결과를 사람이 재검증
- 부서장에게 개별 확인 (전화, 메일)
- 수정 → 재계산 → 재검증... 2~3일 반복
- 임원 결재 → 은행 이체 파일 생성 → 급여 지급
- 명세서 PDF를 개별 생성하여 이메일/그룹웨어로 발송

이 과정에 **매월 5~7일**이 소요되며, 6개 법인을 관리하는 본사 인사팀은 법인별 취합 및 보고에 추가 시간을 투입하고 있습니다.

### GP#3의 목표

**HR 담당자의 역할을 "계산하는 사람"에서 "검증하는 사람"으로 전환합니다.**

| 항목 | 현재 | GP#3 이후 |
|------|------|----------|
| 근태 데이터 취합 | 엑셀 추출 + 수작업 합치기 | 버튼 하나로 근태 마감 |
| 수당 계산 | 수작업 | 시스템 자동 계산 |
| 이상 항목 발견 | 사람이 눈으로 검토 | 시스템이 자동 감지 + 하이라이트 |
| 승인 | 종이/메일 결재 | 시스템 내 승인 버튼 |
| 명세서 발송 | 개별 생성 + 발송 | 자동 생성 + 자동 알림 |
| 소요 시간 | 매월 5~7일 | 매월 1~2일 (검토 중심) |

## 2. 현재 vs 미래

### Before: 단절된 모듈

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  근태     │     │  급여     │     │  명세서   │
│  시스템   │ ──? │  계산    │ ──? │  발행    │
│          │     │          │     │          │
│ (엑셀)   │     │ (더존)   │     │ (PDF)   │
└──────────┘     └──────────┘     └──────────┘
     ↑ 수작업          ↑ 수작업          ↑ 수작업
```

각 시스템이 분리되어 있어 매번 사람이 데이터를 옮겨야 합니다.

### After: 하나의 파이프라인

```
┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
│STEP 1│───▶│STEP 2│───▶│STEP  │───▶│STEP 3│───▶│STEP 4│───▶│STEP 5│
│ 근태 │    │ 자동 │    │ 2.5  │    │ 이상 │    │      │    │명세서│
│ 마감 │    │ 계산 │    │수동  │    │ 검토 │    │ 승인 │    │ 발행 │
│      │    │      │    │조정  │    │      │    │      │    │+이체 │
└──────┘    └──────┘    └──────┘    └──────┘    └──────┘    └──────┘
  HR 클릭     자동       HR 입력     HR 검토    HR/임원      자동
```

HR 담당자가 하는 일: **마감 클릭 → 추가/수정 입력 → 이상 확인 → 승인 요청**. 나머지는 시스템이 자동 처리합니다.

## 3. 전체 파이프라인 흐름

### 6단계 상세

| 단계 | 담당 | 설명 | 소요 시간 |
|------|------|------|----------|
| **STEP 1** 근태 마감 | HR 담당자 | 해당 월 근태 데이터 확정, 수정 불가 잠금 | 10분 |
| **STEP 2** 급여 자동계산 | 시스템 | 기본급+수당-공제 자동 산출 | 자동 (1~2분) |
| **STEP 2.5** 수동 조정 | HR 담당자 | 소급 지급, 경조사비, 수정 항목 입력 | 30분~2시간 |
| **STEP 3** 이상 검토 | HR 담당자 | 시스템이 감지한 이상 항목 확인/처리 | 30분~1시간 |
| **STEP 4** 승인/확정 | HR 팀장/임원 | 최종 금액 확인 후 승인 | 10분 |
| **STEP 5** 명세서 발행 | 시스템 | 명세서 자동 생성 + 직원 알림 + 이체파일 | 자동 |

### 상태 흐름

```
PayrollRun 상태:

  DRAFT → CALCULATING → ADJUSTMENT → REVIEW → PENDING_APPROVAL → APPROVED → PUBLISHED
   (생성)    (계산중)      (수동조정)   (검토중)     (승인대기)        (확정)     (발행완료)
```

각 상태에서 이전 단계로 되돌리기 가능 (실수 대비). 단, PUBLISHED 이후에는 되돌릴 수 없으며, 수정이 필요하면 다음 달에 소급 처리합니다.

## 4. STEP 1: 근태 마감

### 개요

급여 계산의 출발점입니다. "이번 달 근태 데이터가 확정되었다"는 기준을 세우는 단계입니다. 마감하지 않으면 급여 산출 후 누군가 출퇴근을 수정할 경우 숫자가 불일치합니다.

### 화면 구성

**위치:** `/payroll/close-attendance`

**상단:** 월 선택기 (`◀ 2026년 3월 ▶`)

**본문:** 법인별 마감 현황 카드

```
┌─────────────────────────────────────────────────────────┐
│  CTR-KR                                                  │
│  대상: 70명 │ 확정: 68명 │ 미확정: 2명 ⚠️               │
│  ████████████████████████████░░  97%                     │
│                                                          │
│  미확정 사유:                                             │
│  · 김영수 — 3/28 출근 기록 없음                            │
│  · 이민호 — 3/15~17 휴가 미승인                            │
│                                                          │
│           [미확정 직원 제외하고 마감]  [전체 마감하기]       │
├─────────────────────────────────────────────────────────┤
│  CTR-CN                                                  │
│  대상: 18명 │ 확정: 18명 │ 미확정: 0명 ✅                │
│  ██████████████████████████████  100%                    │
│                                                          │
│                                          [마감하기]       │
└─────────────────────────────────────────────────────────┘
```

### 마감 전 확인 모달

[마감하기] 클릭 시 요약 정보를 확인합니다:

```

│                                              │
│  총 근무일: 22일                              │
│  대상 인원: 68명 (2명 제외)                    │
│  총 정규 근무시간: 12,496시간                  │
│  총 초과근무시간: 342시간                      │
│  무급휴가: 5건                                │
│                                              │
│  ⚠️ 마감 후에는 해당 월 근태 수정이 불가합니다.  │
│     (마감 해제는 급여 계산 전까지만 가능)        │
│                                              │
│              [취소]  [마감 확정]               │
└─────────────────────────────────────────────┘
```

### 주요 규칙

- **미확정 직원이 있어도 마감 가능:** "미확정 직원 제외하고 마감" 옵션 제공. 한두 명 때문에 전체 급여가 지연되면 안 됩니다.
- **마감 해제:** 급여 계산(STEP 2) 시작 전까지만 해제 가능. 이후에는 해제 불가.
- **잠금 효과:** 마감된 월의 출퇴근 기록은 수정/삭제 불가. 수정이 필요하면 다음 달 소급 처리.

## 5. STEP 2: 급여 자동계산

### 개요

근태 마감 직후 시스템이 자동으로 실행합니다. HR 담당자가 별도로 버튼을 누를 필요 없이, 마감 이벤트에 의해 자동 트리거됩니다. (수동 트리거도 가능)

### 계산 항목 (한국 법인 기준)

**지급 항목:**

| 항목 | 계산 방식 |
|------|----------|
| 기본급 | 월급 ÷ 소정근로일 × 실제근무일 |
| 연장근무수당 | 통상시급 × 1.5 × 연장근무시간 |
| 야간근무수당 | 통상시급 × 0.5 × 야간시간(22시~06시) |
| 휴일근무수당 | 통상시급 × 1.5 × 휴일근무시간 |
| 직책수당 | 직급별 고정금액 |
| 식대 | 월 200,000원 (비과세) |
| 교통비 | 월 100,000원 (비과세) |

**공제 항목:**

| 항목 | 계산 방식 |
|------|----------|
| 국민연금 | 기준소득월액 × 4.5% |
| 건강보험 | 보수월액 × 3.545% |
| 장기요양보험 | 건강보험료 × 12.81% |
| 고용보험 | 월 급여 × 0.9% |
| 소득세 | 간이세액표 기준 |
| 지방소득세 | 소득세 × 10% |

**무급휴가/결근 차감:**

| 항목 | 계산 방식 |
|------|----------|
| 무급휴가 차감 | 일급 × 무급휴가 일수 |
| 결근 차감 | 일급 × 결근 일수 (무급휴가와 중복 시 1회만 차감) |

### 화면 구성

**위치:** `/payroll/run`

```

│  2026년 3월 급여 계산                                     │
│                                                          │

│  CTR-CN  ████████████████░░░░░░░░░░░  12/18 진행중...    │
```

## 6. STEP 2.5: 수동 조정

### 왜 필요한가?

매달 전체 인원의 **5~10%**에서 자동계산에 포함되지 않는 추가/수정 항목이 발생합니다. 이는 예외가 아니라 **정규 업무**입니다.

### 주요 사례

| 유형 | 예시 | 빈도 |
|------|------|------|
| 소급 지급 | 지난달 야근 누락분 추가 | 매월 발생 |
| 경조사비 | 결혼축의금, 조위금 등 | 수시 |
| 수당 수정 | 직책수당 적용 시기 오류 정정 | 간헐적 |
| 특별 지급 | 성과급, 프로젝트 보너스 | 분기/반기 |
| 차감 수정 | 과지급분 환수 | 간헐적 |

### 화면 구성

**위치:** `/payroll/adjustments`

```
┌─────────────────────────────────────────────────────────┐
│  2026년 3월 수동 조정                    총 12건 등록됨   │
│                                                          │
│  [+ 조정 추가]  [일괄 업로드 (엑셀)]                      │
│                                                          │
│  ┌────────────────────────────────────────────────────┐  │
│  │ 직원        유형       항목         금액      사유   │  │
│  ├────────────────────────────────────────────────────┤  │
│  │ 김영수     소급지급   연장근무수당  +380,000  2월분  │  │
│  │ 박지현     경조사비   결혼축의금    +500,000  본인   │  │
│  │ 이승우     차감       직책수당 오류  -200,000  3월분  │  │
│  │ 최민아     소급지급   야간수당      +125,000  2월분  │  │
│  │ ...                                                │  │
│  └────────────────────────────────────────────────────┘  │
```

- 모든 조정에는 **사유 필수** 입력 — 감사 추적용

- 조정 완료 후 시스템이 자동으로 재계산하여 최종 금액에 반영

## 7. STEP 3: 이상 검토

### 왜 이 단계가 핵심인가?

HR 담당자가 **시스템을 믿을 수 있는지** 판단하는 순간입니다. 이상한 것을 정확히 잡아주면 신뢰가 생기고, 놓치면 "차라리 엑셀이 낫다"가 됩니다.

### 화면 구성

**위치:** `/payroll/review`

**상단: 요약 KPI 카드**

```
┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│   총 급여     │ │  평균 급여    │ │  이상 항목    │ │  전월 대비    │
```

| 3 | 법정 한도 초과 | 주 52시간 초과 근무 반영 | 🔴 위험 | 허용 오차 없음 |
| 4 | 극단적 기본급 | 무급휴가/결근으로 기본급 50% 이하 | ⚠️ 확인 | 허용 오차 없음 |
| 5 | 신규 입사자 첫 급여 | 일할계산 정확성 자동 검증 | ℹ️ 정보 | 1,000원 미만 차이 무시 |
| 6 | 퇴직자 최종 급여 | 퇴직월 일할계산 + 미사용 연차 정산 | ℹ️ 정보 | 1,000원 미만 차이 무시 |

### [v1.1] 허용 오차 설정 (Tolerance)

> 4대보험 요율 반올림, 세금 단수 차이 등으로 매달 10~50원 수준의 미세 변동이 발생합니다.
> 이런 자잘한 차이 때문에 HR 담당자가 매번 "정상 확인"을 누르는 피로를 줄여야 합니다.

**설정 원칙:**

각 이상 탐지 규칙에 **금액 기준 + 비율 기준 이중 필터**를 적용합니다.

- **금액 기준:** 변동 금액이 N원 미만이면 이상 항목에서 제외
- **비율 기준:** 변동 비율이 N% 이내이면 이상 항목에서 제외
- **둘 다 충족해야 제외** — 비율로만 하면 저소득자가 과잡히고, 금액으로만 하면 고소득자가 빠짐

**예시:**
```
규칙: 전월 대비 ±30% 변동
허용 오차: 5만원 미만

김사원 — 전월 3,000,000원 → 이번달 3,030,000원 (+1%, +30,000원)
  → 비율 1%로 30% 미만 → 정상 (오차 이전에도 안 잡힘)

박인턴 — 전월 200,000원 → 이번달 280,000원 (+40%, +80,000원)
  → 비율 40%로 30% 초과 + 금액 8만원으로 5만원 초과 → ⚠️ 이상 항목

이대리 — 전월 3,500,000원 → 이번달 3,520,000원 (+0.6%, +20,000원)
  → 비율 0.6%로 30% 미만 → 정상
```

**허용 오차 기본값은 시스템에서 제공하되, 향후 Settings에서 법인별로 조정 가능합니다.**

> `// TODO: Move to Settings (Payroll) — 이상 탐지 허용 오차 범위 (금액/비율)`

### 예외 등록 (Whitelist)

매달 반복적으로 잡히지만 문제가 아닌 케이스(예: CEO 연봉이 밴드 초과)를 매번 "정상 확인" 누르는 것은 비효율적입니다.

**예외 등록 기능:**
- "이 직원의 이 항목은 향후 이상으로 잡지 않음" 설정
- 등록 시 사유 + 등록자 + 등록일 기록
- 예외 목록 관리 화면에서 언제든 해제 가능
- 예외 등록된 항목도 감사 로그에는 기록 (투명성 확보)

### 핵심 원칙

**이상 항목이 0건이 되어야 다음 단계(승인 요청)로 넘어갈 수 있습니다.** 모든 이상을 "정상 확인" 또는 "수정"하여 명시적으로 처리해야 합니다. 이를 통해 문제 발생 시 책임 소재를 명확히 합니다.

## 12. 예외 상황 처리

| 상황 | 처리 방식 |
|------|----------|
| 중간 입사자 (3/15 입사) | 일할계산 — 자동으로 해당 일수분만 계산 |
| 중간 퇴사자 (3/20 퇴직) | 일할계산 + 미사용 연차 수당 + 퇴직금은 별도 처리 |
| 근태 마감 후 수정 필요 | 마감 해제(급여 계산 전까지만) → 수정 → 재마감 → 재계산 |
| 급여 승인 후 오류 발견 | 승인 취소(최종 승인자만) → STEP 3으로 되돌림 |
| 명세서 발행 후 오류 발견 | 되돌릴 수 없음 — 다음 달 소급 조정으로 처리 |
| 소급 적용 (전월 누락분) | STEP 2.5 수동조정에서 "소급 지급" 유형으로 등록 |
| 법인별 급여일 차이 | 각 법인 독립 타임라인으로 진행 |
| 환율 변동 | 글로벌 급여 합산 시 해당 월 1일 기준 환율 적용 |

## 15. 피드백 요청 사항

이 설계서에 대해 다음 사항을 검토해 주시기 바랍니다:

### 인사팀에게

1. 현재 급여 처리 시 위 6단계 외에 **빠진 업무**가 있는지?
2. STEP 2.5 "수동 조정"에서 위에 나열한 유형 외에 **추가 유형**이 있는지?
3. 이상 탐지 규칙 6가지가 **현실적인지**, 추가로 필요한 규칙이 있는지?
4. 법인별 승인 체계가 **정확한지** 확인

### 경영지원팀에게

5. 은행 이체파일 포맷이 현재 사용 중인 포맷과 **호환되는지**?
6. 회계 전표용 엑셀의 **계정과목 분류**가 CTR 회계 기준과 일치하는지? (811 급여, 812 제수당, 822 복리후생비, 831 법정복리후생비, 826 퇴직급여)
7. 계정과목 외에 **추가 분류 기준**(코스트센터, 프로젝트 코드 등)이 필요한지?

### 각 법인 HR에게

8. 본인 법인의 급여 계산 규칙(초과근무 배율, 사회보험 등)이 **정확한지** 확인
