### 1. 급여 처리 프로세스 (/payroll)

#### 1-1. 급여 실행 관리 — HR_ADMIN

**급여 실행 목록**
- DataTable: 실행명 / 법인 / 급여 기간 / 대상 인원 / 총액 / 상태
- 상태: DRAFT → CALCULATING → REVIEW → APPROVED → PAID → CANCELLED

**급여 실행 생성**
- 폼:
  - 실행명 (예: "2025년 3월 급여")
  - 법인 선택
  - 급여 기간: 시작일 ~ 종료일 (예: 2025-03-01 ~ 2025-03-31)
  - 지급 예정일
  - 유형: MONTHLY / BONUS / SEVERANCE / SPECIAL
- payroll_runs INSERT (status=DRAFT)

#### 1-2. 급여 계산 — "계산 실행" 버튼

**자동 계산 로직 ($transaction)**

대상: 해당 법인의 ACTIVE + 해당 기간 재직 직원

직원별 계산:
1. **기본급** — compensation_history 최신 레코드 기반 (월 환산)
2. **초과근무 수당** — attendances 기간 내 overtime_hours 합산
   - lib/labor/{country}.ts의 overtime_rates 적용
   - 시간당 기본급 × overtime_hours × multiplier
3. **수당** — allowance_records 기간 내 합산
   - 또는 benefit_policies의 MONTHLY 자동 산출
4. **공제**:
   - 국민연금 (한국: 4.5%)
   - 건강보험 (한국: 3.545%)
   - 장기요양 (건강보험의 12.81%)
   - 고용보험 (한국: 0.9%)
   - 소득세 (간이세액표 기반 — Phase 1: 고정 비율 근사)
   - 지방소득세 (소득세의 10%)
   ★ 해외법인: 각국 법정 공제는 Phase 2 (Phase 1은 총액만)
5. **순지급액** = 기본급 + 초과근무 + 수당 - 공제

**payroll_items 생성 (직원별)**
- employee_id, payroll_run_id
- base_salary, overtime_pay, allowances (JSON 상세)
- gross_pay = base + overtime + allowances
- deductions (JSON: 항목별 공제액)
- net_pay = gross - total_deductions
- currency

**계산 완료** → payroll_runs.status = CALCULATING → REVIEW

#### 1-3. 급여 검토 — HR_ADMIN

**검토 화면 (/payroll/:runId/review)**
- 요약 카드: 대상 인원 / 총 지급액 / 총 공제액 / 총 순지급액
- DataTable:
  직원명 / 부서 / 기본급 / 초과근무 / 수당 / 총 지급 / 공제 / 순 지급
- 이상 항목 하이라이트:
  - 초과근무 > 월 60시간 → 🟡
  - 전월 대비 급여 차이 > 20% → 🔴
  - 신규 입사자 (일할 계산) → ℹ️
- 개별 수정: 행 클릭 → 항목별 수동 조정 + 사유 입력

**AI 급여 이상 감지**
- "AI 검증" 버튼 → lib/claude.ts:
```ts
payrollAnomalyCheck(
  payroll_summary: {
    period: string,
    company: string,
    total_employees: number,
    total_gross: number,
    total_net: number
  },
  anomalies: Array<{
    employee_name: string,
    item: string,
    current_value: number,
    previous_value: number,
    change_pct: number
  }>,
  new_hires: number,
  resignations: number
): Promise<{
  risk_level: 'LOW' | 'MEDIUM' | 'HIGH',
  findings: string[],
  items_to_review: Array<{
    employee_name: string,
    issue: string,
    severity: 'INFO' | 'WARNING' | 'ERROR'
  }>,
  recommendation: string
}>
prompt_version: "payroll-anomaly-v1.0"
```
- AiGeneratedBadge + ai_logs (PAYROLL_ANOMALY_CHECK)

#### 1-4. 승인 + 지급

**승인 — HR_ADMIN 또는 EXECUTIVE**

- audit_logs (PAYROLL_APPROVED)

**지급 완료 처리**
- 실제 이체 후 → "지급 완료" 버튼 → status = PAID
- paid_at 기록
- Phase 2: 은행 API 연동 자동 이체

#### 1-5. 급여 명세서 — EMPLOYEE

**내 급여 명세서 (/payroll/me)**
- 월별 명세서 목록 (최신순)
- 명세서 카드:
  - 기본급 / 초과근무 / 수당 / 총 지급
  - 공제 항목별 내역
  - 순 지급액 (큰 폰트)

  - 법인명 + 직원정보 + 급여 기간 + 항목별 내역 + 서명란
  - lib/pdf.ts → PDFKit 또는 Puppeteer HTML→PDF

### 11. AI Executive Summary

**월간 HR 리포트 자동 생성 — HR_ADMIN + EXECUTIVE**

**트리거**: HR_ADMIN → "월간 리포트 생성" 버튼 (/analytics/report)
또는 매월 1일 자동 생성 (cron)

**lib/claude.ts:**
```ts
generateExecutiveSummary(
  period: string,                    // "2025년 3월"
  company: string,                   // 법인명 또는 "그룹 전체"
  metrics: {
    headcount: { total: number, hires: number, exits: number },
    turnover_rate: number,
    avg_overtime_hours: number,
    overtime_compliance: boolean,     // 법정 기준 초과 여부
    avg_pulse_score: number,
    pulse_trend: 'UP' | 'STABLE' | 'DOWN',
    high_risk_attrition_count: number,
    ems_distribution: Array<{ block: number, count: number }>,
    recruitment_active: number,
    recruitment_hired: number,
    avg_compa_ratio: number,
    training_completion_rate: number,
    key_departures: Array<{ name: string, department: string, ems_block: number }>,
    recognition_count: number,
    discipline_count: number
  }
): Promise<{
  executive_summary: string,        // 3~5문장 요약
  highlights: string[],             // 긍정 하이라이트 3~5개
  concerns: string[],               // 주의 사항 3~5개
  action_items: Array<{
    priority: 'HIGH' | 'MEDIUM' | 'LOW',
    action: string,
    owner: string                    // 담당 부서/역할
  }>,
  trend_analysis: string,           // 전월 대비 추세 분석
  ctr_value_spotlight: {             // CTR 핵심가치 활동 하이라이트
    most_recognized_value: string,
    recognition_trend: string
  }
}>
prompt_version: "executive-summary-v1.0"
```

**결과 표시**
- 리포트 페이지 (/analytics/report/:period):
  - 헤더: CTR 로고 + 기간 + 법인
  - Executive Summary 카드
  - Highlights (🟢) + Concerns (🔴) 리스트
  - Action Items 테이블 (우선순위 뱃지)
  - 추세 분석 본문
  - CTR 핵심가치 스포트라이트
- PDF 다운로드: 전체 리포트 PDF 생성
- AiGeneratedBadge + ai_logs (EXECUTIVE_SUMMARY)
