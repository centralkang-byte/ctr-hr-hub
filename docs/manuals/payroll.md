# 급여 매뉴얼 (Payroll)

> **최종 갱신**: 2026-05-11
> **담당**: 인사관리팀 · 재무팀
> **상태**: Draft v1 (CEO 리뷰 전)

---

## 1. 개요

### 1.1 무엇을 다루는가

CTR HR Hub의 급여 모듈은 다음을 통합 관리합니다.

- **월 급여 계산** — 한국 본사·국내 법인 직접 계산 (기본급 + 수당 + 시간외근무 - 4대보험 - 소득세 = 실수령액)
- **해외 법인 급여** — 외부 처리 결과를 업로드하여 시스템에 통합 (CTR-CN, CTR-VN, CTR-US, CTR-MX, CTR-RU, CTR-EU 모두 동일 방식)
- **수당·공제 마스터** — 회사별 수당·공제 항목 정의 + 직원별 정기 수당 설정
- **급여 밴드(Salary Band)** — 회사·직급·직군별 최소·중간·최대 급여 범위 + 보상 변경 시 검증
- **보상 변경 이력** — 정기 인상·승진·시장 조정·특별 인상 등 모든 보상 변경의 시점별 이력
- **8단계 상태 워크플로** — 근태 마감 → 자동 계산 → 수동 조정 → 이상 검토 → 결재 → 지급
- **다단계 결재** — 본부장 → 대표 (2단계, 결재 매뉴얼 §2.1 참조)
- **이상 검토** — 전월 대비 변동·법정 한도 초과·급여 밴드 초과 등 6종 규칙 자동 검출
- **급여명세서(Payslip) 발급** — PDF 생성, 직원 본인 열람·이메일 발송 추적
- **연말정산** — 한국 11단계 정산 (총급여·소득공제·세액공제·결정세액·기납부세·환급/추가납부)
- **퇴직금 계산** — 최근 3개월 평균임금 × 재직일수 / 365 × 30일
- **다국적 통화** — 7개 통화 지원 (KRW/CNY/USD/MXN/VND/RUB/PLN), 월별 환율 업로드

### 1.2 도입 의도

- **법규 준수 자동화** — 한국 근로기준법·최저임금법·소득세법·4대보험 관련 법규의 시스템 자동 적용
- **다국적 통합 시각화** — 13개 법인의 급여 결과를 한 시스템에서 통화 변환된 KPI로 조회
- **휴먼 에러 방지** — 8단계 워크플로 + 자동 이상 검출로 잘못된 급여 지급 차단
- **감사 가능성** — 모든 수동 조정에 사유·증빙·작성자 기록, 결재 이력 추적
- **드리프트 방지** — 인사 발령(입퇴사·휴직·법인전환)이 발생하면 급여에 자동 반영 (휴직 매뉴얼 §3.3, 인사관리 매뉴얼 §3.5 참조)
- **연말정산 디지털화** — 종이/엑셀 정산을 시스템에서 11단계 자동 계산

### 1.3 인사관리·휴직 매뉴얼과의 관계

| 항목 | 본 매뉴얼 (급여) | 인사관리 / 휴직 매뉴얼 |
|------|-----------------|----------------------|
| 직원의 현재 급여 | 직원별 기본급 + 정기 수당 (보상 이력의 최신값) | — |
| 보상 변경 | 본 매뉴얼이 정의 (정기·승진·시장·특별) | 인사관리는 발령만, 급여 변경은 본 매뉴얼 |
| 휴직 중 급여 | 자동 조정 (월별 분리, 일할 차감) | 휴직 유형별 급여 정책은 휴직 매뉴얼 §3.3 |
| 법인전환 시 급여 | 도착 법인 급여 밴드 재조정 | 정책은 인사관리 매뉴얼 §3.5 |
| 퇴직금 정산 | 본 매뉴얼이 정의 (마지막 3개월 평균) | 인사관리는 퇴직 발령만, 정산은 본 매뉴얼 |

## 2. 급여 구성 요소

### 2.1 급여 항목 4종

| 카테고리 | 의미 | 예시 |
|----------|------|------|
| 기본급 (Base) | 직원의 정규 급여 | 월 보상의 핵심, 직급별 차등 |
| 수당 (Allowance) | 정기·비정기 지급 | 식대, 차량유지비, 직책수당, 야간수당, 출산수당 |
| 상여 (Bonus) | 성과·명절 등 비정기 | 연간 성과급, 명절 상여, 경조사비 |
| 공제 (Deduction) | 실수령액에서 차감 | 4대보험, 소득세, 지방세, 노조비, 식대 자기부담 |

### 2.2 수당 마스터 (PayAllowanceType)

회사별로 사용 가능한 수당 종류를 정의합니다.

| 속성 | 설명 |
|------|------|
| 코드·이름 | 화면 표시·시드 식별용 (예: `MEAL`, `식대`) |
| 카테고리 | 기본급·수당·복리후생 등 |
| 비과세 여부 | 소득세 비과세 적용 대상 |
| 비과세 한도 | 비과세 적용 시 월 한도 (식대 20만원 등) |
| 연봉 산입 여부 | 연봉 환산 시 포함 여부 |
| 계산 방식 | 고정금액 / 기본급 % / 시간 비례 등 |
| 기본 금액 | 표준 지급액 (직원별 오버라이드 가능) |
| 정렬 순서 | 화면 표시 순서 |

### 2.3 공제 마스터 (PayDeductionType)

| 속성 | 설명 |
|------|------|
| 카테고리 | 4대보험 / 세금 / 회사 공제 / 기타 |
| 국가 코드 | 국가별로 적용 (한국 4대보험은 한국만, 미국 401k는 미국만 등) |
| 계산 방식 | 고정 / 비율 / 누진 |
| 기본 비율 | 표준 공제율 |

### 2.4 직원별 정기 급여 항목 (EmployeePayItem)

직원에게 정기적으로 지급·공제되는 수당·공제 항목을 직원별로 등록합니다.

- **시점 기반**: `시작일 / 종료일`로 적용 기간 관리 (보상 이력처럼 시점별 추적)
- **수당 예시**: 차량유지비 30만원 (2025-04~), 직책수당 50만원 (팀장 발령 후~)
- **공제 예시**: 노조비 1만원, 사내 동호회비 5천원

### 2.5 급여 밴드 (SalaryBand)

각 회사·직급·직군 조합에 대해 급여의 정상 범위를 정의합니다.

- **최소·중간·최대 (min/mid/max)**: 해당 직급의 적정 급여 구간
- **통화**: 회사별 (한국은 KRW, 미국은 USD 등)
- **시행일·종료일**: 시점별 밴드 (매년 또는 분기별 갱신)
- **활용**: 보상 변경 시 밴드 범위 검증, 법인전환 시 도착 법인 밴드 재조정, 이상 검토에서 밴드 초과 검출

### 2.6 보상 이력 (CompensationHistory)

모든 보상 변경을 시점별로 기록합니다.

| 항목 | 설명 |
|------|------|
| 변경 유형 | 정기 인상 / 승진 / 시장 조정 / 특별 인상 / 기타 |
| 이전 기본급·새 기본급 | 변경 전후 |
| 변경 비율 | 자동 계산 (%) |
| 시행일 | 적용 시작일 |
| 사유 | 자유 텍스트 |
| 결재자 | 누가 승인했는지 |
| 평가 등급 (스냅샷) | 변경 시점의 평가 등급 (보상 기획 연동) |
| 예외 처리 | 밴드 초과·특별 인상 시 예외 표시 + 사유 |
| 미래 시행 예약 | 미래 날짜 자동 반영 (Off-cycle) |

### 2.7 4대보험 요율 + 비과세 한도 + 세금 구간

연도별로 갱신되는 마스터데이터.

- **4대보험 요율 (한국)** — 국민연금 4.5% / 건강보험 3.545% / 장기요양 (건강보험 ×12.81%) / 고용보험 0.9% / 산재 (사업주만)
- **비과세 한도** — 식대 월 20만원, 차량유지비 월 20만원, 육아수당 등 코드별
- **소득세 구간 (TaxBracket)** — 한국 누진 구간 (8단계), 회사·국가·세금유형별 정의
- **갱신 주기**: 매년 1월 시드 업데이트 또는 시점별 시행일

## 3. 핵심 정책

### 3.1 한국 본사 직접 계산 vs 해외 외부 처리

| 법인 | 처리 방식 |
|------|----------|
| **국내 7개 법인** (CTR-HOLD, CTR, CTR-MOB, CTR-ECO, CTR-ROB, CTR-ENR, CTR-FML) | **시스템 직접 계산** — 본 매뉴얼의 8단계 워크플로 그대로 적용 |
| **해외 6개 법인** (CTR-CN, CTR-VN, CTR-US, CTR-MX, CTR-RU, CTR-EU) | **외부 처리 → 결과 업로드** — 현지에서 계산한 결과를 엑셀 등으로 받아 시스템에 업로드 |

**의도**: 각국 노동법·세무·사회보험은 복잡하고 변동이 잦아 시스템 내재화 비용이 큼. 한국 본사·국내 법인만 직접 계산하고, 해외는 외부 처리한 결과를 시스템에 통합하여 전사 가시성 확보.

> **데이터 흐름 주의**: 해외 결과 업로드는 현재 수동 엑셀 업로드 기반입니다. 외부 시스템과의 정식 자동 연동은 Phase 2 작업입니다 (§9 #1 참조).

### 3.2 8단계 상태 워크플로

월 급여 처리는 다음 8단계를 순서대로 거칩니다.

| # | 상태 | 의미 | 작업 |
|---|------|------|------|
| 1 | 초안 (DRAFT) | 새 급여 회차 생성 | 인사담당자가 회차 등록 (연·월 지정) |
| 2 | 근태 마감 | 출퇴근 데이터 마감 | 출퇴근 모듈에서 해당 월 근태 마감, 시간외근무 확정 |
| 3 | 계산 중 (CALCULATING) | 자동 계산 진행 | 시스템이 직원별 PayrollItem 자동 생성 (기본급·수당·시간외·4대보험·세금) |
| 4 | 수동 조정 (ADJUSTMENT) | 인사담당자 검토·조정 | 소급 지급·보너스·오류 정정·과지급 환수 등 직원별 조정 추가 |
| 5 | 이상 검토 (REVIEW) | 자동 이상 검출 + 해결 | 6종 규칙으로 이상 자동 검출 → 인사담당자가 확인·정정·화이트리스트 처리 |
| 6 | 결재 대기 (PENDING_APPROVAL) | 다단계 결재 진행 | 본부장 → 대표 결재 (결재 매뉴얼 §2.1) |
| 7 | 승인 완료 (APPROVED) | 모든 결재 완료 | 지급 준비 |
| 8 | 지급 완료 (PAID) | 급여 지급 + 명세서 발행 | 은행 송금, 직원별 급여명세서 PDF 생성·발행 |

**상태 전이 강제**: 각 단계는 이전 단계가 완료되어야 진행 가능. 역행 불가 (취소 외).

**취소(CANCELLED)**: 어느 단계에서든 인사담당자가 취소 가능. 취소 시 해당 회차의 PayrollItem·Adjustment·결재는 보존되나 비활성.

### 3.3 한국 4대보험·소득세 자동 계산

**4대보험 (근로자 부담분)**:
- 국민연금: 4.5% (월 보수 590만원 상한)
- 건강보험: 3.545%
- 장기요양보험: 건강보험료 × 12.81% (특수 계산)
- 고용보험: 0.9%
- 산재보험: 사업주 전액 부담 (직원 차감 없음)

**소득세**: 한국 누진 구간 (TaxBracket 모델, 시점별 갱신). 월 급여에서 간이세액표 적용.

**지방소득세**: 소득세 × 10%.

**통상시급**: 월 기본급 ÷ 209시간 (주 40시간 기준).

### 3.4 시간외근무 가산

| 유형 | 가산율 |
|------|--------|
| 평일 야간 (22:00~익일 06:00) | 1.5배 |
| 평일 연장근로 | 1.5배 |
| 휴일 근로 | 1.5배 |
| 휴일 8시간 초과 | 2배 |
| 야간 + 연장근로 동시 | 2배 |

근태 모듈에서 마감된 시간외근무 시간(분 단위)을 통상시급 × 가산율로 계산.

> **52시간 한도**: 한국 법인은 주 52시간 한도 초과 시 시스템이 자동으로 시간외근무 입력을 차단합니다 (출퇴근 매뉴얼 참조). 차단 임계값은 회사별 설정 가능.

### 3.5 비과세 적용

수당 마스터에서 "비과세 한도"가 설정된 항목은 한도 내에서 과세 대상에서 제외됩니다.

- **식대**: 월 20만원까지 비과세
- **차량유지비**: 월 20만원까지 비과세 (본인 차량 + 업무 사용 조건)
- **육아수당**: 월 10만원까지 비과세 (6세 이하 자녀)
- **연구활동비**: 월 20만원 (특정 직군 한정)
- **기타**: NontaxableLimit 마스터에서 코드별 한도 관리

한도를 초과한 부분은 과세 대상으로 소득세 계산에 포함.

### 3.6 일할 계산 (Pro-rated)

월 중 발생한 입사·퇴사·휴직 시 일할 계산.

- **신규 입사자**: 입사일 ~ 월말까지 평일 수 / 해당 월 총 평일 수 × 월급
- **퇴사자**: 월초 ~ 마지막 근무일까지 평일 수 / 해당 월 총 평일 수 × 월급
- **휴직 중**: 휴직 기간 동안 평일 수만큼 일할 차감 (휴직 유형별 급여 정책은 휴직 매뉴얼 §3.3 참조)
- **법인 전환**: 출발 법인에서 전환일 전날까지, 도착 법인은 전환일부터 (양쪽 일할 계산)

평일 기준 계산이 표준. 공휴일 처리는 회사별 정책.

### 3.7 수동 조정 5종

자동 계산 결과에 인사담당자가 추가하는 조정 항목.

| 유형 | 의미 | 예시 |
|------|------|------|
| 소급 지급 | 과거 시점의 미지급분을 이번 달에 추가 지급 | 연봉 인상이 3월 시행이었으나 4월 급여에 반영 → 3월 차액 소급 |
| 보너스 | 비정기 지급 | 경조사비, 명절 상여, 특별 성과급 |
| 정정 | 이전 회차 오류를 이번 달에 정정 | 3월 식대 누락 → 4월에 추가 지급 |
| 환수 | 과지급분 차감 | 2월 야간수당 이중 지급 → 4월에서 차감 |
| 기타 | 위 4종 외 | 자유 사유 입력 |

**필수 입력**: 사유 (감사 대상), 증빙 파일 (선택)
**자동 생성**: 휴직 모듈에서 휴직 활성화 시 월별 차감 조정 자동 생성 (휴직 매뉴얼 §11.2 `payroll-adjustment.ts`)

### 3.8 이상 검토 6종 규칙

자동 검출 규칙. 이상 검토 단계에서 인사담당자에게 표시됨.

| 규칙 | 임계값 | 심각도 |
|------|--------|--------|
| 전월 대비 ±30% 초과 변동 | 기본 30% / 최소 금액 5만원 | 경고 |
| 급여 밴드 초과 | 직원 직급의 max 초과 | 경고 |
| 법정 시간외근무 52시간 초과 | 52시간 | **위험** |
| 기본급 정상치 50% 미만 | 직급 밴드 min × 50% | **위험** |
| 신규 입사자 첫 급여 | 일할계산 검증 안내 | 정보 |
| 퇴사자 마지막 급여 | 일할계산 + 퇴직금 검증 안내 | 정보 |

**해결 방법**:
- 정상 확인 → "정상으로 확인" 처리
- 정정 → 수동 조정 추가 후 재계산
- 화이트리스트 → 사유와 함께 통과 (감사 대상)

모든 이상이 해결되어야 결재 단계로 진행 가능 (위험 항목은 화이트리스트 필수).

### 3.9 다단계 결재 (본부장 → 대표)

급여 결재는 위임전결규정상 2단계 결재가 표준입니다.

- **1단계**: 본부장 (부서장 또는 임원)
- **2단계**: 대표
- 결재 매뉴얼 §2.1 급여 흐름과 동일
- 결재 모델은 별도(`PayrollApproval` + `PayrollApprovalStep`) 사용 — 일반 결재 흐름의 `ApprovalFlow`와 분리 (이력 추적 정밀도 위해)

### 3.10 급여명세서 발급

승인 완료된 회차에서 직원별 PDF 명세서를 자동 생성합니다.

- **포함 항목**: 기본급·수당 항목별 / 시간외근무 / 상여 / 4대보험 항목별 / 소득세 / 지방세 / 회사 공제 / 실수령액
- **발행 경로**: 직원 본인 화면(`/payroll/me`)에서 열람 + 선택적 이메일 발송
- **열람 추적**: 직원이 명세서를 본 시점 기록 (감사 + 미열람 알림)

### 3.11 통화 변환

해외 법인 급여를 본사 KPI에 통화 변환하여 표시.

- **환율 매스터(ExchangeRate)**: 연·월·통화쌍별 환율 등록
- **갱신 주기**: 매월 1일 업로드 (전월 평균 환율 기준)
- **표시 통화**: 본사 KPI는 KRW 환산, 각 법인 화면은 현지 통화
- **법인별 통화**: KRW(국내 7) / CNY(중국) / VND(베트남) / USD(미국) / MXN(멕시코) / RUB(러시아) / PLN(폴란드)

### 3.12 연말정산 (한국)

매년 1~2월 시행되는 한국 근로소득 연말정산.

**11단계 자동 계산**:
1. 연간 총급여 집계 (해당 연도 모든 승인·지급된 회차)
2. 근로소득공제 (총급여 구간별 공제율)
3. 근로소득금액 = 총급여 - 근로소득공제
4. 종합소득공제 (인적공제·연금보험·특별공제 등)
5. 과세표준 = 근로소득금액 - 종합소득공제
6. 세율 적용 (누진세율)
7. 산출세액
8. 세액공제 합계 (근로소득세액공제·자녀세액공제·연금계좌 등)
9. 결정세액
10. 기납부세액 집계
11. 환급/추가납부 = 결정세액 - 기납부세액

**연말정산 영수증 PDF** 자동 생성 (yearEndReceiptPdf).

### 3.13 퇴직금 정산

**산정 공식** (한국 근로기준법):
```
퇴직금 = 최근 3개월 평균임금 × 재직일수 / 365 × 30일
```

- **평균임금**: 퇴직 전 3개월 동안 받은 모든 임금(기본급 + 수당 + 시간외근무 + 상여)의 1일 평균
- **재직 기간**: 1년 이상 근무 시에만 발생 (1년 미만은 미지급)
- **중간정산 (SeveranceInterimPayment)**: 주택 구입·의료비 등 법정 사유 시 중간정산 신청 가능
- **법인전환 시**: 근속 통산(인사관리 매뉴얼 §3.4)이므로 최초 그룹 입사일 기준

## 4. 워크플로

### 4.1 월 급여 라이프사이클

```
[1일: 회차 생성]
     │
     ↓
[근태 마감] ── 출퇴근 모듈에서 해당 월 시간외근무 확정
     │
     ↓
[자동 계산] ── 직원별 PayrollItem 생성 (기본급 + 수당 + 시간외 - 4대보험 - 소득세)
     │         · 휴직 중 직원: 일할 차감 자동 적용
     │         · 신규 입사·퇴사자: 일할 계산
     │         · 시간외근무: 가산율 적용
     │
     ↓
[수동 조정] ── 인사담당자가 직원별 조정 추가 (소급·보너스·정정·환수·기타)
     │         · 휴직 차감은 시스템 자동 추가
     │         · 사유·증빙 필수
     │
     ↓
[이상 검토] ── 6종 규칙 자동 검출 → 인사담당자 해결 (확인/정정/화이트리스트)
     │
     ↓
[결재 대기] ── 본부장 → 대표 2단계 결재
     │         (결재 매뉴얼 §2.1)
     │
     ↓
[승인 완료] ── 모든 결재 통과
     │
     ↓
[지급 + 명세서 발행]
     │         · 은행 송금 데이터 생성 (BankTransfer)
     │         · 직원별 급여명세서 PDF 자동 생성
     │         · 직원 본인 화면에 발행, 이메일 발송 옵션
     ↓
[완료]
```

### 4.2 해외 법인 외부 처리 워크플로

```
[외부에서 급여 계산 (현지 처리)]
     │
     ↓
[엑셀 파일 수령] ── 회사별 양식 파일 수신
     │
     ↓
[업로드 매핑 선택] ── 회사별 PayrollImportMapping (컬럼 매핑 정의)
     │
     ↓
[파일 업로드 + 파싱]
     │         · 직원 매칭 (employee_no)
     │         · PayrollItem 생성
     │         · 업로드 로그(PayrollImportLog) 기록
     │
     ↓
[검증 → 확인] ── 인사담당자가 결과 확인 후 confirmedAt
     │
     ↓
[승인 + 지급]
```

### 4.3 보상 변경 (Off-cycle) 워크플로

월 급여와 별개로 직원의 기본급을 변경하는 경우.

```
[변경 요청] ── 인사담당자가 OffCycleCompRequest 등록
     │
     ↓
[밴드 검증] ── 도착 직급 밴드 내인지 확인 (초과 시 예외 처리 필요)
     │
     ↓
[결재] ── 직속 팀장 → 인사담당자 (결재 매뉴얼 §2.1 비정기 급여 조정)
     │
     ↓
[CompensationHistory 기록] ── 시행일 기준
     │
     ↓
[다음 급여 회차에서 자동 반영] ── 또는 미래 시행일 예약 (isScheduled)
```

### 4.4 자동 알림

- **근태 마감 임박**: 매월 20일경 근태 마감 안내 — **자동 알림 cron 미등록** (`vercel.json` crons 또는 nudge 룰에 등록되어 있지 않음). 현재는 인사담당자가 캘린더로 직접 관리. **추후 개선** — nudge 룰 또는 별도 cron 등록 검토 (§9 #14)
- **이상 검출 발생**: 위험 등급 이상 즉시 인사담당자에게 알림
- **결재 요청**: 결재자에게 알림 (Teams + 이메일 + 푸시, 결재 매뉴얼 §4.3)
- **급여 지급 완료**: 직원에게 명세서 발행 알림
- **미열람 명세서**: 명세서 발행 7일 후 미열람 직원에게 안내

- **전사관리자**: 13개 법인 전체

## 6. 화면 안내

### 6.1 직원 — 내 급여 화면

위치: `급여 → 내 급여` 또는 `프로필 → 급여`

- **현재 보상**: 기본급 + 정기 수당
- **명세서 목록**: 월별 명세서 다운로드 (PDF)
- **명세서 상세**: 기본급·수당·공제 항목별 상세
- **연봉 정보**: 연간 총 급여 추이 (그래프)
- **연말정산 영수증** (1~2월): 결과 조회

### 6.2 인사담당자 — 급여 대시보드

위치: `급여 → 대시보드`

- **이번 달 KPI**: 총 인원·총 지급액·평균 급여·시간외 합계·전월 대비 변동
- **회차 진행 현황**: 회차별 8단계 상태 시각화
- **이상 검토 알림**: 미해결 이상 건수
- **연도별·법인별 추이** (전사관리자만): 13개 법인 통화 환산 KPI

### 6.3 급여 회차 상세

위치: `급여 → 회차 [선택]`

탭 구조:
1. **개요** — 회차 정보·상태·KPI
2. **직원별 명세** — PayrollItem 목록 (검색·필터·CSV 다운로드)
3. **수동 조정** — 조정 항목 추가·수정
4. **이상 검토** — 6종 규칙별 검출 항목 (해결 작업)
5. **결재** — 결재 진행 상태 (결재 매뉴얼 통합)
6. **명세서 발행** — 직원별 발행 상태·열람 추적
7. **은행 송금** — 송금 데이터 다운로드

### 6.4 수당·공제 마스터 (인사담당자)

위치: `급여 → 설정 → 수당` / `공제`

- 회사 선택 → 수당·공제 항목 목록
- 신규 등록·수정·비활성
- 비과세 한도·계산 방식·기본 금액 설정

### 6.5 급여 밴드 (인사담당자)

위치: `보상 → 급여 밴드`

- 회사·직급·직군별 밴드 등록
- 시점별 갱신 (`시행일`)
- 시각화: 직급별 min/mid/max 차트

### 6.6 시뮬레이션

위치: `급여 → 시뮬레이션`

- 가상 직원·연봉을 입력하여 월급 시뮬레이션
- 4대보험·소득세·실수령액 즉시 계산
- 채용 면접 시 연봉 협상에 활용

### 6.7 해외 급여 업로드

위치: `급여 → 해외 업로드`

- 회사 선택 → 업로드 매핑 선택 → 엑셀 업로드
- 컬럼 매핑 결과 미리보기
- 업로드 이력 조회

### 6.8 연말정산

위치: `급여 → 연말정산`

- 직원별 연말정산 계산 (11단계 자동)
- 결과 검토 → 결정세액 확정
- 영수증 PDF 일괄 발행

### 6.9 퇴직금 정산

위치: `급여 → 퇴직 정산` 또는 퇴사자 발령 시 자동

- 마지막 3개월 평균임금 자동 계산
- 재직일수 × 30 / 365 적용
- 결과 PDF 생성

## 7. 특수 상황 처리

### 7.1 휴직 중 직원의 급여

- 휴직 활성화 시 시스템이 휴직 기간을 월별로 자동 분리하여 각 월 급여에 차감 조정 자동 생성
- 무급 휴직: 휴직 일수만큼 일할 차감
- 일부 유급 (예: 중국 병가 60%): 비율(%) 적용 일할 차감
- 보험 지급 (예: 한국 육아휴직): 사업주 부담분은 일할 차감, 정부 지급분은 시스템 외부 정산
- 휴직 유형별 정책은 휴직 매뉴얼 §3.3 참조

### 7.2 신규 입사자 첫 급여

- 입사일 ~ 월말까지 평일 수 / 해당 월 총 평일 수 × 월급으로 일할 계산
- 이상 검토에서 "신규 입사자 첫 급여" 항목으로 자동 표시 (정보 등급)
- 인사담당자가 일할 계산 결과 확인 후 통과

### 7.3 퇴사자 마지막 급여

- 월초 ~ 마지막 근무일까지 평일 수로 일할 계산
- 미사용 연차 수당 자동 추가 (휴가 매뉴얼 §7.2 참조)
- 마이너스 연차 정산 자동 차감 (휴가 매뉴얼 §3.6 참조)
- 퇴직금 별도 계산 (1년 이상 근무 시)
- 4대보험 상실 신고는 시스템 외부 (외부 시스템 또는 노무사)

### 7.4 법인 전환 직원의 급여

- 인사관리 매뉴얼 §3.5 정책에 따라 처리
- 출발 법인: 전환일 전날까지 일할 계산하여 마지막 급여 지급
- 도착 법인: 전환일부터 일할 계산, 도착 법인 급여 밴드 기준으로 재조정된 급여 적용
- 근속은 통산 유지 (최초 그룹 입사일)

### 7.5 소급 지급

- 연봉 인상이 과거 시점 시행이었으나 그 월 급여에 반영 안 된 경우
- 이번 달 회차에서 수동 조정으로 차액 소급 지급 (Adjustment Type: 소급)
- 사유에 "시행일 ~ 적용일 차액" 명시
- 이상 검토에서 전월 대비 변동 ±30% 초과 시 자동 검출 → 화이트리스트 필요

### 7.6 보너스·경조사비

- 연 1~2회 명절 상여, 분기·연간 성과급, 경조사비 등
- 회차 내 수동 조정으로 추가 (Adjustment Type: 보너스)
- 별도 회차(RunType: BONUS)로 분리 처리 가능 (월 급여와 별도 결재)

### 7.7 과지급 환수

- 이전 회차 오류로 과지급한 경우
- 이번 달 회차에서 수동 조정으로 차감 (Adjustment Type: 환수, 음수 금액)
- 사유에 원인·시점 명시
- 직원이 일시 차감을 부담스러워하는 경우 분할 차감 (여러 달 나눠 처리)

### 7.8 급여 밴드 초과

- 보상 변경 시 도착 직급의 max 초과
- 시스템이 자동 차단하지 않고 "예외 처리" 표시 + 사유 입력 요구
- 시장가치·특별 기여 등 정당한 사유로 등록 가능
- 다음 급여 회차의 이상 검토에서 "급여 밴드 초과" 경고로 표시

### 7.9 시간외근무 52시간 초과

- 한국 법인: 주 52시간 한도 초과 시 출퇴근 모듈에서 자동 차단 (회사별 설정 가능)
- 차단을 우회한 경우(예: 특별인가 연장근로) 이상 검토에서 위험 등급으로 표시
- 화이트리스트 처리 필수 + 사유·근거(특별인가증 등) 첨부

### 7.10 해외 법인 외부 처리 결과 불일치

- 외부에서 받아 업로드한 결과와 시스템 검증치 사이에 차이가 발생한 경우
- 현재 정책: 외부 결과를 진실로 간주 (현지 노동법·세무에 정통한 외부 처리 결과 우선)
- 시스템은 KPI 통합·이력 보관 용도로 사용
- **추후 개선**: 외부 시스템과의 정식 자동 연동 후 이상 검출 강화 (§9 #1)

### 7.11 연말정산 결과 추가납부

- 결정세액이 기납부세액보다 큰 경우 (추가 납부)
- 다음 1~2월 급여에서 분할 차감 (보통 2회 또는 회사 정책)
- 환급의 경우: 다음 회차 급여에서 일괄 환급

### 7.12 퇴직금 중간정산

- 주택 구입·의료비·천재지변 등 법정 사유로 직원이 신청
- 인사담당자 검토 → 대표 결재 → 지급
- 중간정산 후 재직 기간 리셋 (이후 퇴직 시점부터 다시 계산)
- 시스템에 SeveranceInterimPayment 레코드 기록

## 8. 자주 묻는 질문

**Q. 내 급여명세서는 어디서 보나요?**
A. `급여 → 내 급여` 또는 `프로필 → 급여`에서 월별 명세서를 조회·PDF 다운로드할 수 있습니다. 이메일로도 발송됩니다.

**Q. 명세서에 적힌 4대보험이 회사에서 차감한 금액과 다른데요?**
A. 명세서에는 직원 부담분만 표시됩니다 (4대보험 직원·사업주 분담). 사업주 부담분(산재 전액 + 4대보험 사업주 분)은 회사가 별도 납부하며 직원 명세서에 안 나옵니다.

**Q. 비과세 한도를 초과한 식대는 어떻게 처리되나요?**
A. 한도(월 20만원)까지는 비과세, 초과분은 과세 대상으로 소득세 계산에 포함됩니다. 명세서에는 비과세분과 과세분이 분리 표시됩니다.

**Q. 시간외근무 52시간을 넘기면 어떻게 되나요?**
A. 한국 법인은 출퇴근 모듈에서 자동 차단됩니다. 특별인가 등으로 우회한 경우 급여 이상 검토에서 위험 등급으로 검출되고 화이트리스트 처리(사유·근거 첨부)가 필요합니다.

**Q. 휴직 중인데 급여가 계속 들어와요. 왜죠?**
A. 휴직 유형에 따라 다릅니다. 한국 출산휴가는 **현재 시스템상 일률 60일 사업주 + 30일 고용보험으로 계산**됩니다 (사업주 분은 회사 급여로 들어옴). 한국 고용보험법상 우선지원대상기업은 90일 전액 고용보험 지급이 적용되어야 하나, **시스템은 회사별 우선지원기업 분기 처리를 하지 않음** (시스템 fix 트랙 — §9 참조). 무급 휴직(가족돌봄·개인사유 등)은 휴직 기간 일할 차감됩니다. 휴직 매뉴얼 §3.3 표를 확인하세요.

**Q. 연봉이 인상되었는데 이번 달 급여에 반영이 안 됐어요.**
A. 보상 변경 시행일과 급여 회차 마감일이 맞지 않은 경우 다음 회차에서 차액을 소급 지급(Adjustment Type: 소급)으로 처리합니다. 인사담당자에게 문의하세요.

**Q. 해외 법인 직원도 시스템에서 급여를 직접 계산하나요?**
A. 아닙니다. 국내 7개 법인만 시스템 직접 계산이며, 해외 6개 법인은 외부에서 처리한 결과를 시스템에 업로드합니다. 직원 본인 화면에서는 동일하게 명세서 조회가 가능합니다.

**Q. 연말정산 결과를 어디서 확인하나요?**
A. 매년 1~2월 `급여 → 연말정산`에서 결정세액·환급/추가납부액·영수증 PDF를 조회할 수 있습니다. 추가납부의 경우 다음 1~2월 급여에서 분할 차감됩니다.

**Q. 퇴직금은 언제 지급되나요?**
A. 퇴사 후 14일 이내 지급이 한국 법정 요건입니다. 인사관리 모듈에서 퇴직 발령 등록 시 시스템이 자동 계산(최근 3개월 평균임금 × 재직일수/365 × 30)하며 인사담당자가 검토 후 지급 처리합니다. 1년 미만 근무 시 미지급.

**Q. 본부장이 시스템에 없는데 급여 결재는 어떻게 되나요?**
A. 현재 시스템 역할에는 "본부장"이 명시적으로 없어 해당 부서의 부서장(부서장 또는 임원 역할 보유자)이 1단계 결재자로 동작합니다. 본부장 매핑은 결재 매뉴얼 §9 #5 / 인사관리 매뉴얼 §9 #5에서 추후 개선 항목입니다.

**Q. 급여 회차를 잘못 만들었는데 삭제할 수 있나요?**
A. 결재 진행 전 단계(DRAFT/근태마감/계산/조정/검토)에서는 취소 가능합니다. 결재·승인·지급 완료 후에는 삭제 불가, 별도 정정 회차로 처리해야 합니다.

**Q. 환율은 어떻게 결정되나요?**
A. 매월 1일 인사담당자(전사관리자)가 전월 평균 환율로 ExchangeRate 마스터를 업데이트합니다. 본사 KPI 환산 시 이 환율 사용. 직원 명세서는 현지 통화 그대로 표시.

| 15 | 한국 출산휴가 우선지원기업 분기 처리 미구현 | 시스템은 한국 출산휴가를 일률 60일 사업주 + 30일 고용보험으로 계산. 한국 고용보험법상 우선지원대상기업은 90일 전액 고용보험 지급이 적용되어야 하나 회사별 분기 코드 없음. CTR 13개 법인 중 일부 자회사는 우선지원기업에 해당할 수 있음. **추후 개선** — 회사별 우선지원기업 여부 입력 + 계산 분기 (시스템 fix 트랙) |

### 11.1 데이터 모델

핵심 모델 (모두 `prisma/schema.prisma`):

- `PayrollRun` — 월 급여 회차 (8-state status: DRAFT/ATTENDANCE_CLOSED/CALCULATING/ADJUSTMENT/REVIEW/PENDING_APPROVAL/APPROVED/PAID/CANCELLED)
- `PayrollItem` — 직원별 급여 명세 (기본급/시간외/상여/수당/공제/실수령액)
- `PayrollAdjustment` — 수동 조정 (AdjustmentType: RETROACTIVE/BONUS/CORRECTION/DEDUCTION/OTHER, LOA 연동)
- `PayrollAnomaly` — 이상 검출 (AnomalySeverity: CRITICAL/WARNING/INFO, AnomalyStatus: OPEN/RESOLVED/WHITELISTED)
- `PayrollApproval` + `PayrollApprovalStep` — 다단계 결재 (별도 모델, 결재 모듈의 ApprovalFlow와 분리)
- `Payslip` — 급여명세서 발행 추적
- `PayAllowanceType` / `PayDeductionType` — 수당·공제 마스터 (회사별)
- `EmployeePayItem` — 직원별 정기 수당·공제 (시점 기반)
- `SalaryBand` — 회사·직급·직군별 급여 밴드 (시점 기반)
- `CompensationHistory` — 보상 변경 이력 (CompensationChangeType)
- `CompensationLetter` — 보상 변경 통지서 PDF
- `OffCycleCompRequest` + `OffCycleApprovalStep` — 비정기 보상 변경 결재

- `InsuranceRate` — 4대보험 요율 (연도별)
- `NontaxableLimit` — 비과세 한도 (연도·코드별)
- `TaxBracket` — 세금 구간 (회사·국가·세금유형·시점별)

- `PayrollSimulation` — 시뮬레이션

- `SeveranceInterimPayment` — 퇴직금 중간정산
- `YearEndDeductionConfig` — 연말정산 공제 설정
- `BankTransferBatch` + `BankTransferItem` — 은행 송금 배치
- `CompensationSetting` — 회사별 보상 설정 (급여 항목·인상률 매트릭스·성과급 규칙 JSON)

**핵심 enum**:
- `PayrollStatus`: 8-state (DRAFT → PAID + CANCELLED)
- `PayrollRunType`: MONTHLY / BONUS / SEVERANCE / SPECIAL

- `AnomalyStatus`: OPEN / RESOLVED / WHITELISTED

### 11.5 알려진 위험 코드 경로

- `src/lib/payroll/kr-tax.ts` — **PROTECTED**. 한국 4대보험·소득세 핵심 로직. 매년 1월 요율 갱신 필수 (DB 시드 + 코드 fallback 동기화). 수정 전 architecture review 필요
- `PayrollRun.status` 전이 — 8-state 순서 강제. 역행·점프 시 데이터 무결성 깨짐. 트랜잭션 내 검증 필수
- `PayrollAdjustment.loaId` — 휴직 모듈에서 자동 생성하는 조정. 휴직 취소 시 연결된 조정도 함께 처리 필요 (현재는 별도 처리)
- 일할 계산 (`getWeekdaysInMonth`, `getWeekdaysBetween`) — 평일/공휴일 정의가 회사·국가별로 달라야 하나 현재는 일관 적용. 공휴일 데이터 부재 법인은 부정확 가능 (휴가 매뉴얼 §9 #2와 동일 이슈)
- 환율 변환 — 본사 KPI 환산 시 ExchangeRate 누락 월이 있으면 변환 실패. graceful degradation 미구현
- `kpmg-interface.ts` `parseKpmgExcel` — Phase 2 미구현, throw 처리 (코드 호출 시 에러 발생, 정상 동작)
- 다단계 결재의 사이드이펙트 — 결재 진행 중 직원 발령 변경(전보·휴직 등) 발생 시 명세 재계산이 필요할 수 있으나 현재 자동 재계산 안 함
- 통화별 소수 자리 처리 — Decimal(15, 2)와 Decimal(15, 0)이 혼재. JPY/KRW 같은 무소수 통화 vs USD/EUR 소수 통화 일관성 확인 필요
