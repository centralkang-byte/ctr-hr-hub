# 출퇴근 매뉴얼 (Attendance)

> **최종 갱신**: 2026-05-12
> **담당**: 인사관리팀
> **상태**: Draft v1 (CEO 리뷰 전)

## 1. 개요

### 1.1 무엇을 다루는가

CTR HR Hub의 출퇴근 모듈은 다음을 통합 관리합니다.

- **출퇴근 기록** — 직원의 출근·퇴근 시각 기록 (단말기·웹·앱 다중 채널)
- **근무 유형** — 고정·유연·교대·재택 4종 (회사별·직원별 적용)
- **일일·주간·월간 근태** — 일별 근무 시간·시간외근무·야간근무·휴식 등 자동 집계
- **52시간 한도 관리** — 한국 법인 주간 근무 시간 자동 감시 + 경고 + 차단
- **공휴일 관리** — 회사별 공휴일 등록 + 영업일 계산에 반영 (휴가·급여 일할 계산 SSOT)
- **교대 근무** — 2조·3조·연속근무·4-on-2-off 등 교대 패턴 정의 + 조별 일정 관리
- **교대 변경 신청** — 직원이 교대 일정 교환·변경 신청
- **근태 결재** — 출퇴근 정정·연장근무 사전 승인·교대 변경 (결재 매뉴얼 §2.1 근태)
- **휴가·휴직 통합** — 휴가·휴직 기간은 자동으로 근태에 반영 (별도 입력 불필요)
- **단말기 연동** — 지문·카드·얼굴인식 단말기와 API 연동

### 1.2 도입 의도

- **법규 준수 자동화** — 한국 근로기준법 주 52시간 / 야간·휴일 가산 / 휴게 시간 등 자동 적용
- **급여 정확성** — 시간외근무·야간근무 시간이 급여 계산의 입력값 (급여 매뉴얼 §3.4 시간외 가산율 참조)
- **수동 작업 제거** — 단말기·웹·앱으로 자동 기록되어 엑셀 출퇴근표 작성 부담 제거
- **다국적 법인 대응** — 13개 법인이 각국 노동법(주간 한도·야간 가산·휴식)에 맞춰 차등 운영
- **교대 근무 시각화** — 공장·제조 부문의 2조·3조 교대 일정을 캘린더로 직관 관리

### 1.3 휴가·급여·결재 매뉴얼과의 관계

| 항목 | 본 매뉴얼 (출퇴근) | 관련 매뉴얼 |
|------|------------------|-----------|
| 일별 근무 시간 SSOT | 본 매뉴얼 | 급여 매뉴얼이 참조 (월 마감 후) |
| 시간외근무 시간 | 본 매뉴얼이 계산 (가산율은 적용 안 함) | 급여 매뉴얼 §3.4가 가산율 적용 |
| 공휴일 마스터 | 본 매뉴얼이 관리 | 휴가 매뉴얼 §3.2 영업일 계산이 참조 |
| 휴가 사용일 | 휴가 모듈 입력 → 본 매뉴얼이 자동 반영 | 휴가 매뉴얼 §4 |
| 휴직 기간 | 휴직 모듈 입력 → 본 매뉴얼이 자동 반영 | 휴직 매뉴얼 §4.2 |
| 출퇴근 정정 결재 | 본 매뉴얼이 신청 | 결재 매뉴얼 §2.1 근태 |

## 2. 출퇴근 구성 요소

### 2.1 근무 유형 4종

회사·직원별로 다른 근무 유형이 적용됩니다.

| 유형 | 의미 | 적용 예시 |
|------|------|----------|
| 고정 (FIXED) | 정시 출퇴근 (예: 09:00~18:00) | 본사 사무직 표준 |
| 유연 (FLEXIBLE) | 코어타임 내 자유 출퇴근 | 본사 일부 부서 (개발·기획 등) |
| 교대 (SHIFT) | 교대 패턴 기반 (2조·3조 등) | 공장·생산 부문 |
| 재택 (REMOTE) | 원격 근무 | 본사 일부·해외 일부 |

각 유형마다 출퇴근 기록 방식·근무 시간 계산 방식이 다릅니다.

### 2.2 출퇴근 기록 방식

| 방식 | 의미 | 적용 |
|------|------|------|
| 단말기 (지문·카드·얼굴인식) | 출입 단말기 자동 기록 | 공장·생산 시설 표준 |
| 웹 (`/attendance`) | PC 브라우저 출근/퇴근 버튼 | 사무직 표준 |
| 모바일 앱 | 스마트폰 앱 + 위치 인증 | 재택·외근 직원 |
| 수동 입력 | 인사담당자가 직원 대신 입력 | 단말기 장애·증빙 정정 등 예외 |

### 2.3 근무 일정 (WorkSchedule + EmployeeSchedule)

- **WorkSchedule (회사별 표준 일정)**: 회사 기본 근무시간 (주 5일 / 주 40시간 / 일 8시간 등), 점심시간, 휴게시간
- **EmployeeSchedule (직원별 일정)**: 직원이 어떤 WorkSchedule을 따르는지 시점별 매핑 (정규직 → 시간제 전환 등)

### 2.4 교대 근무 구성

| 항목 | 의미 |
|------|------|
| 교대 패턴 (ShiftPattern) | 회사별 교대 패턴 (2조 12시간, 3조 8시간, 주야교대, 4-on-2-off, 사용자 정의 등) |
| 슬롯 (Slot) | 패턴 내 시간대 (예: A슬롯 06:00-14:00, B슬롯 14:00-22:00, C슬롯 22:00-06:00) |
| 야간 가산 | 슬롯별 야간 가산 비율 (22:00~06:00 근무) |
| 교대 조 (ShiftGroup) | A조·B조·C조 (UI 색상 구분) |
| 조 멤버 | 조에 속한 직원 (시점 기반 가입·탈퇴) |
| 교대 일정 (ShiftSchedule) | 직원·날짜별 어떤 슬롯 근무 (예정 / 실제 / 결근 / 교환) |

### 2.5 공휴일 (Holiday)

- 회사별 연도별 공휴일 등록
- 대체공휴일 표시 (`isSubstitute=true`)
- 영업일 계산에서 제외 (휴가 매뉴얼 §3.2)
- 시간외 가산율의 휴일 근로 판정 기준 (급여 매뉴얼 §3.4)

> **데이터 흐름 주의**: 현재 공휴일 데이터가 한국(CTR-HOLD/CTR/CTR-MOB/CTR-ECO/CTR-ROB/CTR-ENR/CTR-FML)·중국(CTR-CN) 외 법인은 등록되어 있지 않습니다. CTR-VN, CTR-US, CTR-MX, CTR-RU, CTR-EU는 공휴일 미입력으로 영업일 계산이 부정확할 수 있습니다 (§9 #1 참조).

### 2.6 근태 설정 (회사별)

각 회사는 다음 근태 정책을 설정합니다.

| 항목 | 기본값 | 의미 |
|------|--------|------|
| 일 표준 근무시간 | 8시간 | 정규 근무시간 |
| 주 표준 근무일 | 5일 | 주 5일 근무 표준 |
| 주 최대 근무시간 | 52시간 | 한국 법정 한도 |
| 교대 활성화 | 사용 안 함 | 교대 근무 운영 여부 |
| 유연근무 설정 | 비활성 | 코어타임 / 일일 최소 근무시간 |
| 52시간 경고 임계값 | 주의 44h / 경고 48h / 차단 52h | 주별 경고·차단 임계값 |
| 차단 기능 | **한국만 ON, 나머지 법인 OFF** | 임계값 초과 시 시스템 차단 여부 |
| 타임존 | Asia/Seoul | 회사 표준 타임존 |

## 3. 핵심 정책

### 3.1 일별 근무 시간 계산

직원의 일별 근무 시간은 다음 공식으로 자동 계산됩니다.

```
총 근무 시간 = (퇴근 시각 - 출근 시각) - 휴식 시간
```

- **휴식 시간**: 표준 4시간 이상 근무 시 30분, 8시간 이상 근무 시 1시간 휴식 (한국 근로기준법). 회사별 조정 가능
- **출퇴근 누락**: 출근 또는 퇴근 기록이 없으면 결근 또는 정정 신청 필요
- **자정 통과 근무**: 교대 야간 근무 시 자정을 넘는 경우 다음날 출근 기록과 합산

### 3.2 시간외근무 시간 계산

```
시간외 = 총 근무 시간 - 표준 근무 시간 (일 8시간)
```

추가 분류:
- **평일 야간**: 22:00~익일 06:00 사이 근무 시간 (별도 집계)
- **휴일 근로**: 주말·공휴일에 근무한 시간
- **교대 야간**: 교대 패턴의 야간 슬롯에서 근무한 시간

**중요**: 본 매뉴얼은 **시간(분)만 집계**하며 가산율은 적용하지 않습니다. 가산율 적용은 급여 모듈이 담당 (급여 매뉴얼 §3.4).

### 3.3 근무 유형별 정책 차이

| 유형 | 출퇴근 방식 | 시간 계산 |
|------|-----------|----------|
| 고정 | 정시 출퇴근 기록 | 표준 시간 외는 시간외근무로 계산 |
| 유연 | 코어타임 내 자유 | 일일 최소 시간(예: 4시간) 충족 검증 |
| 교대 | 교대 슬롯 기준 | 예정 슬롯 시간 기준 계산, 슬롯 외 시간은 시간외 |
| 재택 | 앱·웹으로 시작·종료 기록 | 고정과 동일 (출퇴근 위치만 다름) |

### 3.4 52시간 한도 관리 (한국 법인)

한국 근로기준법은 주 52시간(기본 40 + 연장 12)을 최대로 규정합니다.

**3단계 임계값** (회사별 조정 가능):
- **주의 (44시간)**: 직원·팀장에게 정보성 알림
- **경고 (48시간)**: 직원·팀장·인사담당자에게 위험 알림
- **차단 (52시간)**: 추가 시간외근무 신청·기록 자동 차단 (한국 법인만)

**임박 감지**: 직원 퇴근(clock-out) 등록 시점에 직원별 주간 누적 시간 즉시 계산 (`checkWorkHourAlert` 동기 호출) → 임계값 도달 시 WorkHourAlert 생성.

**차단 해제**: 특별인가 연장근로 등 정당한 사유 시 인사담당자가 수동으로 차단 해제 + 사유 입력 (감사 대상).

> **데이터 흐름 주의**: 차단 기능은 기본값으로 **한국 법인만 활성화**되어 있습니다. 해외 법인은 알림만 발송하고 차단하지 않습니다. 각국 노동법에 맞춰 회사별로 활성화 정책 결정 가능.

### 3.5 지각·조퇴·결근 처리

직원의 출퇴근 상태는 자동 판정됩니다.

| 상태 | 판정 기준 |
|------|----------|
| 정상 (NORMAL) | 표준 출근 시각 이내 출근 + 표준 퇴근 시각 이후 퇴근 |
| 지각 (LATE) | 표준 출근 시각 초과 출근 |
| 조퇴 (EARLY_OUT) | 표준 퇴근 시각 이전 퇴근 |
| 결근 (ABSENT) | 출근·퇴근 기록 모두 없음 (휴가·휴직 제외) |

**유연근무 직원**: 코어타임 내 출근하면 정상으로 판정.
**교대 직원**: 예정 슬롯 시작 시각 기준으로 판정.

### 3.6 휴가·휴직 자동 반영

- **휴가**: 휴가 신청이 승인되면 시스템이 휴가 기간을 출퇴근 결근으로 표시하지 않고 "휴가" 상태로 표시
- **휴직**: 휴직 활성화 시 휴직 기간 동안 출퇴근 입력 불가, 자동으로 "휴직 중" 상태로 표시 (휴직 매뉴얼 §4.2)
- **공휴일**: 시스템에 등록된 공휴일에는 출퇴근 입력 없어도 결근 처리되지 않음 — 단, 해외 5법인(CTR-VN, CTR-US, CTR-MX, CTR-RU, CTR-EU)은 공휴일 미등록 상태이므로 자국 공휴일이 결근으로 자동 처리될 수 있음 (§9 #1 참조)
- **지정연차**: 회사 지정연차일에는 자동으로 휴가 처리 (휴가 매뉴얼 §3.4)

### 3.7 교대 근무 정책

**교대 패턴 5종**:
- 2조 (TWO_SHIFT): 12시간 교대 (주간·야간)
- 3조 (THREE_SHIFT): 8시간 교대 (오전·오후·야간)
- 연속 야간 (DAY_NIGHT_OFF): 주간 → 야간 → 휴무 반복
- 4-on-2-off: 4일 근무 → 2일 휴무
- 사용자 정의 (CUSTOM): 회사별 자유 패턴 정의

**교대 일정 관리**:
- 인사담당자가 월·주 단위로 조별 일정 배정
- 직원은 본인 일정 캘린더에서 확인
- 교대 변경 신청은 결재 매뉴얼 §2.1 근태 흐름 따름

### 3.8 야간 근무 정의

야간 근무는 22:00~익일 06:00 사이에 근무한 시간입니다.

- **고정 직원**: 시간외근무가 야간에 발생한 경우
- **교대 직원**: 야간 슬롯(`isNightShift=true`) 근무 시간
- **자정 통과**: 출근 22:00, 퇴근 06:00 → 8시간 전부 야간으로 분류

야간 근무 가산은 급여 매뉴얼 §3.4에서 적용 (1.5배 또는 시간외 + 야간 중첩 시 2배).

### 3.9 출퇴근 정정·연장근무 결재

직원이 출퇴근 시각을 잘못 기록했거나 사전에 연장근무가 필요한 경우 결재 흐름을 거칩니다.

| 신청 유형 | 처리 |
|----------|------|
| 출퇴근 정정 | 직원이 새 시각 + 사유 입력 → 직속 팀장 결재 → 승인 시 기록 정정 |
| 사전 연장근무 신청 | 예정 연장근무 시간 입력 → 직속 팀장 결재 → 승인 시 차단 면제 |
| 교대 변경 신청 | 변경 희망 일자 + 사유 → 직속 팀장 결재 |

결재 흐름 상세는 결재 매뉴얼 §5.2 근태 모듈 참조.

## 4. 워크플로

### 4.1 일일 출퇴근 라이프사이클

```
[출근]
   │     · 단말기/웹/앱으로 출근 시각 기록
   │     · 출근 후 휴가 신청·정정 등 신청 가능
   ↓
[근무 중]
   │     · 시간외근무 시 차단 임계값 도달 시 알림 또는 차단
   │     · 외출·외근은 별도 신청 (시스템에서 지원 안 함, 현재 정정 절차로 처리)
   ↓
[퇴근]
   │     · 단말기/웹/앱으로 퇴근 시각 기록
   │     · 시스템이 일일 총 근무 시간·시간외·야간 자동 계산
   │     · 지각·조퇴·결근 상태 자동 판정
   ↓
[일별 집계 완료]
```

### 4.2 주간 누적 + 52시간 감시

```
[직원 퇴근 등록 시점]
   │
   ↓
[직원별 주간 누적 시간 계산]
   │     · 일별 총 근무 시간 합산
   │     · 시간외근무 시간 별도 합산
   ↓
[임계값 도달 확인]
   │
   ├── 44h 주의 → 알림만
   ├── 48h 경고 → 알림 + 인사담당자 보고
   └── 52h 차단 → 추가 시간외 자동 차단 (한국 법인)
   ↓
[WorkHourAlert 생성]
   │
   ↓
[인사담당자 검토 → 사유 입력 + 해결 처리]
```

### 4.3 월별 근태 마감 (급여 연동)

```
[월말 ~ 익월 5일경]
   │
   ↓
[근태 마감 시작] ── 인사담당자가 해당 월 모든 출퇴근·휴가·휴직 확정
   │
   ↓
[정정 신청 마감] ── 직원의 출퇴근 정정 신청 기한
   │
   ↓
[근태 마감 처리] ── 시스템에 "마감" 표시
   │
   ↓
[급여 모듈로 데이터 전달] ── 급여 매뉴얼 §3.2 8단계 워크플로의 2단계 (근태 마감)로 진입
```

### 4.4 자동 알림

- **출근 미체크**: 표준 출근 시각 + 1시간 경과 시 직원·팀장에게 알림
- **주간 시간 임계값 도달**: 주의(44h)/경고(48h)/차단(52h) 단계별 알림
- **출퇴근 정정 결재 도착**: 결재자(팀장)에게 알림
- **교대 변경 결재 결과**: 신청자에게 알림
- **단말기 장애**: 단말기 heartbeat 중단 시 인사담당자에게 알림

## 6. 화면 안내

### 6.1 직원 — 내 근태 화면

위치: `근태 → 내 근태` 또는 `프로필 → 근태`

- **오늘 출퇴근 카드**: 오늘 출근·퇴근 시각, 일일 총 근무 시간
- **출근/퇴근 버튼**: 큰 버튼 (사무직), 단말기는 자동
- **주간 요약**: 이번 주 일별 근무 시간 + 시간외 + 잔여 한도
- **월간 캘린더**: 일별 상태(정상/지각/조퇴/결근/휴가/휴직/공휴일) 시각화
- **정정 신청 버튼**: 출퇴근 정정 신청 폼
- **사전 연장근무 신청**: 예정 일자·시간 입력

### 6.2 팀장 — 팀 근태 화면

위치: `근태 → 팀 근태`

- **팀원 목록**: 오늘 출퇴근 상태 + 주간 누적 시간
- **52시간 경고 표시**: 임계값 도달 팀원 강조
- **결재 대기 목록**: 정정·연장·교대 변경 신청 (결재 매뉴얼 통합 결재함과 별도)
- **팀 월간 캘린더**: 팀원 일정 시각화

### 6.3 인사담당자 — 근태 관리 화면

위치: `근태 → 관리`

탭 구조:
1. **전사 출퇴근** — 회사별·날짜별 조회·필터·검색
2. **52시간 알림** — 미해결 알림 + 차단 해제 처리
3. **공휴일 관리** — 연도별 회사 공휴일 등록·삭제
4. **근태 정책** — 회사별 표준 시간·52시간 임계값·차단 활성화
5. **교대 패턴·조 관리** — 패턴 정의·조 멤버 관리·일정 배정
6. **단말기 관리** — 단말기 등록·heartbeat 모니터링
7. **월별 마감** — 회차별 근태 마감·해제

### 6.4 교대 캘린더

위치: `근태 → 교대 캘린더` 또는 `근태 → 교대 일정`

- **월간 캘린더**: 조별 색상 구분 (A조 빨강, B조 파랑, C조 초록 등)
- **직원별 조회**: 특정 직원의 한 달 슬롯 일정
- **일정 일괄 배정**: 반복 패턴으로 1~3개월 일정 자동 생성
- **개별 변경**: 특정 일자의 슬롯 변경 (사유 입력)

## 7. 특수 상황 처리

### 7.1 단말기 장애로 출퇴근 미기록

- 단말기 heartbeat가 24시간 이상 끊기면 인사담당자에게 알림
- 영향받는 직원은 임시로 웹/앱으로 출퇴근 기록
- 단말기 복구 후 보정 데이터 일괄 적재 (인사담당자 수동)
- 직원이 본인 정정 신청해도 됨

### 7.2 출퇴근 깜박 잊음

- 출근 또는 퇴근 한 쪽만 누락된 경우 → 정정 신청
- 양쪽 다 누락된 경우 → 결근으로 자동 분류, 정정 신청 또는 인사담당자가 사후 입력
- 정정 신청 결재는 직속 팀장 (결재 매뉴얼 §2.1 근태)

### 7.3 자정 통과 야간 근무

- 출근 22:00, 퇴근 익일 06:00 같은 자정 통과 근무
- 시스템이 자동으로 같은 일자(출근일)로 묶어 처리
- 시간 계산: 22:00~24:00 + 00:00~06:00 = 8시간 (전부 야간)
- 교대 슬롯이 자정 통과로 정의된 경우 슬롯 시간 그대로

### 7.4 휴가·휴직 중 출퇴근 입력 시도

- 휴가 승인된 일자에 출퇴근 시도: 시스템이 차단 또는 경고
- 휴직 중인 직원: 출퇴근 입력 불가
- 예외: 휴가 중에도 잠깐 출근한 경우 인사담당자가 수동으로 휴가 취소 후 출근 입력 (실무에서 드문 케이스)

### 7.5 재택 근무자의 출퇴근

- 모바일 앱·웹으로 시작 시각·종료 시각 기록
- 위치 인증(GPS) 옵션 (회사 정책)
- 단말기 출퇴근 면제

### 7.6 외근·출장 직원

- 외근·출장은 별도 신청·승인 시스템 없음 (현재 미지원)
- 운영 방법: 외근 시간 동안 사무실 단말기 출퇴근 못 함 → 정정 신청으로 처리
- **추후 개선** 항목: 외근·출장 신청·자동 출퇴근 처리 (§9 #4)

### 7.7 교대 변경 신청

- 직원 A가 B와 교대 일정 교환 희망
- A가 신청 → 양쪽 직속 팀장 결재 → 승인 시 ShiftSchedule 양쪽 swap
- 결재 매뉴얼 §2.1 근태 흐름 따름

### 7.8 52시간 차단 우회

- 특별인가 연장근로(정부 인가) 등 정당한 사유로 52시간 초과 필요
- 인사담당자가 해당 직원·해당 주 차단 해제 + 사유·근거(특별인가증) 입력
- 다음 급여 회차의 이상 검토에서 경고 표시 (급여 매뉴얼 §3.8)
- 감사 대상으로 모든 차단 해제 이력 보존

### 7.9 해외 법인 야간 근무 가산 정책 차이

- 한국: 22:00~06:00, 1.5배 가산 (시간외 중복 시 2배)
- 중국·미국 등: 각국 노동법 따름 (시스템은 시간만 집계, 가산은 외부 처리)
- 본 매뉴얼은 한국 정책 기반. 해외는 외부 처리 결과 업로드 시 가산 적용된 상태로 들어옴 (급여 매뉴얼 §3.1 참조)

### 7.10 휴일에 일한 경우

- 공휴일 또는 회사 휴무일에 출근한 경우
- 시간외근무로 자동 분류 (휴일 근로)
- 급여 매뉴얼 §3.4에서 가산율 1.5배(기본) 또는 2배(8시간 초과분) 적용

## 8. 자주 묻는 질문

**Q. 단말기 없는 곳에서 출퇴근하려면?**
A. 웹(`/attendance`) 또는 모바일 앱에서 출근/퇴근 버튼을 누르면 됩니다. 사무직은 웹, 외근·재택은 앱이 표준입니다.

**Q. 출근 도장을 깜박 안 찍었어요. 어떻게 하나요?**
A. `근태 → 내 근태`에서 정정 신청을 등록하세요. 출근 시각과 사유를 입력하면 직속 팀장 결재 후 정정됩니다.

**Q. 시간외근무 52시간 한도 알림이 왔어요. 일을 계속해도 되나요?**
A. 한국 법인은 주 52시간이 법정 한도입니다. 44시간 주의·48시간 경고는 정보성 알림이며 계속 근무 가능합니다. 52시간 도달 시 시스템이 시간외근무 추가 입력을 차단합니다. 특별 인가 필요 시 인사담당자에게 문의하세요.

**Q. 휴가 사용한 날에도 출근 도장을 찍나요?**
A. 아닙니다. 휴가 신청이 승인되면 그 날은 자동으로 "휴가" 상태로 표시되며 결근으로 처리되지 않습니다.

**Q. 공휴일에 출근하면 어떻게 계산되나요?**
A. 공휴일 근무는 시간외근무(휴일 근로)로 자동 분류됩니다. 급여 모듈에서 1.5배(8시간 초과분은 2배) 가산이 적용됩니다.

**Q. 교대 근무자도 시간외근무가 발생하나요?**
A. 네. 예정 교대 슬롯 시간을 초과해 근무한 경우 시간외근무로 집계됩니다. 야간 슬롯은 별도로 야간 근무 시간으로도 집계되어 급여 가산이 중첩 적용됩니다.

**Q. 자정을 넘는 야간 근무는 어느 날짜로 기록되나요?**
A. 출근 일자 기준으로 묶입니다. 예: 화요일 22:00 출근 → 수요일 06:00 퇴근 = 화요일 8시간 근무로 기록되며 전부 야간 시간으로 분류됩니다.

**Q. 외근·출장 다녀온 시간은 어떻게 처리되나요?**
A. 현재 시스템은 외근·출장 신청 기능을 별도로 제공하지 않습니다. 외근 시간 동안 단말기 출퇴근이 어려운 경우 정정 신청으로 처리하세요. 외근·출장 정식 워크플로는 **추후 개선** 항목입니다 (§9 #4).

**Q. 단말기 장애가 났어요. 어떻게 하나요?**
A. 시스템이 단말기 heartbeat 중단을 자동 감지하여 인사담당자에게 알림을 보냅니다. 직원은 웹 또는 앱으로 임시 출퇴근하시고, 단말기 복구 후 인사담당자가 정정 처리합니다.

**Q. 팀장이 제 출퇴근 시간을 다 보나요?**
A. 네. 팀장은 본인 직속 팀원의 출퇴근 기록을 조회할 수 있습니다 (다만 급여는 조회 불가 — 급여 매뉴얼 §5.3 참조). 다른 부서 직원은 점선·매트릭스 관계가 있는 경우만 조회 가능합니다.

**Q. 해외 법인은 52시간 한도가 적용되나요?**
A. 시스템은 한국 법인만 차단을 활성화합니다. 해외 법인은 알림만 발송하고 차단하지 않으며, 각국 노동법에 맞춰 회사별 정책으로 활성화 여부를 결정합니다.

## 11. 부록: 기술 메모

> 본문이 정책 설명, 이 섹션은 개발/QA 참고용. HR은 건너뛰어도 무방합니다.

### 11.1 데이터 모델

핵심 모델 (모두 `prisma/schema.prisma`):

- `Attendance` — 일별 출퇴근 기록 (clockIn/clockOut/workType/totalMinutes/overtimeMinutes/status)
- `AttendanceTerminal` — 출입 단말기 (지문·카드·얼굴인식, heartbeat 추적)
- `WorkSchedule` — 회사별 표준 근무 일정 (daily/shift config JSON)
- `EmployeeSchedule` — 직원별 일정 매핑 (시점 기반)
- `Holiday` — 공휴일 (회사·연도별, 대체공휴일 표시)
- `DesignatedLeaveDay` — 회사 지정연차일 (휴가 매뉴얼 §3.4와 공유)
- `AttendanceSetting` — 회사별 근태 설정 (표준 시간·52시간 임계값·차단 활성화·타임존)
- `WorkHourAlert` — 52시간 알림 로그 (주 단위, alertLevel: caution/warning/blocked)
- `ShiftPattern` — 교대 패턴 (slots JSON: name/start/end/breakMin/nightPremium)
- `ShiftGroup` — 교대 조 (A조·B조·C조 등)
- `ShiftGroupMember` — 조 멤버 (시점 기반)
- `ShiftSchedule` — 교대 일정 (employee·date·slot·night/status)
- `ShiftChangeRequest` — 교대 변경 신청
- `AttendanceApprovalRequest` + `AttendanceApprovalStep` — 통합 근태 결재 (결재 매뉴얼 §11.1 모델 참조)

**enum**:
- `AttendanceStatus`: NORMAL / LATE / EARLY_OUT / ABSENT
- `WorkType`: NORMAL / OVERTIME / NIGHT / HOLIDAY (근무 유형 분류)
- `ClockMethod`: 단말기 / 웹 / 앱 등 (Attendance.clockInMethod·clockOutMethod)
- `TerminalType`: FINGERPRINT / CARD_READER / FACE_RECOGNITION
- `ShiftPatternType`: TWO_SHIFT / THREE_SHIFT / DAY_NIGHT_OFF / FOUR_ON_TWO_OFF / CUSTOM
- `ShiftScheduleStatus`: SCHEDULED / WORKED / ABSENT / SWAPPED
- `ShiftChangeRequestStatus`: SCR_PENDING / SCR_APPROVED / SCR_REJECTED

### 11.5 알려진 위험 코드 경로

- `workTypeEngine.ts` — 자정 통과 야간 근무 처리, 4가지 근무 유형 분기. 신규 유형 추가 시 분기 보완 필수
- 52시간 차단의 race condition — 동시에 여러 시간외근무 신청 시 임계값 계산 갱신 race 가능 (트랜잭션 보호 필요)
- 휴식 시간 자동 차감 — `workTypeEngine`이 4시간/8시간 기준으로 30분/1시간 차감. 실제 휴식이 다르면 부정확
- 자정 통과 일자 묶음 — 출근 일자 기준 묶음 정책. 글로벌 타임존 차이가 큰 경우(예: 멕시코 야간 → 미국 본사 다음날) 명확성 확인 필요
- 공휴일 누락 법인의 영업일 계산 — `getWeekdaysInMonth`가 공휴일 부재 법인에서 주말만 제외, 공휴일 영업일로 카운트 → 휴가 일수·급여 일할 계산 부정확 가능 (휴가 매뉴얼 §9 #2와 동일 위험)
- 교대 슬롯 시각 문자열 — `startTime`/`endTime`이 'HH:mm' 문자열. 시간 비교·계산 시 매번 파싱 필요. Date 변환 누락 시 silent bug 가능
- 단말기 API secret — `AttendanceTerminal.apiSecret`이 평문 저장. 보안 강화 필요 (해시·암호화)
