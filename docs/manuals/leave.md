> **최종 갱신**: 2026-05-11
> **담당**: 인사관리팀

## 1. 개요

### 1.1 무엇을 다루는가

CTR HR Hub의 휴가관리 모듈은 다음을 통합 관리합니다.

- **휴가 유형 정의** — 회사별 사용 가능한 휴가 종류 (연차/병가/경조/특별 등)
- **부여** — 입사일/근속연수/회계연도에 따른 자동 부여
- **신청·승인 흐름** — 직원 신청 → 팀장 승인 → 잔액 차감
- **잔액 관리** — 연도별 부여/사용/이월/조정 잔액 추적
- **사용촉진** — 미사용 연차 자동 통보 (한국 근로기준법 §61)
- **연말 갱신** — 이월/소멸 처리 + 신규 부여
- **통계 대시보드** — 사용률, 사용 예측, 회사별 분포

### 1.2 도입 의도

- **취업규칙·경조지침 디지털화** — 종이/엑셀로 관리하던 13개 법인 휴가 정책을 시스템에서 통합 운영
- **법규 준수 자동화** — 한국 근로기준법 §60(연차), §61(사용촉진), 모성보호 관련 법규를 시스템에서 자동 적용
- **일관성** — 다국적 법인의 정책 차이를 회사별로 명확히 구분 (한국 / 중국 / 베트남 / 멕시코 등)
- **감사 가능성** — 모든 부여/사용/조정에 대한 이력 추적

## 2. 휴가 유형

휴가 유형은 **회사별로 별도 정의**됩니다. 한국 본사(CTR)는 한국 법규 기반, 해외 법인은 각국 노동법 기반입니다. 총 13개 법인에 약 220건의 휴가 유형이 사전 등록되어 있습니다.

### 2.1 카테고리 6종

| 카테고리 | 예시 휴가 유형 |
|----------|---------------|
| 연차 | 연차유급휴가 |
| 경조사 | 본인 결혼, 부모 사망, 자녀 결혼, 배우자 사망 등 |
| 모성·육아 | 출산휴가, 배우자 출산휴가, 유산휴가 |
| 건강 | 병가, 생리휴가 |
| 병역 | 예비군훈련, 민방위 |
| 기타 | 특별휴가, 보상휴가 등 |

**화면 그룹핑**: 휴가 신청 화면에서 이 6개 카테고리로 그룹핑되어 표시되며, 직원이 카테고리 단위로 접고 펼칠 수 있습니다.

### 2.2 각 휴가 유형이 가지는 속성

| 속성 | 설명 |
|------|------|
| 이름 | 표시 이름 (예: 연차유급휴가, 본인 결혼) |
| 유급 여부 | 급여 지급 / 미지급 |
| 카테고리 | 위 6종 중 하나 |
| 일수 계산 방식 | 영업일 기준 / 달력일 기준 |
| 공휴일 포함 여부 | 통산(경조사) / 제외(연차) |
| 반차 사용 가능 | 0.5일 단위 신청 가능 여부 |
| 증빙 필수 여부 | 진단서·청첩장 등 첨부 필수 |
| 최소 사전 신청 일수 | 며칠 전까지 신청해야 하는지 |
| 최대 연속 사용 일수 | 한 번에 최대 며칠까지 사용 가능 |
| 분할 사용 가능 | 기간을 나눠 사용 가능 (예: 배우자 출산휴가) |
| 분할 사용 기한 | 분할 사용 시 마감 기한 (일) |
| 연간 최대 사용 횟수 | 평생/연간 최대 사용 횟수 (예: 본인 결혼 1회) |
| 부분 유급 시 유급 일수 | 일부만 유급인 경우 며칠까지 유급 |
| 경조금 | 경조사 시 지급 금액 (원) |

### 2.3 한국 vs 해외 법인 차이

**한국 (CTR)**:
- 근로기준법 §60 기반 연차 (1년 미만 월 1일, 1년 이상 15일, 2년마다 +1일, 최대 25일)
- 경조사 휴가 14종 (취업규칙 별표) — 본인 결혼 5일, 부모 사망 5일 등
- 모성보호: 출산휴가 90일, 배우자 출산휴가 10일, 유산휴가 5~90일 (임신 주차별)
- 생리휴가 (월 1일, 무급 가능)
- 예비군훈련, 민방위

**중국 (CTR-CN)**:
- 근속 기반 연차: 1년 미만 0일, 1~10년 5일, 10~20년 10일, 20년+ 15일
- 병가 별도

**멕시코·미국 (CTR-US)**:
- 멕시코 연방노동법(Ley Federal del Trabajo, 2023 개정) 기준 — 정기 휴가, 병가, 출산휴가, 부성휴가, 경조사휴가, 무급휴가
- 시드는 스페인어로 정확하게 작성되어 있음

**베트남·러시아·유럽**: 각국 노동법 기반 별도 시드.

> **데이터 흐름 주의**: 해외 5개 법인(미국·멕시코·베트남·러시아·유럽) 휴가 유형 시드가 마스터 적재 절차에 연결되어 있지 않아 **실제 운영 DB에 들어가지 않았을 가능성**이 있습니다. 별도 작업으로 정리 예정 (§9 알려진 제약 #1 참조).

## 3. 핵심 정책

### 3.1 연차 부여 방식

**한국 법인** 기준 (근로기준법 §60):

| 근속 기간 | 부여 일수 | 부여 시점 |
|----------|----------|----------|
| 1개월 ~ 1년 미만 | 매월 개근 시 1일 (최대 11일) | 매월 만근 후 |
| 1년 이상 | 15일/년 | 입사 기념일 또는 1월 1일 |
| 3년 이상 | 16일/년 | 이후 2년마다 +1일 |
| 5년 이상 | 17일/년 | 〃 |
| ... | 2년마다 +1일 | 〃 |
| 21년 이상 | 25일/년 (상한) | 〃 |

**부여 방식 3종**:
- **연 1회 일괄 부여** — 1월 1일 또는 입사 기념일에 한꺼번에 부여
- **매월 부여** — 입사 첫 해에 매월 1일씩 부여
- **수동 부여** — 인사담당자가 직접 입력 (특수 케이스)

**부여 기준일**: 회계연도(매년 1월 1일) 또는 입사 기념일 — 회사별 정책으로 선택.

### 3.2 일수 계산

휴가 일수 계산은 **일수 계산 방식** × **공휴일 포함 여부** 4가지 조합으로 동작합니다.

| 일수 계산 방식 | 공휴일 포함 | 동작 |
|--------------|------------|------|
| 영업일 기준 | 제외 (기본) | 주말 + 공휴일 제외 |
| 영업일 기준 | 포함 | 주말만 제외, 공휴일은 사용일수 차감 |
| 달력일 기준 | 포함 | 모든 날 포함 (경조사·출산 통산) |
| 달력일 기준 | 제외 | 주말 포함, 공휴일 제외 |

**예시 (한국 기준)**:
- **연차 5일 신청** (월~금) → 영업일 기준 + 공휴일 제외 → 5일 차감
- **부모 사망 5일** (목~월, 토일 포함) → 달력일 + 통산 → 5일 차감 (주말 포함)
- **출산휴가 90일** → 달력일 + 통산 → 모든 날 포함

**반차**: 휴가 유형이 반차를 허용하면 0.5일 단위 신청 가능. 오전/오후 구분.

### 3.3 이월 규칙

연말에 미사용 잔액을 다음 해로 넘기는 정책. 휴가 유형별로 다르게 설정.

| 이월 정책 | 동작 |
|----------|------|
| 이월 없음 | 연말 자동 소멸 |
| 무제한 이월 | 전액 이월 |
| 상한 이월 | 최대 N일까지만 이월 |
| 기한 만료 이월 | 이월 후 N개월 내 사용, 미사용 시 소멸 |

**한국 법인 표준**: 연차는 미사용 시 익년도 일부만 이월(법정 사용촉진 절차 충족 시 소멸 가능).

### 3.4 지정연차

회사가 특정 날짜를 연차 사용일로 지정하는 기능.

- 예: "여름 집단휴가" (8월 첫째 주), "설 연휴 사이 평일"
- 회사별로 연 단위 지정
- 직원은 별도 신청 없이 해당 일에 자동으로 연차 1일 차감
- 인사담당자가 설정 → 휴가 → 지정연차 탭에서 등록

### 3.5 잔액 관리

연도별 휴가 잔액은 다음 항목으로 구성됩니다.

| 항목 | 의미 |
|------|------|
| 부여 | 회사가 부여한 일수 (연 1회 또는 매월 누적) |
| 사용 | 승인 완료 후 사용한 일수 |
| 신청 중 | 신청 후 승인 대기 중인 일수 (잔액에서 잠정 차감) |
| 이월 | 전년도에서 이월된 일수 |
| 조정 | 인사담당자가 수동 조정한 일수 (보너스 부여, 차감 등) |
| 소멸 예정일 | 이월분 소멸 예정일 |

**가용 일수 계산**: `부여 + 이월 + 조정 - 사용 - 신청 중`

### 3.6 마이너스 연차

회사 정책에 따라 잔액보다 더 많이 신청할 수 있도록 허용 (한도 내).

- 회사별 설정 (활성화 여부 + 한도 일수)
- 예: 한도 −3일 → 잔액이 0이어도 3일까지 추가 신청 가능
- 퇴사 시 마이너스 잔액은 일당 단가로 환산하여 마지막 급여에서 자동 차감

### 3.7 사용촉진제도

한국 근로기준법 §61 자동 이행.

- 매년 회계연도 종료 60일 전 → 사용 가능 일수 1차 통보
- 30일 전 → 2차 통보
- 10일 전 → 3차 최종 통보
- 모든 통보 이력 저장 → 미사용 잔액 소멸 시 법적 근거가 됨

## 4. 워크플로

### 4.1 휴가 신청 → 사용 라이프사이클

```
[직원]           [팀장]             [시스템]
신청 ──────────→ 승인 대기 ────────→ 잔액에서 잠정 차감
                  │
                  ├── 승인 ────────→ 승인 완료 → 사용 확정
                  └── 반려 ────────→ 반려 → 잔액 복구
```

**상태 4종**: 승인 대기 / 승인 완료 / 반려 / 취소

### 4.2 신청 시 검증

시스템이 자동으로 확인하는 항목:

- 잔액 충분한가? (마이너스 허용 정책 고려)
- 사전 신청 일수 충족? (예: 3일 전 신청 필수)
- 최대 연속 사용일 초과 안 함?
- 증빙 첨부 필요한 휴가 유형인가? (병가/경조사 등)
- 같은 팀에서 동시 부재가 많은가? (경고 표시, 차단 아님)
- 동일 기간 중복 신청 방지 (동시 신청 충돌 방지)

### 4.3 취소 시나리오 3종

| 시나리오 | 처리 |
|---------|------|
| **사용 전 취소** (시작일 전) | 직원이 직접 취소 → 잔액 복구 |
| **사용 중 취소** (시작~종료일 내) | 사용일까지 차감, 나머지 복구. 팀장 승인 필요 |
| **사용 후 취소** (종료일 후) | 인사담당자만 가능. 사유 기록 필수 |

### 4.4 자동 알림

- **휴가 시작 D-7/D-3/D-1**: 직원·팀장에게 사전 알림
- **승인 대기**: 팀장에게 결재 요청 알림 (Teams + 이메일 + 푸시)
- **승인/반려 결과**: 신청자에게 알림
- **사용촉진**: 60일/30일/10일 전 미사용 잔액 통보

### 5.2 결재 라우팅 — 누가 누구의 신청을 결재하는가

**시스템 현재 동작**:

| 휴가 유형 | 시스템 결재자 | 단계 | 비고 |
|----------|--------------|------|------|
| **일반 휴가 (연차/병가/경조 등 모든 유형)** | 신청자의 **직속 팀장** | 1단계 전결 | 위임전결규정(CP-A-03-04) 반영. 시스템 기본값 |
| 인사담당자/전사관리자 직접 승인 | 인사담당자 또는 전사관리자 | 1단계 | 직속 팀장 우회 가능. 출산휴가 등 HR 대리 등록 시 자동 승인 경로로 사용 |
| 팀장 부재 시 위임 | 위임받은 사람 | 1단계 | 위임 설정 기반 |

**다른 모듈의 결재 라우팅 (참고용)**:

| 모듈 | 결재자 | 단계 |
|------|--------|------|
| 급여 | 본부장 → 대표 | 2단계 |
| 채용 | 본부장 | 1단계 |
| 징계 | 인사담당자 → 대표 | 2단계 |
| 인사발령 | 대표 전결 | 1단계 |
| 퇴직/사직 | 본부장 → 대표 | 2단계 |

**Settings UI 현황과 한계**:

`설정 → 시스템 → 결재 플로우` 탭에서 13개 모듈(휴가 포함) × 회사별로 결재 흐름을 등록할 수 있는 화면이 존재합니다. 단계마다 직속 팀장 / 인사담당자 / 본부장 / 대표 / 특정 사용자를 결재자로 지정 가능합니다.

**그러나 휴가 모듈은 현재 이 설정을 읽지 않습니다.** 설정 화면에서 휴가 → 본부장 2단계로 등록해도 실제 휴가 신청·승인 시 시스템이 무시하고 직속 팀장 1단계로만 동작합니다. 시스템 한계:

| 항목 | 현재 동작 |
|------|----------|
| 결재 플로우 UI 등록 | 가능 (화면은 동작) |
| 휴가 신청 시 결재자 자동 지정 | **안 함** (휴가 신청 코드가 결재자 필드를 설정 안 함) |
| 다단계 결재 흐름 실행 | **안 함** (휴가 승인 코드가 1단계로 종료) |
| 휴가 유형별 차등 결재 (예: 출산휴가만 대표 결재) | **불가** (데이터 모델 자체가 모듈 단위만 지원, 유형별 매핑 필드 없음) |

**중요 사실 요약**:

- 휴가 모듈에는 본부장·대표가 결재자로 등록되어 있지 않습니다. 시스템 기본값은 직속 팀장 1단계입니다.
- 장기휴가(5일+) 2단계 시드(직속 팀장 → 인사담당자)는 데이터로 존재하지만 코드 미연동으로 실행되지 않습니다.
- 본부장·대표 결재가 운영상 필요한 경우의 현재 우회 방법: (a) 인사담당자가 대리 등록 + 자동 승인, (b) 위임 설정으로 결재 권한 임시 부여.

**추후 개선** (별도 기능 개발 트랙):

휴가 유형별·회사별 차등 결재 흐름을 제대로 동작시키려면 다음 작업이 필요합니다.

1. 데이터 모델 확장 — 결재 흐름에 휴가 유형 매핑 필드 추가
2. 휴가 신청 코드 수정 — 결재 흐름 조회 후 첫 단계 결재자를 자동 지정
3. 휴가 승인 코드 수정 — 다단계 흐름 실행 (다음 단계 라우팅 또는 완료 처리)
4. 결재 플로우 설정 UI에 휴가 유형 선택 추가

### 6.1 직원 — 내 휴가 화면

- **상단 KPI 카드**: 연차 잔액, 지정연차 일수, 올해 사용 일수
- **신청 버튼**: 모달 폼 (휴가 유형 → 카테고리 그룹핑 → 기간 선택 → 사유 → 증빙 첨부)
- **내 신청 이력**: 상태별(대기/승인/반려/취소) 필터, 페이지네이션
- **휴가 캘린더**: 본인의 휴가 + 같은 팀원의 휴가 시각화

### 6.2 팀장 — 팀 휴가 화면

- **결재 대기 목록**: 팀원 신청 건 일괄 조회 + 일괄 승인/반려
- **팀 현황 캘린더**: 팀원 휴가 시각화 (동시 부재 확인)
- **팀원별 잔액 현황**

### 6.3 인사담당자 — 휴가 관리 화면

탭 구조:

1. **휴가 유형 관리** — 회사별 휴가 유형 등록/수정, 카테고리/일수 계산 방식 설정
2. **부여 규칙 관리** — 휴가 유형별 부여 규칙 설정
3. **잔액 관리** — 직원별 잔액 조회, 수동 조정, 일괄 부여
4. **연말 갱신** — 회계연도 종료 시 이월/소멸/신규 부여 일괄 실행
5. **지정연차 관리** — 연도별 회사 지정일 등록
6. **사용촉진 로그** — 자동 통보 이력 조회
7. **통계 대시보드** — 사용률, 사용 예측, 회사별 분포

## 7. 특수 상황 처리

### 7.1 휴직 중 직원의 휴가

- 휴직 기간 동안 신규 휴가 신청 불가
- 휴직 시작 전 승인된 휴가는 그대로 유지
- 휴직 종료 후 복귀 시점에 따라 연차가 비례 부여될 수 있음 (회사 정책)

### 7.2 퇴사 예정자의 잔여 휴가

- **잔여 연차**: 퇴사일 전까지 사용 권장. 미사용 시 회사 정책에 따라 수당 지급 (한국 표준: 연차수당)
- **마이너스 연차**: 일당 단가로 환산 → 마지막 급여에서 차감
- **퇴직 절차**: 정산 단계에서 자동 계산

### 7.3 법인 변경자 (전입·전출)

- 한 직원이 한 법인에서 다른 법인으로 이동 시
- 이동 전 법인의 잔액은 정산 (퇴사 동일 처리)
- 이동 후 법인의 정책으로 신규 부여 (입사일 기준)
- 겸직인 경우 양 법인 모두에서 휴가 신청 가능 (각 법인 잔액 별도 관리)

### 7.4 입사 첫 해 (Pro-rated)

- 한국: 매월 만근 시 1일 부여 (최대 11일까지)
- 1년 도달 시 15일 일괄 부여 (월별 부여분과 별개)
- 첫 해 11일 + 1년차 15일 = 최대 26일 사용 가능 (근로기준법 §60②)

### 7.5 해외 법인 정책 차이 (예시)

- **중국**: 1년 미만 0일, 근속 기반 부여
- **베트남**: 일부 휴가 유형의 유급/무급 처리 정합성 HR팀 확인 필요 (조부모사망 등)
- **멕시코**: 시드는 정확히 작성되어 있으나 마스터 적재 절차 미연결 (§9 참조)

## 8. 자주 묻는 질문

**Q. 휴가 신청 후 취소하면 잔액이 즉시 복구되나요?**
A. 사용 전 취소는 자동으로 복구됩니다. 사용 중/후 취소는 사용일 만큼 차감된 후 나머지가 복구되며, 사용 후 취소는 인사담당자만 처리 가능합니다.

**Q. 반차를 두 번 쓰면 연차 1일이 차감되나요?**
A. 네. 0.5일 단위로 차감되며, 오전·오후 동일 날짜에 두 번 쓰면 1일 차감됩니다. 단, 반차를 허용하지 않는 휴가 유형은 반차 불가입니다.

**Q. 경조사 휴가에 주말이 포함되면 어떻게 계산되나요?**
A. 대부분 경조사 휴가는 달력일 + 공휴일 포함(통산) 정책입니다. 예: 부모 사망 5일을 목요일에 시작하면 토·일 포함 5일이 차감됩니다.

**Q. 연차 사용촉진 통보를 못 받았는데 미사용 잔액이 소멸되나요?**
A. 시스템이 60/30/10일 전 자동 통보합니다. 통보 이력이 시스템에 기록되며, 통보 후에도 사용 안 한 잔액만 법적으로 소멸 가능합니다.

**Q. 한국 본사와 중국 법인의 연차 일수가 다른데, 어디서 설정하나요?**
A. 휴가 유형은 회사별로 별도 정의됩니다. 인사담당자가 휴가 관리 → 휴가 유형 관리 탭에서 회사를 선택하여 등록·수정합니다.

**Q. 마이너스 연차는 항상 허용되나요?**
A. 회사별 설정입니다. 인사담당자가 회사 정책에서 마이너스 연차 허용 토글로 활성화하고 한도를 지정합니다. 기본값은 비활성화입니다.

**Q. 본부장이나 대표가 직접 승인해야 하는 휴가가 있나요?**
A. 현재 시스템에서 휴가는 직속 팀장 1단계 결재가 기본값이며, 본부장·대표는 휴가 결재자로 시드되어 있지 않습니다. 인사담당자/전사관리자는 모든 휴가를 직접 승인할 수 있습니다 (출산휴가 등 HR 대리 등록 시 자동 승인 경로).

**Q. 설정 화면에서 결재 흐름을 휴가 → 본부장 2단계로 등록할 수 있나요?**
A. 설정 → 시스템 → 결재 플로우 탭에서 화면상으로는 등록 가능하지만, **현재 휴가 모듈은 이 설정을 읽지 않습니다.** 등록해도 실제 동작은 직속 팀장 1단계로만 진행됩니다. 휴가 유형별(예: 출산휴가만 대표 결재) 차등도 데이터 모델에 없습니다. 이 두 가지는 **추후 개선** 항목입니다 (§5.2 + §9 #4·#5 참조).
