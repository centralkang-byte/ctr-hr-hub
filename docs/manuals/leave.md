## 1. 개요

### 1.1 무엇을 다루는가

CTR HR Hub의 휴가관리 모듈은 다음을 통합 관리합니다.

- **휴가 유형 정의** — 회사별 사용 가능한 휴가 종류 (연차/병가/경조/특별 등)
- **부여(Accrual)** — 입사일/근속연수/회계연도에 따른 자동 부여
- **신청·승인 워크플로** — 직원 신청 → 팀장 승인 → 잔액 차감
- **잔액 관리** — 연도별 부여/사용/이월/조정 잔액 추적
- **사용촉진** — 미사용 연차 자동 통보 (한국 근로기준법 §61)
- **연말 갱신** — 이월/소멸 처리 + 신규 부여
- **통계 대시보드** — 사용률, burn-down 예측, 회사별 분포

### 1.2 도입 의도

- **취업규칙·경조지침 디지털화** — 종이/엑셀로 관리하던 13개 법인 휴가 정책을 시스템에서 통합 운영
- **법규 준수 자동화** — 한국 근로기준법 §60(연차), §61(사용촉진), 모성보호 관련 법규를 코드로 강제
- **일관성** — 다국적 법인의 정책 차이를 명시적 데이터로 구분 (한국 ↔ 중국 ↔ 베트남 ↔ 멕시코 등)
- **감사 가능성** — 모든 부여/사용/조정에 대한 추적 가능한 이력

## 2. 휴가 유형

휴가 유형은 **회사별로 별도 정의**됩니다. 한국 본사(CTR)는 한국 법규 기반 7개 카테고리, 해외 법인은 각국 노동법 기반으로 시드됩니다. 총 13개 법인에 약 220건의 휴가 유형이 사전 등록되어 있습니다.

### 2.1 카테고리 6종

| 카테고리 | 영문 코드 | 예시 휴가 유형 |
|----------|----------|---------------|
| 연차 | `annual` | 연차유급휴가 |
| 경조사 | `family_event` | 본인 결혼, 부모 사망, 자녀 결혼 등 |
| 모성·육아 | `maternity` | 출산휴가, 배우자 출산휴가, 유산휴가 |
| 건강 | `health` | 병가, 생리휴가 |
| 병역 | `military` | 예비군훈련, 민방위 |
| 기타 | `other` | 특별휴가, 보상휴가 등 |

**UI 그룹핑 정책 (Phase B7)**: 휴가 신청 화면에서 이 6개 카테고리로 그룹핑되어 표시됩니다. 직원이 휴가 유형을 빠르게 찾을 수 있도록 카테고리 단위로 접고 펼칠 수 있습니다.

### 2.2 핵심 속성

각 휴가 유형은 다음 속성을 가집니다.

| 속성 | 설명 | 예시 |
|------|------|------|
| `code` | 시스템 코드 | `KR-ANNUAL`, `marriage_self`, `bereavement_parent` |
| `name` | 표시 이름 | 연차유급휴가, 본인 결혼, 부모 사망 |
| `isPaid` | 유급 여부 | true / false |
| `category` | 카테고리 | annual / family_event / maternity / health / military / other |
| `subcategory` | 하위 분류 (경조사) | celebration (경사) / condolence (조사) |
| `countingMethod` | 일수 계산 방식 | business_day (영업일) / calendar_day (역일) |
| `includesHolidays` | 공휴일 포함 여부 | true (경조사 통산) / false (연차) |
| `allowHalfDay` | 반차 사용 가능 | true / false |
| `requiresProof` | 증빙 필수 여부 | true (병가/경조사) / false |
| `minAdvanceDays` | 최소 사전 신청 일수 | 3 (3일 전 신청 필수) |
| `maxConsecutiveDays` | 최대 연속 사용 일수 | 30 (한 번에 30일 이상 불가) |
| `isSplittable` | 분할 사용 가능 | true (배우자 출산 등) |
| `splitDeadlineDays` | 분할 사용 기한 | 90 (출산일 후 90일 이내) |
| `maxPerYear` | 연간 최대 사용 횟수 | 1 (본인 결혼 평생 1회) |
| `paidDaysPerYear` | 부분 유급 시 유급 일수 | 60 (병가 60일까지 유급) |
| `condolenceAmount` | 경조금 (KRW) | 500000 (부모 사망 50만원) |

### 2.3 한국 vs 해외 법인 차이

**한국 (CTR)**:
- 근로기준법 §60 기반 연차 (1년 미만 월 1일, 1년 이상 15일, 2년마다 +1일, 최대 25일)
- 경조사 휴가 14종 (취업규칙 별표) — 본인 결혼 5일, 부모 사망 5일 등
- 모성보호: 출산휴가 90일, 배우자 출산휴가 10일, 유산휴가 5~90일 (임신 주차별)
- 생리휴가 (월 1일, 무급 가능)
- 예비군훈련, 민방위

**중국 (CTR-CN)**:
- 근속 기반 연차: 1년 미만 0일, 1~10년 5일, 10~20년 10일, 20년+ 15일
- 병가 (Sick Leave) 별도

**멕시코·베트남·미국·러시아·유럽**: 각국 노동법 기반 별도 시드 (`prisma/seeds/35-statutory-leave-types.ts` 참조).

- 멕시코: 연방노동법(Ley Federal del Trabajo, 2023 개정) 기준 — `Vacaciones`, `Incapacidad por enfermedad`, `Permiso de paternidad` 등 스페인어 시드
- 베트남: 일부 휴가 유형의 유급/무급 처리 정합성 법무팀 검토 대기

> **데이터 흐름 주의**: `35-statutory-leave-types.ts`는 시드 정의는 정확하나 마스터 orchestrator(`prisma/seed.ts`)에 연결되어 있지 않음 → **실제 staging/prod DB에 해외 법인 휴가 유형이 들어가지 않았을 가능성**. 별도 코드 PR로 fix 예정 (§9 알려진 제약 #1 참조).

## 3. 핵심 정책

### 3.1 연차 부여 방식 (Accrual)

**한국 법인** 기준 (근로기준법 §60):

| 근속 기간 | 부여 일수 | 부여 시점 |
|----------|----------|----------|
| 1개월 ~ 1년 미만 | 매월 개근 시 1일 (최대 11일) | 매월 만근 후 |
| 1년 이상 | 15일/년 | 입사 기념일 또는 1월 1일 |
| 3년 이상 | 16일/년 | 〃 (2년마다 +1일) |
| 5년 이상 | 17일/년 | 〃 |
| ... | 2년마다 +1일 | 〃 |
| 21년 이상 | 25일/년 (상한) | 〃 |

**부여 방식 (accrualType)**:
- `annual` — 연 1회 일괄 부여 (1월 1일 또는 입사 기념일)
- `monthly` — 매월 부여 (입사 첫 해 적용)
- `manual` — 인사담당자 수동 입력 (특수 케이스)

**부여 기준일 (accrualBasis)**:
- `calendar_year` — 매년 1월 1일 기준
- `hire_date_anniversary` — 입사 기념일 기준 (한국 근로기준법 표준)

### 3.2 일수 계산 엔진

휴가 일수 계산은 `countingMethod` × `includesHolidays` 4가지 조합으로 동작합니다.

| 계산 방식 | 공휴일 포함 | 동작 |
|----------|------------|------|
| `business_day` | false (기본) | 주말 + 공휴일 제외 |
| `business_day` | true | 주말만 제외, 공휴일 사용일수 차감 |
| `calendar_day` | true | 모든 날 포함 (경조사·출산 통산) |
| `calendar_day` | false | 주말 포함, 공휴일 제외 |

**예시 (한국 기준)**:
- **연차 5일 신청** (월~금) → `business_day` + 공휴일 제외 → 5일 차감
- **부모 사망 5일** (목~월, 토일 포함) → `calendar_day` + 통산 → 5일 차감 (주말 포함)
- **출산휴가 90일** (calendar_day + 통산) → 모든 날 포함

**반차 (Half-day)**:
- `allowHalfDay = true` 인 휴가 유형에 대해 0.5일 단위 신청 가능
- 오전/오후 구분 (`halfDayType`)

### 3.3 이월 규칙 (Carry-over)

연말에 미사용 잔액의 다음 해 이월 정책. 휴가 유형별로 다르게 설정.

| carry-over type | 동작 |
|----------------|------|
| `none` | 이월 없음 (연말 자동 소멸) |
| `unlimited` | 전액 이월 |
| `capped` | 최대 N일까지만 이월 (`carryOverMaxDays`) |
| `expiry_months` | 이월 후 N개월 내 사용 (`carryOverExpiryMonths`) |

**한국 법인 표준**: 연차는 미사용 시 익년도 일부만 이월(법정 사용촉진 절차 충족 시 소멸 가능).

### 3.4 지정연차 (Designated Leave Day)

회사가 특정 날짜를 연차 사용일로 지정하는 기능 (Phase B3).

- 예: "여름 집단휴가" (8월 첫째 주), "설 연휴 사이 평일"
- 회사별로 연 단위 지정
- 직원은 별도 신청 없이 해당 일에 자동으로 연차 1일 차감
- 인사담당자가 Settings → 휴가 → 지정연차 탭에서 등록

### 3.5 잔액 관리 — 단일 SSOT

**LeaveYearBalance**가 연도별 휴가 잔액의 단일 출처입니다 (Phase 6에서 통합).

| 필드 | 의미 |
|------|------|
| `entitled` | 회사가 부여한 일수 (연 1회 또는 매월 누적) |
| `used` | 승인 완료 후 사용한 일수 |
| `pending` | 신청 후 승인 대기 중인 일수 (잔액에서 잠정 차감) |
| `carriedOver` | 전년도에서 이월된 일수 |
| `adjusted` | 인사담당자가 수동 조정한 일수 (보너스 부여, 차감 등) |
| `expiresAt` | 이월분 소멸 예정일 |

**가용 일수 계산**: `entitled + carriedOver + adjusted - used - pending`

### 3.6 마이너스 연차 (Negative Balance)

회사 정책에 따라 잔액보다 더 많이 신청할 수 있도록 허용 (한도 내).

- 회사별 설정 (`LeaveSetting.allowNegativeBalance` + `negativeBalanceLimit`)
- 예: limit = -3.0 → 잔액이 0이어도 3일까지 추가 신청 가능
- 퇴사 시 마이너스 잔액은 일당 단가로 환산하여 급여에서 자동 차감

### 3.7 사용촉진제도 (Phase B2)

한국 근로기준법 §61 자동 이행.

- 매년 회계연도 종료 60일 전 → 사용 가능 일수 1차 통보
- 30일 전 → 2차 통보
- 10일 전 → 3차 최종 통보
- 모든 통보 이력은 `LeavePromotionLog`에 저장 → 미사용 잔액 소멸 시 법적 근거가 됨

## 4. 워크플로 (State Machine)

### 4.1 휴가 신청 → 사용 라이프사이클

```
[직원]           [팀장]             [시스템]
신청 ──────────→ PENDING ──────────→ 잔액에서 pending 차감
                  │
                  ├── 승인 ────────→ APPROVED ──→ used 확정, pending 0
                  └── 반려 ────────→ REJECTED ──→ pending 복구
```

**상태 4종**: `PENDING` / `APPROVED` / `REJECTED` / `CANCELLED`

### 4.2 신청 시 검증

- 잔액 충분한가? (마이너스 허용 정책 고려)
- 사전 신청 일수 충족? (`minAdvanceDays`)
- 최대 연속 사용일 초과 안 함? (`maxConsecutiveDays`)
- 증빙 첨부 필요한 휴가 유형인가? (`requiresProof`)
- 같은 팀에서 동시 부재가 많은가? (경고 표시, 차단 아님)
- 동일 기간 중복 신청 안 함 (concurrency guard, DB 트랜잭션)

### 4.3 취소 시나리오 3종 (Phase 2)

| 시나리오 | 처리 |
|---------|------|
| **사용 전 취소** (시작일 전) | 직원이 직접 취소 → `CANCELLED` + 잔액 복구 |
| **사용 중 취소** (시작~종료일 내) | 사용일까지 차감, 나머지 복구. 팀장 승인 필요 |
| **사용 후 취소** (종료일 후) | 인사담당자만 가능. 사유 기록 필수 (`cancelNote`) |

### 4.4 자동 알림

- **휴가 시작 D-7/D-3/D-1**: 직원·팀장에게 사전 알림
- **승인 대기**: 팀장에게 결재 요청 알림 (Teams + 이메일 + 푸시)
- **승인/반려 결과**: 신청자에게 알림
- **사용촉진**: 60일/30일/10일 전 미사용 잔액 통보

### 5.2 결재 라우팅 — 누가 누구의 신청을 결재하는가

**시스템 기본 정책 (위임전결규정 CP-A-03-04 반영)**:

| 휴가 유형 | 결재자 | 단계 | 근거/비고 |
|----------|--------|------|----------|
| 일반 휴가 (연차/병가/경조 등) | 신청자의 **직속 팀장** | 1단계 전결 | `seeds/42-approval-flow-defaults.ts`: `module: leave, steps: [direct_manager]` |
| 출산휴가/육아 관련 | (실무) 인사담당자 직접 등록 | 0단계 | 시스템 강제 없음. 실무 컨벤션 — HR이 신청자 대리로 등록 + HR_ADMIN bypass로 자동 승인 |
| 휴직 (육아/질병 등) | **별도 모듈 (LOA)** 다단계 결재 | 2-3단계 | `LeaveOfAbsence` 모델 + 6-state 워크플로. 본 매뉴얼 범위 밖 (LOA 매뉴얼 참조) |
| 팀장 부재 시 위임 | 위임받은 사람 | 1단계 | F-2 Delegation. `ApprovalDelegation` 모델 |
| HR 검토 필요한 장기 연차 (예: 10일+) | **시스템 차등 없음** | — | 코드 강제 X. 회사 정책상 인사담당자 사전 협의 권장 |
| HR/SUPER_ADMIN 직접 승인 | HR_ADMIN/SUPER_ADMIN | 1단계 | 모든 신청에 대해 직속 팀장 우회 가능 (`isDirectApprover` bypass) |

**중요 사실**:
- **모든 휴가 유형이 시스템상 동일한 1단계 결재 룰**입니다. 휴가 유형별 결재자 차등은 시드에 정의되어 있지 않습니다.
- `ApprovalFlow` 모델 구조상 회사별·휴가유형별로 다른 flow(다단계, autoApproveDays 등)를 정의 가능하지만, **현재는 활용되지 않습니다**.
- 모성/장기 휴가 등 실무상 차등 처리가 필요한 경우는 (a) HR이 대리 등록 + 자동 승인, (b) LOA 별도 모듈 사용 — 두 가지 우회 방식으로 운영됩니다.

## 6. 화면 안내

### 6.1 직원 — `/my/leave`

- **상단 KPI 카드**: 연차 잔액, 지정연차 일수, 올해 사용 일수
- **신청 버튼**: 모달 폼 (휴가 유형 → 카테고리 그룹핑 → 기간 선택 → 사유 → 증빙 첨부)
- **내 신청 이력**: 상태별(대기/승인/반려/취소) 필터, 페이지네이션
- **휴가 캘린더**: 본인의 휴가 + 같은 팀원의 휴가 시각화

### 6.2 팀장 — `/leave/team`

- **결재 대기 목록**: 팀원 신청 건 일괄 조회 + 일괄 승인/반려
- **팀 현황 캘린더**: 팀원 휴가 시각화 (동시 부재 확인)
- **팀원별 잔액 현황**

### 6.3 인사담당자 — `/leave/admin`

탭 구조:

1. **휴가 유형 관리** — 회사별 휴가 유형 CRUD, 카테고리/일수 계산 방식 설정
2. **부여 규칙 관리** — 휴가 유형별 accrual rule 설정
3. **잔액 관리** — 직원별 잔액 조회, 수동 조정, 일괄 부여 (`bulk-grant`)
4. **연말 갱신** — 회계연도 종료 시 이월/소멸/신규 부여 일괄 실행
5. **지정연차 관리** — 연도별 회사 지정일 등록
6. **사용촉진 로그** — 자동 통보 이력 조회
7. **통계 대시보드** — 사용률, burn-down 예측, 회사별 분포

## 7. 특수 상황 처리

### 7.1 휴직 중 직원의 휴가

- 휴직(LOA) 기간 동안 신규 휴가 신청 불가
- 휴직 시작 전 승인된 휴가는 그대로 유지
- 휴직 종료 후 복귀 시점에 따라 연차가 비례 부여될 수 있음 (회사 정책)

### 7.2 퇴사 예정자의 잔여 휴가

- **잔여 연차**: 퇴사일 전까지 사용 권장. 미사용 시 회사 정책에 따라 수당 지급 (한국 표준: 연차수당)
- **마이너스 연차**: 일당 단가로 환산 → 마지막 급여에서 차감
- **오프보딩 워크플로**: GP#2 정산 단계에서 자동 계산 (`negative-balance-settlement.ts`)

### 7.3 법인 변경자 (전입·전출)

- 한 직원이 한 법인에서 다른 법인으로 이동 시
- 이동 전 법인의 잔액은 정산 (퇴사 동일 처리)
- 이동 후 법인의 정책으로 신규 부여 (입사일 기준)
- 겸직(`Concurrent Assignment`)인 경우 양 법인 모두에서 휴가 신청 가능 (각 법인 잔액 별도 관리)

### 7.4 입사 첫 해 (Pro-rated)

- 한국: 매월 만근 시 1일 부여 (최대 11일까지)
- 1년 도달 시 15일 일괄 부여 (월별 부여분과 별개)
- 첫 해 11일 + 1년차 15일 = 최대 26일 사용 가능 (근로기준법 §60②)

### 7.5 해외 법인 정책 차이 (예시)

- **중국**: 1년 미만 0일, 근속 기반 부여
- **베트남**: 일부 휴가 유형 무급 → 한국 시드 그대로 두면 유급으로 잘못 표기될 수 있음 (법무팀 검토 필요)
- **멕시코**: 전면 재검토 대기 중

## 8. 자주 묻는 질문 (FAQ)

**Q. 휴가 신청 후 취소하면 잔액이 즉시 복구되나요?**
A. 사용 전 취소는 자동으로 복구됩니다. 사용 중/후 취소는 사용일 만큼 차감된 후 나머지가 복구되며, 사용 후 취소는 인사담당자만 처리 가능합니다.

**Q. 반차를 두 번 쓰면 연차 1일이 차감되나요?**
A. 네. 0.5일 단위로 차감되며, 오전·오후 동일 날짜에 두 번 쓰면 1일 차감됩니다. 단, `allowHalfDay = false` 휴가 유형은 반차 불가입니다.

**Q. 경조사 휴가에 주말이 포함되면 어떻게 계산되나요?**
A. 대부분 경조사 휴가는 `calendar_day` + 공휴일 포함(통산) 정책입니다. 예: 부모 사망 5일을 목요일에 시작하면 토·일 포함 5일이 차감됩니다.

**Q. 연차 사용촉진 통보를 못 받았는데 미사용 잔액이 소멸되나요?**
A. 시스템이 60/30/10일 전 자동 통보합니다. 통보 이력이 `LeavePromotionLog`에 기록되며, 통보 후에도 사용 안 한 잔액만 법적으로 소멸 가능합니다.

**Q. 한국 본사와 중국 법인의 연차 일수가 다른데, 어디서 설정하나요?**
A. 휴가 유형은 회사별로 별도 정의됩니다. 인사담당자가 `/leave/admin` → 휴가 유형 관리 탭에서 회사를 선택하여 등록·수정합니다.

**Q. 마이너스 연차는 항상 허용되나요?**
A. 회사별 설정입니다. 인사담당자가 `LeaveSetting`에서 `allowNegativeBalance` 토글로 활성화하고 한도(`negativeBalanceLimit`)를 지정합니다. 기본값은 비활성화입니다.
