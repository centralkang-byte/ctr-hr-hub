> **UAT 가이드와의 차이**: UAT 가이드는 "어떻게 테스트하나", 본 매뉴얼은 "시스템이 어떻게 작동하나".

## 표준 목차 (각 모듈)

```
1. 개요 (무엇을 다루는가 + 도입 의도)
2. 카테고리/유형 정의
3. 핵심 정책
4. 워크플로 (State Machine)
5. 역할별 권한
6. 화면 안내
7. 특수 상황 처리
8. 자주 묻는 질문 (FAQ)
9. 알려진 제약
10. 부록: 기술 메모 (HR은 건너뛰기 OK)
    - 데이터 모델
    - 핵심 함수/파일
    - 관련 ADR 링크
```

비즈니스 정책 95% + 기술 메모 5% 비율. HR이 본문만 읽고, 개발/QA가 부록에서 코드 reference를 얻습니다.

## 작성 원칙

- **drift 방지**: 정책 변경 시 같은 PR에서 매뉴얼도 갱신
- **markdown 우선**: 추후 Notion/docx 변환은 별도 트랙으로 결정
- **모듈별 분리**: 한 번에 한 모듈씩, 1 모듈 = 1 PR
- **평이한 한국어**: 개발자 jargon 최소화 (HR 친화)
- **시각 자료**: 필요 시 `docs/uat/screenshots-v2/` 활용 또는 모듈별 신규 캡처
