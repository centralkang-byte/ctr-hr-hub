## 1. 개요

### 1.1 무엇을 다루는가

CTR HR Hub의 휴직 모듈은 다음을 통합 관리합니다.

- **휴직 유형 정의** — 회사별 사용 가능한 휴직 종류 (육아·출산·가족돌봄·질병·산재·병역·개인·학업 등)
- **신청·승인 흐름** — 직원 신청 → 승인 → 휴직 시작 → 복직 신청 → 복직 완료
- **재직 상태 자동 전환** — 휴직 시작 시 직원 상태를 휴직 중으로, 복직 시 정상으로 자동 전환
- **급여 자동 연동** — 휴직 유형별 급여 정책(무급/유급/일부 유급/보험 지급)에 따라 월 급여 자동 조정
- **복귀 알림** — 복직 예정일 7일/3일/1일 전 직원과 인사담당자에게 자동 통보
- **증빙 관리** — 진단서·출생증명서 등 첨부 파일 보관
- **분할 휴직** — 육아휴직 등 분할 사용이 허용되는 유형의 차수 관리

### 1.2 도입 의도

- **휴가(Leave)와 별개 영역** — 휴가는 수일 단위로 재직 상태 유지, 휴직은 수개월~수년 단위로 재직 상태 자체가 전환됨. 두 영역의 정책·워크플로·급여 처리가 근본적으로 달라 별도 모듈로 분리
- **법정 휴직 준수 자동화** — 한국 근로기준법·남녀고용평등법·산업재해보상보험법, 각국 노동법의 법정 휴직 정책을 시스템에서 자동 적용
- **다국적 법인 대응** — 13개 법인이 각국 노동법에 따라 휴직 유형·기간·급여를 차등 운영
- **급여 정합성** — 휴직 기간 동안의 일할 차감, 보험금 정산, 소급 정산을 급여 모듈과 자동 연동
- **복귀 보장 의무 추적** — 원직 복귀 보장 휴직(출산·육아·산재 등)의 복직 처리를 시스템에서 명확하게 관리

### 1.3 휴가(Leave)와의 차이

> 휴가 모듈은 별도 — [leave.md](leave.md) 참조

| 항목 | 휴가 | 휴직 |
|------|------|------|
| 기간 | 수일 | 수개월~수년 |
| 재직 상태 | 정상 유지 | "휴직 중"으로 전환 |
| 복귀 처리 | 종료일 자동 복귀 | 명시적 복직 신청 + 승인 필요 |
| 급여 | 단순 유급/무급 | 보험 지급, 일부 유급, 사업주 부담 등 복합 |
| 신청 주체 | 직원 본인 | 직원 본인 (인사담당자 대리 등록 가능) |
| 잔액 차감 | 휴가 잔액에서 차감 | 잔액 개념 없음 (유형별 기간 한도) |
| 법적 보호 | 일반적 | 법정 휴직은 거부 불가, 불이익 금지 |

## 2. 휴직 유형

휴직 유형은 **회사별로 별도 정의**됩니다. 한국 본사(CTR)는 한국 노동법 기반 8종, 해외 법인은 각국 노동법 기반 5~6종입니다. 총 13개 법인에 약 87건의 휴직 유형이 사전 등록되어 있습니다.

### 2.1 한국 법인 8종 (국내 7개 법인 공통)

| 유형 | 분류 | 최대 기간 | 급여 정책 | 분할 |
|------|------|----------|----------|------|
| 육아휴직 | 법정 | 1년 (365일) | 고용보험 (정부 지급) ~80% | 2회 |
| 출산전후휴가 | 법정 | 90일 | 사업주 60일 + 고용보험 30일 (혼합 100%) | 불가 |
| 가족돌봄휴직 | 법정 | 90일/년 | 무급 | 3회 |
| 질병휴직 | 사규 | 180일 | 무급 | 불가 |
| 업무상재해 휴직 | 법정 | 무제한 | 산재보험 ~70% | 불가 |
| 병역휴직 | 법정 | 548일 (18개월) | 무급 | 불가 |
| 개인사유 휴직 | 사규 | 365일 | 무급 | 불가 |
| 학업휴직 | 사규 | 730일 (2년) | 무급 | 불가 |

**적용 법인**: CTR-HOLD, CTR, CTR-MOB, CTR-ECO, CTR-ROB, CTR-ENR, CTR-FML (국내 7개 법인 공통 규정)

### 2.2 해외 법인별 휴직 유형

**중국 (CTR-CN, 쑤저우/장자강 — 장쑤성 규정)** — 6종:
- 출산휴가 158일 (국가 98일 + 장쑤성 60일), 배우자 출산휴가 15일, 육아휴직 연 10일(3세 미만), 병가 3~24개월, 산재 무제한, 개인사유

**베트남 (CTR-VN)** — 5종:
- 출산휴가 180일, 배우자 출산 5~14일, 병가 60일/년, 산재 무제한, 개인사유

**미국 (CTR-US)** — 5종:
- FMLA 12주/년 (50인+ 사업장 적용), 출산휴가 FMLA 내, USERRA 군복무 5년, 의료휴직, 개인사유

**러시아 (CTR-RU)** — 5종:
- 출산휴가 140일, 육아휴직 3년 (1.5년까지 보험 ~40% 지급), 병가 무제한, 군복무, 개인사유

**멕시코 (CTR-MX)** — 5종:
- 출산휴가 84일 (IMSS 100%), 배우자 출산 5일, 병가 364일, 산재 364일, 개인사유

**폴란드 (CTR-EU)** — 5종:
- 출산휴가 20주, 육아휴직 41주, 배우자 출산 14일, 병가 182일/년 (33일까지 사업주), 무급휴가

> **데이터 흐름 주의**: 해외 6개 법인 휴직 유형 시드는 마스터 적재 절차에 연결되어 있어 정상 운영 중입니다 (휴가 시드와 달리 LOA 시드는 `prisma/seed.ts`에 연결됨). 다만 미국 FMLA 50인+ 사업장 요건은 시스템에서 자동 검증하지 않으므로 인사담당자가 적용 가능성을 별도로 확인해야 합니다.

### 2.3 휴직 유형이 가지는 속성

| 속성 | 설명 |
|------|------|
| 분류 | 법정 / 사규 (각국 노동법으로 강제되는지 회사 자율인지) |
| 최대 기간 | 한 번에 사용 가능한 최대 일수 (`null` = 무제한, 산재 등) |
| 급여 정책 | 유급 / 무급 / 일부 유급 / 보험 지급 / 혼합 5종 |
| 급여 비율 | 일부 유급·보험 지급 시 적용 비율 (%) |
| 급여 재원 | 사업주 / 정부 / 보험 / 혼합 |
| 자격 요건 | 신청 가능한 최소 근속 개월 수 (없으면 입사 즉시 가능) |
| 근속기간 산입 | 휴직 기간이 근속연수에 포함되는지 |
| 연차 출근일 산입 | 휴직 기간이 다음 해 연차 부여 시 출근일수에 산입되는지 |
| 분할 사용 가능 | 한 휴직을 여러 차례 나눠 사용 가능 (육아 등) |
| 최대 분할 횟수 | 분할 가능 시 횟수 한도 |
| 증빙 필수 | 진단서·증명서 등 첨부 필수 여부 |
| 증빙 종류 | 어떤 증빙이 필요한지 안내 문구 |
| 사전 신청 일수 | 며칠 전까지 신청해야 하는지 (긴급 가능 시 `null`) |
| 원직 복귀 보장 | 복직 시 원래 직위 보장 의무 (법정 휴직은 일반적으로 보장) |

## 3. 핵심 정책

### 3.1 신청 자격 (Eligibility)

각 휴직 유형마다 다음 자격 요건이 적용됩니다.

| 휴직 유형 | 자격 요건 |
|----------|----------|
| 출산전후휴가 | 없음 (입사 즉시 가능) |
| 업무상재해 휴직 | 없음 (사고 발생 시 즉시) |
| 병역휴직 | 없음 (입영 통지 시 즉시) |
| 육아휴직 | 근속 6개월 이상 |
| 가족돌봄휴직 | 근속 6개월 이상 |
| 개인사유 휴직 | 근속 1년 이상 |
| 학업휴직 | 근속 2년 이상 |
| 질병휴직 | 회사 정책 (자격 요건 없음) |

법정 휴직은 자격 요건을 충족하는 경우 회사가 거부할 수 없으며, 자격을 충족 안 한 경우에도 사규 휴직으로 대체 신청 가능합니다.

### 3.2 기간 한도

각 휴직 유형마다 한 번에 사용 가능한 최대 기간이 정해져 있습니다.

- **유한 한도**: 육아 365일, 출산 90일, 가족돌봄 90일, 병역 548일 등 (한국 기준)
- **무제한**: 업무상재해 휴직 (산재 인정 기간만큼 사용, 무기한)
- **분할 가능 휴직의 한도**: 분할 차수마다 일정 기간 이상 사용 필요 (예: 가족돌봄휴직 1회 30일 이상 분할)

연장이 필요한 경우 별도 신청 (§7.6 참조).

### 3.3 급여 정책 5종

| 정책 | 의미 | 예시 |
|------|------|------|
| 유급 | 사업주가 100% 지급 | 중국 배우자 출산휴가 |
| 무급 | 휴직 기간 동안 급여 없음 | 가족돌봄·개인사유·학업 |
| 일부 유급 | 사업주가 일정 비율(%) 지급 | 중국 병가 60% |
| 보험 지급 | 고용보험·산재보험·사회보험에서 지급 (회사 부담 없음) | 한국 육아휴직 ~80%, 한국 산재 ~70% |
| 혼합 | 일정 기간 사업주 + 나머지 보험 | 한국 출산전후휴가 (60일 사업주 + 30일 고용보험) |

**급여 자동 정산**: 휴직이 활성화되면 시스템이 휴직 기간을 월별로 분리하여 각 월의 급여를 자동 조정합니다. 부분 휴직(월 중 시작·종료)도 일할 계산. 급여 처리 메커니즘은 [payroll.md §3.6](payroll.md) 참조.

### 3.4 근속·연차 산입

휴직이 근속연수와 다음 해 연차 부여에 어떻게 반영되는지는 유형별로 다릅니다.

| 유형 | 근속 산입 | 연차 출근일 산입 |
|------|:---------:|:--------------:|
| 육아휴직 | ✅ | ✅ |
| 출산전후휴가 | ✅ | ✅ |
| 가족돌봄휴직 | ✅ | ❌ |
| 업무상재해 휴직 | ✅ | ✅ |
| 병역휴직 | ✅ | ❌ |
| 질병휴직 | ❌ | ❌ |
| 개인사유 휴직 | ❌ | ❌ |
| 학업휴직 | ❌ | ❌ |

- **근속 산입**: 정기 호봉 인상·퇴직금 산정·승진 연한에 포함
- **연차 출근일 산입**: 다음 해 연차 부여 시 출근일로 인정 (산입 안 되면 그만큼 연차 비례 차감)

### 3.5 분할 사용

일부 휴직 유형은 한 번에 모두 사용하지 않고 차수로 나눠 사용할 수 있습니다.

| 유형 | 분할 가능 | 최대 차수 |
|------|:--------:|:--------:|
| 육아휴직 | ✅ | 2회 |
| 가족돌봄휴직 | ✅ | 3회 |
| 중국 육아휴직 | ✅ | 무제한 |
| 폴란드 육아휴직 | ✅ | 5회 |
| 그 외 | ❌ | — |

**분할 차수 관리**: 각 차수가 별도 휴직 레코드로 생성되며, 원본 레코드와 부모-자식 관계로 연결됩니다. 차수 번호(1차·2차·3차)가 시스템에서 자동 부여됩니다.

### 3.6 원직 복귀 보장

법정 휴직은 복직 시 원래 직위 또는 동등한 직위로의 복귀가 의무화되어 있습니다.

**원직 복귀 보장 휴직** (한국):
- 육아휴직 / 출산전후휴가 / 가족돌봄휴직 / 업무상재해 / 병역

**원직 복귀 미보장**:
- 질병휴직 / 개인사유 / 학업

복직 시 직원이 동등한 직위로 복귀하지만, 부서 개편 등으로 정확히 같은 직위 복귀가 어려우면 인사담당자가 복직 시 새 직위를 지정할 수 있습니다.

### 3.7 증빙 관리

- 증빙 필수 유형은 신청 시 첨부 파일(진단서·출생증명서·산재 인정서 등) 제출 필수
- 첨부는 시스템 파일 저장소에 보관되며 결재자·인사담당자만 열람 가능
- 증빙 누락 시 신청이 반려되거나 보류됨

## 4. 워크플로

### 4.1 휴직 라이프사이클 (6단계 상태)

```
[신청]            [HR/팀장]            [시스템]              [직원/HR]            [HR]
신청 ─────────→  승인 또는 반려
                       │
        ┌──── 반려 ────┘
        │              │
        │              └─→ 승인 ─────→ (시작일 도래) ─→ 휴직 활성화
        │                                                    │
        │                                                    ├─→ 직원 상태 = 휴직 중
        │                                                    └─→ 월 급여 자동 조정 시작
        │                                                              │
        │                                                              ↓
        │                                              [직원 또는 HR이 복직 신청]
        │                                                              │
        │                                                              ↓
        │                                                       복직 완료 처리
        │                                                              │
        │                                                              ├─→ 직원 상태 = 정상 (휴직 종료)
        │                                                              └─→ 복직 직위 발령
        │
        └──── (모든 단계에서) 취소 가능
```

**상태 7종**:

| 상태 | 의미 |
|------|------|
| 신청 | 직원이 신청서 제출, 승인 대기 중 |
| 승인 | 결재 완료, 시작일 도래 대기 |
| 활성 | 휴직 시작일 도래, 현재 휴직 중 |
| 복직 신청 | 직원 또는 인사담당자가 복직 의사 표시 |
| 완료 | 복직 처리 완료, 휴직 종료 |
| 취소 | 어느 단계에서든 취소된 경우 |
| 반려 | 신청 단계에서 반려 |

### 4.2 시스템 자동 처리

**시작 시 (승인 → 활성)**:
- 직원의 현재 발령(직위·부서·법인)의 종료일 = 휴직 시작일 전날
- 새 발령 생성: 상태 = 휴직 중, 같은 직위 유지
- 휴직 레코드에 발령 ID 연결 (복직 시 추적용)
- 급여 자동 조정 시작 (해당 월부터)

**복직 시 (복직 신청 → 완료)**:
- 휴직 중 발령의 종료일 = 실제 복직일
- 새 발령 생성: 상태 = 정상, 직위 = 복귀 직위 (지정 안 했으면 원직)
- 휴직 레코드에 복직 발령 ID 연결
- 급여 자동 조정 종료 (해당 월부터)

**취소 시**:
- 활성 상태에서 취소: 휴직 중 발령 삭제 + 원래 발령 복원 (롤백)
- 신청·승인 상태에서 취소: 발령 변경 없음, 단순 상태 변경

### 4.3 자동 알림

- **복직 예정일 D-7/D-3/D-1**: 직원과 해당 법인 인사담당자에게 자동 알림 발송 (시스템 자동, 매일 오전 9시 점검)
- **신청·승인·반려·취소 결과**: 직원에게 즉시 알림
- **휴직 시작 임박**: 직원에게 시작일 1일 전 알림 (시작일 도래 시 자동 활성화 안내)

## 6. 화면 안내

### 6.1 직원 — 본인 휴직 화면

위치: `프로필 → 휴직 탭` (직원 상세 페이지 내)

- **본인 휴직 이력**: 차수별 상태·기간·유형 조회
- **신규 신청 버튼**: 모달 폼 (유형 → 기간 → 사유 → 증빙 첨부 → 제출)
- **복직 신청**: 활성 휴직 카드에서 "복직 신청" 버튼

### 6.2 인사담당자 — 휴직 관리 화면

위치: `휴직 관리` 메뉴 (관리자 메뉴)

- **휴직자 목록**: 상태별 필터(신청/승인/활성/복직신청/완료/취소/반려), 회사별 필터
- **상세 모달**: 신청자 정보, 휴직 유형, 기간, 증빙, 결재 이력
- **승인/반려 액션**: 즉시 결재 가능
- **대리 등록**: 출산·산재 등 직원이 직접 신청 어려운 경우 인사담당자가 대신 등록 (자동 승인 경로)
- **복직 처리**: 복직 신청 건을 완료 처리 (직위 변경 옵션 포함)

### 6.3 휴직 유형 설정 (인사담당자)

위치: `설정 → 근태 → 휴직 유형 탭`

- 좌측: 회사 선택 (전사관리자는 13개, 인사담당자는 본인 회사 고정)
- 우측: 회사별 휴직 유형 목록
- **기본 유형 일괄 적용 버튼**: 한국 법인 8종, 중국 6종 등 사전 정의된 기본 유형을 일괄 적용 (Apply Defaults)
- **개별 등록·수정·삭제**: 유형별 14개 속성 편집

### 6.4 직원 상세 — 휴직 탭

위치: `인사관리 → 직원 상세 → 휴직 탭`

- 해당 직원의 모든 휴직 이력 시각화 (타임라인)
- 각 휴직의 상태·유형·기간·증빙·결재 이력 조회
- 인사담당자는 이 화면에서도 대리 등록·승인·복직 처리 가능

## 7. 특수 상황 처리

### 7.1 휴직 중 직원의 휴가

- 휴직 활성 상태 동안 신규 휴가 신청 불가
- 휴직 시작 전 승인된 휴가는 일반적으로 자동 취소 (시스템이 강제하지 않으므로 인사담당자가 확인 필요)
- 복직 후 휴가 잔액은 연차 출근일 산입 정책에 따라 변동 (§3.4 참조)

### 7.2 휴직 중 직원의 급여

- 시스템이 휴직 기간을 월별로 분리하여 각 월의 급여를 자동 조정
- 무급 휴직: 휴직 일수만큼 일할 차감
- 일부 유급: 비율(%)에 따라 일할 차감
- 보험 지급: 사업주 급여는 일할 차감, 정부·보험 지급은 시스템 외부 (회계/사회보험 담당)
- 부분 휴직(월 중 시작·종료): 평일 기준 일할 계산

### 7.3 휴직 중 직원의 성과평가

- 휴직 기간이 평가 기간을 일부 포함하는 경우 시스템이 자동 제외하지 않음
- 인사담당자가 평가 대상에서 수동 제외하거나, 평가 비율 조정 필요
- **추후 개선** 항목: 평가 모듈과 휴직 자동 연계 (§9 #3 참조)

### 7.4 복직 시 직위 변경

- 원직 복귀 보장 유형 (육아·출산·가족돌봄 등)이 기본값
- 부서 개편·직위 폐지 등으로 원직 복귀 어려운 경우 인사담당자가 복직 처리 시 새 직위 지정
- 원직 복귀 미보장 유형 (질병·개인사유·학업)도 회사 정책으로 원직 복귀 가능

### 7.5 분할 휴직 중간의 휴가·휴직

- 분할 휴직의 1차 종료 후 2차 시작 전 기간은 정상 재직 상태
- 이 기간 동안 휴가 신청·다른 휴직 신청 가능
- 2차 휴직 시작 시점에 시스템이 다시 휴직 활성화

### 7.6 휴직 연장

- 현재 시스템은 휴직 기간 연장을 별도 액션으로 제공하지 않음
- **운영 방법**: 활성 휴직을 복직 신청 → 완료로 한 차례 종료한 후 새 휴직 신청
- 분할 가능한 유형(육아 2회 등)은 차수로 자연스럽게 연장 가능
- **추후 개선** 항목: 휴직 기간 직접 연장 기능 (§9 #4 참조)

### 7.7 휴직 중 퇴사

- 휴직 중 직원이 퇴사를 결정하는 경우 인사담당자가 휴직 취소 처리 → 퇴직 절차 시작
- 휴직 기간 동안 발생한 미정산 보험금 등은 회계/사회보험 담당이 별도 정산

### 7.8 휴직 중 법인 변경

- 시스템은 휴직 중 직원의 법인 변경을 자동 처리하지 않음
- 운영 방법: 현재 휴직 취소 → 법인 이동 발령 → 새 법인에서 휴직 재신청 (회사 정책 일관성 위해 흔치 않음)

### 7.9 해외 법인 특수 요건

- **미국 FMLA**: 50인+ 사업장 적용 요건은 시스템이 자동 검증하지 않음. 인사담당자가 적용 가능성을 별도 확인
- **중국 出產假**: 장쑤성 기준 158일 시드. 다른 성으로 사업 확장 시 별도 시드 필요
- **러시아 육아휴직 3년**: 1.5년까지만 보험 ~40% 지급. 시스템은 기간만 추적하고 급여는 회계 담당이 처리

## 8. 자주 묻는 질문

**Q. 휴직과 휴가는 어떻게 다른가요?**
A. 휴가는 수일 단위로 재직 상태가 유지되며 휴가 잔액에서 차감됩니다. 휴직은 수개월~수년 단위로 직원 상태가 휴직 중으로 전환되며 잔액 개념 없이 유형별 기간 한도로 관리됩니다. 두 영역은 시스템에서 완전히 분리된 별도 모듈입니다.

**Q. 휴직 신청 시 어떤 증빙이 필요한가요?**
A. 휴직 유형별로 다릅니다. 육아휴직은 자녀 출생증명서, 출산휴가는 의사 진단서, 산재는 산재 인정 결정서, 병가는 의사 진단서, 병역은 입영통지서 등이 필요합니다. 신청 화면에서 유형 선택 시 필요 증빙이 안내됩니다.

**Q. 휴직 중 급여는 어떻게 되나요?**
A. 유형별로 다릅니다. 육아휴직은 고용보험에서 ~80% 지급, 출산휴가는 60일 사업주 + 30일 고용보험 (100%), 산재는 산재보험 ~70%, 가족돌봄·개인사유·학업·병역은 무급, 중국 병가는 60% 일부 유급 등. 시스템이 휴직 활성화 시점부터 월 급여를 자동 조정합니다.

**Q. 휴직이 근속연수에 포함되나요?**
A. 휴직 유형별로 다릅니다. 육아·출산·가족돌봄·산재·병역은 근속에 산입되어 호봉·승진·퇴직금에 반영됩니다. 질병·개인사유·학업 휴직은 근속 산입되지 않습니다 (§3.4 표 참조).

**Q. 육아휴직을 두 번에 나눠 쓸 수 있나요?**
A. 네. 한국 육아휴직은 2회까지 분할 사용 가능합니다. 중국은 무제한 분할, 폴란드는 5회까지 가능. 각 차수가 별도 휴직 레코드로 관리되며 차수 번호가 자동 부여됩니다.

**Q. 휴직 시작일이 지났는데 휴직 상태로 안 바뀌는 경우는?**
A. 시스템이 시작일 도래 시 자동 활성화합니다. 활성화가 안 되었으면 인사담당자에게 문의하세요. 수동으로 활성 처리 가능합니다.

**Q. 복직 예정일 전에 복직하고 싶은 경우는?**
A. 직원이 본인 휴직 카드에서 "복직 신청" 버튼을 누르고 실제 복직일을 입력. 인사담당자가 복직 완료 처리하면 그 날짜에 복직 발령이 생성됩니다.

**Q. 복직 시 원래 자리가 없어졌다면?**
A. 부서 개편 등으로 원직 복귀 어려운 경우 인사담당자가 복직 처리 시 새 직위를 지정할 수 있습니다. 단, 법정 휴직(육아·출산·가족돌봄·산재 등)의 원직 복귀 의무는 회사가 부담하므로 가능한 동등 직위로 배치해야 합니다.

**Q. 휴직 기간을 연장할 수 있나요?**
A. 현재 시스템은 별도 연장 기능을 제공하지 않습니다. 활성 휴직을 복직으로 한 차례 종료한 후 새 휴직을 신청해야 합니다. 분할 가능한 유형(육아 2회 등)은 차수로 자연스럽게 연장됩니다. 직접 연장 기능은 **추후 개선** 항목입니다.

**Q. 휴직 중 평가 시즌이 오면 어떻게 되나요?**
A. 시스템이 자동으로 평가 대상에서 제외하지 않습니다. 인사담당자가 평가 대상에서 수동 제외하거나 평가 비율을 조정해야 합니다. **추후 개선** 항목입니다 (§9 #3).

**Q. 미국 FMLA는 어떤 직원이 신청 가능한가요?**
A. 미국 연방법 FMLA는 50인+ 사업장에서 12개월 이상 근무한 직원이 대상입니다. 시스템은 자격 요건의 근속(12개월)은 검증하지만 사업장 규모(50인+) 요건은 자동 검증하지 않습니다. CTR-US 인사담당자가 적용 가능성을 별도 확인해야 합니다.

| 4 | 휴직 기간 직접 연장 기능 부재 | 연장하려면 복직 → 재신청 두 단계 필요. 직접 연장 액션 없음. **추후 개선** — 활성 휴직의 expected_end_date 갱신 액션 추가 |
| 5 | 미국 FMLA 사업장 규모 자동 검증 부재 | 50인+ 사업장 요건을 시스템이 검증하지 않음. CTR-US 인사담당자가 직접 확인. **추후 개선** — 회사별 사업장 규모 입력 + 자격 검증 추가 |
| 6 | 휴직 중 휴가 자동 취소 부재 | 휴직 시작 전 승인된 휴가가 휴직 기간에 걸리는 경우 자동 취소 안 함. 인사담당자 수동 확인 필요. **추후 개선** — 휴직 활성화 시 충돌 휴가 자동 처리 |
| 7 | 휴직 중 법인 이동 미지원 | 시스템 워크플로 부재. 현재 운영: 휴직 취소 → 이동 → 재신청. 흔치 않은 케이스로 우선순위 낮음 |
| 8 | 결재 흐름 설정의 휴직 모듈 연동 미흡 | 결재 매뉴얼 §5.2와 동일 문제 — 휴직 모듈도 Settings UI에서 등록한 다단계 흐름을 완전히 따르지 않음. **추후 개선** — 결재 모듈 통합 정합화 시 일괄 처리 |
| 9 | 출산 후 휴가↔육아휴직 자동 연계 부재 | 출산휴가 90일 종료 후 육아휴직으로 자동 이관되지 않음. 직원이 별도 신청 필요. **추후 개선** — 두 휴직 유형 간 자동 연결 기능 |
| 10 | 보험 지급분 정산 외부 처리 | 고용보험·산재보험 등 정부·보험 지급분은 시스템에서 추적만 하고 실제 정산은 회계/사회보험 담당 외부 처리. 데이터 일치 확인을 위한 reconciliation 보고서 부재. **추후 개선** — 정부 지급액 입력·대조 기능 |

## 10. 추후 개선

**기능 개발 트랙 (Phase 2)**:
1. 휴직 대시보드 KPI (현재 휴직자 수, 유형별 분포, 복귀 예정 캘린더) (§9 #1)

**자격 검증**:
8. 미국 FMLA 50인+ 사업장 요건 자동 검증 — 회사별 사업장 규모 입력 + 신청 시 검증 (§9 #5)

**결재 흐름**:
9. 결재 흐름 설정의 휴직 모듈 완전 연동 (결재 매뉴얼 §10 #3·#5와 통합 처리) (§9 #8)

> 본문이 정책 설명, 이 섹션은 개발/QA 참고용. HR은 건너뛰어도 무방합니다.

### 11.1 데이터 모델

핵심 모델 (모두 `prisma/schema.prisma`):

- `LeaveOfAbsenceType` — 회사별 휴직 유형 정의 (87건 시드)
- `LeaveOfAbsence` — 휴직 기록 (6-state 워크플로)
- `EmployeeAssignment` — 휴직 시작·종료 시 자동 생성되는 발령 (status: `ON_LEAVE` ↔ `ACTIVE`)
- `PayrollAdjustment` — 휴직 기간 급여 조정 (`LeaveOfAbsence` 외래키 보유)

**상태 7종 (state machine 문자열)**: `REQUESTED`, `APPROVED`, `ACTIVE`, `RETURN_REQUESTED`, `COMPLETED`, `CANCELLED`, `REJECTED`

**상태 전이 (src/app/api/v1/leave-of-absence/[id]/route.ts:20-23 STATE_TRANSITIONS)**:
```
REQUESTED       → APPROVED, REJECTED, CANCELLED
APPROVED        → ACTIVE, CANCELLED
ACTIVE          → RETURN_REQUESTED, CANCELLED
RETURN_REQUESTED → COMPLETED, CANCELLED
```

### 11.5 알려진 위험 코드 경로

- `[id]/route.ts` activate 트랜잭션 — Assignment STATUS_CHANGE 생성 + PayrollAdjustment 시드 + LOA 상태 변경이 한 트랜잭션. 트랜잭션 실패 시 부분 적용 위험 → 트랜잭션 격리 수준 확인 필요
- `complete` 액션의 Assignment 복원 — 휴직 중 발령(ON_LEAVE)을 종료하고 새 ACTIVE 발령 생성. 직원의 다른 발령(겸직 등)과의 정합성은 별도 검증 없음
- `cancel` 액션의 활성 상태 롤백 — 휴직 중 발령 삭제 + 원래 발령 복원. 휴직 기간 중 다른 발령 변경이 있었으면 충돌 가능 (현재 발생 안 했지만 향후 리스크)
- 분할 휴직의 `parentId` 참조 무결성 — 부모 휴직 삭제 시 자식 휴직의 `parentId`가 dangling 가능. 현재 코드는 deletedAt soft delete로 우회 중
- `payroll-adjustment.ts`의 월별 분리 로직 — 휴직 기간이 월 경계와 정확히 일치하지 않을 때 일할 계산 정확성. 평일/공휴일 처리는 별도 함수(`getWeekdaysInMonth`) 의존
- 분할 차수 자동 부여 — `splitSequence` 필드는 신청 시 자동 부여되지 않고 호출자가 명시. 누락 시 기본값 1로 모든 분할이 1차로 잡힘 (향후 자동화 권장)
