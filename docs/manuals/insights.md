# 인사이트·분석 매뉴얼 (Insights & Analytics)

> **최종 갱신**: 2026-05-12
> **담당**: 인사관리팀 · 임원
> **상태**: Draft v1 (CEO 리뷰 전)

---

## 1. 개요

### 1.1 무엇을 다루는가

CTR HR Hub의 인사이트·분석 모듈은 다음을 통합 관리합니다.

- **인력 분석 (Workforce)** — 인원·성별·연령·근속 등 인구통계 KPI
- **이직률·이직 위험 (Attrition·Turnover)** — 이직률 추이 + AI 기반 직원별 이직 위험도 예측
- **출퇴근 분석** — 시간외근무·52시간 위반·결근 추이
- **급여·보상 분석** — 인건비 추이·compa-ratio·예산 대비 사용률
- **성별 임금 격차 (Gender Pay Gap)** — 직급·직군별 성별 급여 차이 분석 + 보고서 export
- **성과 분포** — 9-Block·등급별 분포·캘리브레이션 결과
- **채용 분석** — 출처별 비용·소요일·합격률
- **팀 건강도 (Team Health)** — 팀별 건강도 점수·추이
- **번아웃 위험** — 시간외근무·휴가 미사용 등 종합 지표
- **예측 분석 (Predictive)** — AI 기반 미래 예측 (이직·인건비·성과)
- **AI 보고서** — 월간·분기 AI 자동 생성 인사 리포트
- **KPI 대시보드** — 사용자 정의 KPI 대시보드

### 1.2 도입 의도

- **데이터 기반 의사결정** — 매니저·임원이 직관 대신 데이터로 인사 결정 (이직·보상·인력 계획)
- **사전 예방** — 이직 위험·번아웃을 사후 대응이 아닌 사전 예측·대응
- **법규 준수** — 성별 임금 격차 등 ESG·DEI 보고서를 자동 생성하여 외부 공시 대응
- **다국적 통합 뷰** — 13개 법인 KPI를 본사 통화로 환산한 통합 시각화
- **AI 자동 인사이트** — 매월·분기 AI가 핵심 트렌드·이상 자동 요약 → 임원 리포트
- **롤업 가능 데이터 모델** — 직원 단위 데이터를 부서·법인·전사 단위로 자유 롤업

### 1.3 모든 모듈과의 관계

본 매뉴얼은 **모든 모듈의 데이터를 소비**합니다. 직접 데이터를 생성하지 않고 다른 모듈의 SSOT를 읽어 시각화·분석:

| 분석 영역 | 데이터 소스 매뉴얼 |
|----------|------------------|
| 인력 분석 | 인사관리 (직원·발령) |
| 이직률·위험 | 인사관리 + 온보딩(퇴직 사유) + 성과 + 출퇴근 |
| 출퇴근 분석 | 출퇴근 |
| 급여·보상 분석 | 급여 + 보상 |
| 성별 임금 격차 | 인사관리(성별) + 보상(기본급) |
| 성과 분포 | 성과 (9-Block·등급) |
| 채용 분석 | 채용 (공고·지원·비용) |
| 팀 건강도 | 성과 (1:1·체크인·펄스) + 출퇴근(52시간) + 이직 위험 |
| 번아웃 위험 | 출퇴근 + 휴가(미사용 잔액) |
| AI 보고서 | 위 모든 영역 종합 |
| 분석 설정 (`AnalyticsConfig` / `KpiDashboardConfig`) | [settings.md](settings.md) §11.1 |

## 2. 분석 카테고리

### 2.1 인력 (Workforce)

- **헤드카운트**: 전체 인원·법인별·부서별·직급별 분포
- **성별·연령·국적**: 인구통계 분석
- **근속 분포**: 신규 입사·5년 이상·10년 이상 비율
- **고용형태**: 정규/계약/파견/인턴 비율
- **신규 입사 vs 퇴직**: 월간 입사·퇴직 추이 + 순증감

### 2.2 이직·유지 (Attrition·Retention)

- **이직률**: 월간·연간 회사·부서별 (자발 vs 비자발 분리)
- **이직 사유 분포**: 6종 사유(보상·커리어·워라밸·매니지먼트·문화·이주·개인)별 (온보딩 §2.8)
- **직원별 이직 위험도**: AI 기반 0~100점 (출퇴근·휴가·성과·1:1 빈도 등 종합)
- **고위험 직원 목록**: 인사담당자가 사전 대응 가능
- **AttritionRiskHistory**: 위험도 추이 시계열

### 2.3 출퇴근 (Attendance Analytics)

- **시간외근무 추이**: 부서·직급별 평균 시간외근무
- **52시간 위반·임계 도달**: 알림 발생 빈도
- **결근률**: 결근·지각·조퇴 추이
- **휴가 사용률**: 부서별 연차 사용 비율

### 2.4 급여·보상 (Payroll·Compensation)

- **월 인건비 추이**: 회사·부서별 (통화 환산 본사 합산)
- **평균 임금**: 직급·직군·성별 평균
- **Compa-ratio 분포**: 직급별 시장 대비 비율 (보상 §2.2)
- **인건비 예산 대비 사용률**: 월·분기·연간
- **연봉 인상률 분포**: 사이클별

### 2.5 성별 임금 격차 (Gender Pay Gap)

- **전체 격차**: 남녀 평균 임금 비율 (Adjusted vs Unadjusted)
- **직급별 격차**: 같은 직급 내 성별 비교
- **직군별 격차**: 직군 내 성별 비교
- **보고서 export**: ESG·DEI 외부 공시용 PDF·Excel

### 2.6 성과 (Performance Analytics)

- **9-Block 분포**: 전사·부서·법인별 (성과 §2.4)
- **등급별 분포**: O/E/M/S 비율 (성과 §2.3)
- **캘리브레이션 효과**: 원래 등급 vs 최종 등급 차이
- **매니저별 평가 편향**: 매니저가 등급을 너무 후하게/박하게 주는지

### 2.7 채용 (Recruitment Analytics)

- **소요일 (Time-to-Fill)**: 공고 발행 ~ 입사일 평균
- **단계별 전환율**: 지원 → 서류 → 1차 → 2차 → 최종 → 합격
- **출처별 효율**: 직접·추천·에이전시·채용사이트별 합격률·비용
- **공고당 평균 비용** (보상 §3.10)

### 2.8 팀 건강도 (Team Health)

`TeamHealthScore` 모델 기반 종합 점수.

**구성 요소**:
- 펄스 응답 (성과 §3.11)
- 1:1 빈도 (성과 §2.7)
- 시간외근무 (출퇴근)
- 휴가 사용률
- 이직률 (해당 팀)
- 평가 등급 분포

점수: 0~100, 색상 코딩 (녹색·노랑·빨강).

### 2.9 번아웃 위험 (Burnout)

다음 지표 종합:
- 누적 시간외근무 (3개월·6개월)
- 미사용 연차 잔액 (잔액 정의는 [leave.md §3.5](leave.md) 참조)
- 펄스 응답 (기분·에너지·소속감)
- 1:1 부재 기간

고위험 직원·팀에 알림.

### 2.10 예측 분석 (Predictive)

AI 기반:
- 이직 위험 예측 (개인·팀)
- 인건비 예측 (다음 분기·연간)
- 시간외근무 예측 (계절성·프로젝트 영향)
- 인력 부족 예측 (입사·퇴직 패턴)

### 2.11 AI 보고서 (AiReport)

월간·분기 자동 생성:
- **포함 내용**: 핵심 KPI 요약 + 이상 트렌드 + 권고 사항
- **수신자**: 임원·인사담당자
- **생성 방식**: AI가 전 모듈 데이터 분석 → 자연어 보고서
- **상태**: 생성 중 / 완료 / 실패

### 2.12 KPI 대시보드 (KpiDashboardConfig)

회사·역할별 사용자 정의 대시보드:
- 카드 위젯 자유 배치 (인력·이직·인건비·성과 등)
- 본인 화면 또는 전사 공유 옵션

## 3. 핵심 정책

### 3.1 데이터 새로고침 주기

| 분석 | 주기 |
|------|------|
| 실시간 (사이드바 뱃지 등) | 즉시 |
| 일별/월별 KPI 갱신 | **HR_ADMIN 수동 트리거** (`/api/v1/analytics/calculate`). 분석 페이지 진입 시 실시간 집계도 동작. 자동 cron 미등록 — §9·§10 참조 |
| AI 위험도 (이직·번아웃) | 관리자가 `/api/v1/attrition/recalculate` 수시 호출. 자동 cron 미등록 — §9·§10 참조 |
| AI 보고서 | 인사담당자 수동 생성 (월간·분기 자동 cron 미구현) — §9·§10 참조 |
| 분석 스냅샷 (AnalyticsSnapshot) | `analytics/calculate` 수동 호출 시점에 upsert (자동 스케줄 없음) |

수동 새로고침: 인사담당자가 인사이트 각 화면의 새로고침 버튼.

### 3.2 데이터 마스킹 정책

분석 데이터에도 권한별 마스킹 적용:

- **직원**: 본인 데이터만 + 익명 회사 통계
- **팀장**: 직속 부하 단위 + 익명 부서 통계
- **인사담당자**: 본인 법인 전체 (실명 포함)
- **전사관리자**: 13개 법인 전체
- **익명 통계**: 퇴직 면담·펄스 응답 등 자유 응답 데이터는 응답 건수 5건 미만 시 자동 익명화(예: 부서별 표시 차단). 일반 인구통계·인건비 분석에는 동일한 임계값이 적용되지 않으므로, 작은 부서의 인원·인건비는 다른 데이터와 함께 노출될 수 있음 (§9 #14 참조)
- **성과 결과**: 캘리브레이션 완료 + 공개 시점 이후만 노출 (성과 §3.8)
- **보상 분석**: 팀장의 팀원 보상 조회는 회사 정책 차등 (보상 §5.3)

### 3.3 13개 법인 통합 뷰

본사 KPI는 13개 법인 데이터를 통화 환산하여 합산:

- **환율 적용**: ExchangeRate 마스터의 월별 환율 (급여 §3.11)
- **본사 환산 통화**: KRW (기본)
- **법인별 화면**: 현지 통화 그대로 표시
- **누락 환율**: 누락 월 fallback 없음 → 분석 실패 가능 (운영 시 환율 적재 확인 필수)

### 3.4 AI 이직 위험도 (Attrition Risk)

다음 신호를 가중합:

| 신호 | 가중치 (예시) |
|------|--------------|
| 연속 1:1 부재 | +20 |
| 펄스 응답 부정 트렌드 | +25 |
| 시간외근무 누적 (3개월) | +15 |
| 휴가 미사용 잔액 과다 | +10 |
| 평가 등급 하락 | +15 |
| 외부 채용 사이트 활동 (선택, 외부 데이터) | +25 |

0~100점, 70+ 고위험. AttritionRiskHistory에 주별 기록.

### 3.5 성별 임금 격차 계산

**Unadjusted Pay Gap** (단순 격차):
```
(남성 평균 임금 - 여성 평균 임금) / 남성 평균 임금 × 100%
```

**Adjusted Pay Gap** (직급·직군 등 통제 변수 보정):
```
같은 직급·직군 내 남성·여성 평균 임금 차이의 가중평균
```

ESG·DEI 보고서: 두 지표 모두 산출 + 부서·직급별 세부.

### 3.6 AI 보고서 자동 생성

- **생성 방식**: 인사담당자가 `보고서 생성` 버튼 클릭 시 생성 (월간·분기 자동 cron 미구현 — §9·§10 참조)
- **AI 모델**: Claude (Anthropic) 사용 — 데이터 분석 + 자연어 보고서
- **검토**: 인사담당자가 보고서 검토·수정 후 임원에게 전달

### 3.7 KPI 대시보드 개인화

각 사용자 또는 역할별 대시보드 구성:
- 위젯 라이브러리 (인력·이직·인건비·성과 등 30+ 카드)
- 드래그앤드롭으로 배치
- 본인 전용 또는 회사·역할에 공유 옵션
- 회사 정책으로 강제 대시보드 (직원에게 일관된 KPI 제공) 가능

### 3.8 외부 export

- **PDF**: ESG·DEI 보고서, AI 보고서, 임원 리포트
- **Excel**: 단계별 데이터 (직원 목록·이직 사유·인건비 명세 등)
- **CSV**: 일괄 데이터 다운로드 (BI 도구 임포트용)
- **외부 BI 연동**: PowerBI·Tableau·Looker 등은 API 토큰으로 직접 연결 (현재 부분 구현)

### 3.9 자동 알림

- **AI 보고서 생성 완료**: 인사담당자·임원 알림
- **고위험 직원 발견**: 인사담당자 알림 (이직 위험 70+ 신규)
- **번아웃 위험**: 매니저·인사담당자 알림 (해당 팀)
- **펄스 응답률 저조**: 인사담당자 알림 (성과 §7.10)
- **52시간 위반 추이 악화**: 인사담당자 알림

## 4. 워크플로

### 4.1 일간 KPI 갱신 라이프사이클

```
[관리자 분석 계산 트리거]
       │
       ↓
[전 모듈 데이터 읽기] — 인사·출퇴근·휴가·급여·성과 등
       │
       ↓
[KPI 계산] — 헤드카운트·이직률·시간외 등
       │
       ↓
[AnalyticsSnapshot upsert] — 일별 스냅샷
       │
       ↓
[대시보드 갱신] — 캐시 무효화 + 새 값 적용
       │
       ↓
[이상 감지] — 임계값 초과 시 알림 발송
```

### 4.2 AI 위험도 갱신 라이프사이클 (주간)

```
[관리자 재계산 실행 시점]
       │
       ↓
[전 직원 신호 집계] — 1:1·펄스·시간외·휴가·평가·이직 등
       │
       ↓
[AI 위험도 계산] — 가중합 0~100점
       │
       ↓
[AttritionRiskHistory 저장]
       │
       ↓
[고위험 직원 알림] — 70+ 신규 등장 시 인사담당자
```

### 4.3 AI 보고서 자동 생성

```
[인사담당자 보고서 생성 클릭 시점]
       │
       ↓
[전 모듈 데이터 수집] — 인력·이직·인건비·성과·채용 등
       │
       ↓
[AI 분석 + 자연어 생성] — Claude API
       │
       ↓
[AiReport 저장] — status: GENERATED
       │
       ↓
[인사담당자 검토] — 수정·보강 옵션
       │
       ↓
[임원에게 전달] — 이메일 + 시스템 내 열람
```

### 4.4 자동 알림

위 모든 흐름 + 새로고침 결과 + 이상 감지

- **개별 직원 위험도**: 인사담당자·임원만, 매니저는 본인 부하 한정
- **성과 결과**: 캘리브레이션 완료 + FINALIZED 이후만 통계 노출
- **보상 분석**: 회사 정책에 따라 팀장 권한 차등 (보상 §5.3)
- **펄스 응답 익명성**: 부서별 최소 응답 수 충족 시에만 분석 결과 노출 (성과 §3.11)

> **데이터 흐름 주의**: 분석 화면이 마스킹 정책을 우회하면 보안 사고. 모든 분석 API는 마스킹 정책을 적용해야 하며, BI 외부 연동 시에도 같은 정책 적용 (역할 기반 토큰).

## 7. 특수 상황 처리

### 7.1 데이터 누락 시 분석

- 일부 모듈 데이터 누락(예: 펄스 미운영) 시 해당 영역만 "데이터 없음" 표시
- 다른 영역은 정상 분석 진행
- 인사담당자에게 누락 알림

### 7.2 환율 누락 법인

- 특정 월의 환율 누락 시 본사 환산 합산 불가
- 해당 법인은 분석에서 제외 + 경고 표시
- 인사담당자가 환율 등록하면 재계산

### 7.3 신규 입사자·퇴직자 일할 처리

- 인원 카운트 시 입사·퇴직 시점에 따라 일할 가중치 적용 가능 (회사 정책)
- 단순 카운트는 월말 기준
- 분석 정확도: 정밀 일할 vs 단순 카운트 선택

### 7.4 익명 통계 임계값 부족

- 퇴직 면담·펄스 응답 등 자유 응답 데이터는 응답 건수 5건 미만 시 자동 익명화 (개인 식별 방지)
- 해당 임계값 미달 시 부서별 결과 노출 안 함 — 회사·전사 단위 합산은 노출
- 일반 인구통계·인건비 분석에는 동일한 임계값이 적용되지 않으므로 작은 부서 노출 가능 (§9 #14)

### 7.5 AI 보고서 생성 실패

- API 장애·데이터 부족으로 AiReport.status = FAILED
- 인사담당자에게 알림 + 재시도 옵션
- 수동 작성 fallback 가능

### 7.6 분석 데이터 vs 운영 데이터 불일치

- 분석 스냅샷(AnalyticsSnapshot)은 매월 1일 시점 데이터
- 운영 데이터는 실시간 변화
- 분석 화면은 스냅샷 + 실시간 혼합 → 차이 발생 시 인사담당자 확인

### 7.7 외부 BI 도구 연동 보안

- API 토큰은 전사관리자만 발급
- 토큰별 권한·범위 제한 (예: 본인 법인 인력만, 보상 제외 등)
- 정기 토큰 갱신 (90일)
- **추후 개선** 항목 — 토큰 자동 만료·갱신·감사

### 7.8 캐싱·실시간 균형

- 대부분 KPI는 캐싱(30s~1800s) 적용 (성능)
- 실시간 필요한 경우(사이드바 결재 뱃지) 캐싱 미적용
- 새로고침 버튼으로 캐시 무효화

### 7.9 매니저 편향 검출 (Calibration)

- 매니저별 평균 등급 + 분포 분석
- 다른 매니저보다 너무 후함·박함 자동 검출
- 인사담당자에게 인사이트 제공 (캘리브레이션 검토 자료)
- **추후 개선** — 매니저 편향 자동 알림 (성과 §9 #6 AI 편향 차단과 통합)

### 7.10 ESG·DEI 보고서 신뢰성

- 성별 임금 격차 보고서는 외부 공시되므로 정확성 중요
- 인사담당자·법무 이중 검토 필수
- 데이터 정정 시 보고서 버전 관리 + 이력 보존

## 8. 자주 묻는 질문

**Q. 우리 부서의 분석은 누가 볼 수 있나요?**
A. 팀장(본인 직속 부하 단위), 인사담당자(본인 법인), 임원(법인 전체)이 볼 수 있습니다. 일반 직원은 익명 통계만 볼 수 있습니다.

**Q. AI 이직 위험도가 정확한가요?**
A. AI 모델은 1:1 빈도·펄스 응답·시간외근무·휴가·평가 등 여러 신호를 종합하여 위험도를 산출합니다. 절대적이지 않으며 보조 도구로 활용하세요. 70+ 고위험으로 표시된 직원은 매니저·인사담당자가 1:1로 직접 확인하는 것이 좋습니다.

**Q. 성별 임금 격차 보고서는 어디서 받나요?**
A. 성별 임금 격차 화면(직접 경로 `/analytics/gender-pay-gap` — 사이드바 미노출)에서 PDF·Excel로 export 가능합니다. ESG·DEI 공시 자료로 직접 사용하기 전 인사담당자·법무 검토 권장합니다.

**Q. 데이터가 매일 자동 갱신되나요?**
A. 현재 일별/월별 KPI는 HR_ADMIN이 `분석 계산` 액션을 실행할 때 갱신됩니다. 자동 cron은 미등록 상태 (§9 참조). 즉시 갱신이 필요하면 `새로고침` 버튼 사용.

**Q. KPI 대시보드를 내 마음대로 구성할 수 있나요?**
A. 네. `인사이트 → Executive Summary`(`/dashboard`)에서 위젯을 드래그앤드롭으로 배치할 수 있습니다. 본인 전용 또는 회사·역할에 공유할 수 있습니다.

**Q. 작은 부서는 분석이 안 보여요. 왜죠?**
A. 퇴직 면담·펄스 응답 같은 자유 응답 데이터는 응답 건수 5건 미만 시 자동 익명화됩니다(개인 식별 방지). 회사·전사 단위 합산 통계는 확인 가능합니다. 참고로 일반 인력·인건비 분석은 이 임계값이 적용되지 않으므로 작은 부서도 수치가 함께 노출될 수 있습니다 (§9 #14, 추후 마스킹 확대 예정).

**Q. 매니저가 평가를 너무 후하게 주는지 알 수 있나요?**
A. 캘리브레이션 분석에서 매니저별 평균 등급·분포를 비교할 수 있습니다. 자동 편향 알림은 추후 개선 항목입니다.

**Q. AI 보고서를 임원에게 자동 발송하나요?**
A. AI가 자동 생성하지만 발송은 인사담당자가 검토 후 수동 또는 자동 옵션을 선택합니다. 회사 정책에 따라 다릅니다.

**Q. 분석 데이터를 BI 도구(Tableau·PowerBI)로 가져올 수 있나요?**
A. 전사관리자가 API 토큰을 발급하면 가능합니다. 다만 권한·범위 제한이 있고 보안상 토큰 정기 갱신이 필요합니다.

**Q. 환율이 바뀌면 과거 분석도 다시 계산되나요?**
A. 과거 분석 스냅샷은 당시 환율로 보존됩니다. 환율 변경은 새 분석에만 적용. 과거 데이터 재계산이 필요한 경우 인사담당자가 수동 트리거합니다.

**Q. 펄스 응답이 익명이라는데 분석에 어떻게 활용되나요?**
A. 응답자 신원은 저장되지 않습니다. 부서·직급 등 메타데이터만 집계되어 팀 건강도·번아웃 위험 지표에 반영됩니다. 부서별 최소 응답 수 충족 시에만 결과를 분석에 활용합니다.

**Q. AI 위험도 점수에 본인이 동의 안 하면 어떻게 하나요?**
A. AI 위험도는 보조 지표일 뿐입니다. 직원이 본인 위험도 점수를 조회하지는 못하며, 매니저·인사담당자가 직접 1:1로 확인합니다. 데이터 처리에 대한 추가 우려는 인사담당자에게 문의하세요.

| 11 | 분석 데이터 export 대량 처리 단순화 | 대량 export(전사 직원 명세 등) 시 비동기 처리·이메일 전송 부재. UI에서 다운로드만. **추후 개선** — 비동기 + 이메일 발송 |
| 12 | AI 보고서 다양화 부족 | 월간·분기 보고서만 자동 생성. 주간·연간·특별 시점 보고서 부분. **추후 개선** — 사용자 정의 보고서 |
| 13 | 마스킹 정책 BI 외부 연동에서 보장 부분 | API 토큰별 권한 적용은 되나 마스킹 정책 BI 도구 측 보장은 부분. **추후 개선** — Row-level Security 확대 |
| 14 | 인력·인건비 분석에 5건 미만 마스킹 미적용 | 현재 자동 익명화는 퇴직 면담·펄스 응답 등 자유 응답 데이터에만 적용 (응답 건수 < 5건). 일반 인구통계·인건비 분석에는 동일 임계값이 적용되지 않아 작은 부서 노출 가능. **추후 개선** — §10에 마스킹 확대 |
| 15 | 분석 자동 갱신 cron 미등록 | `/api/v1/analytics/calculate`, `/api/v1/attrition/recalculate`, AI 보고서 생성 등이 `vercel.json` crons에 등록되어 있지 않음. 현재 HR_ADMIN 수동 트리거만 가능. **추후 개선** §10 참조 |

### 11.5 알려진 위험 코드 경로

- 환율 누락 — 월별 ExchangeRate 누락 시 본사 환산 실패. graceful degradation 미구현 (급여 §11.5와 동일 위험)
- 캐시 무효화 누락 — 데이터 변경 후 분석 캐시 무효화 누락 시 stale KPI 노출
- 익명 통계 임계값 우회 — 퇴직 면담·펄스 응답 등 자유 응답 데이터의 응답 건수 < 5건 자동 익명화 로직 누락 시 개인 식별 가능 (인력·인건비 분석에는 현재 미적용 — §9 #14)
- 권한·마스킹 BI 외부 연동 — API 토큰 발급 시 권한 범위 정확히 제한해야. Row-level Security 부분 구현
- AI 보고서 생성 비용 — Claude API 호출 비용 누적. 토큰·캐시·재시도 정책 명확화 필요
- AttritionRiskHistory 시계열 정책 — 관리자 수시 재계산 시마다 row 생성. 자동 주간 cron은 미등록 — 정책 결정 시 row 증가 정책 필요 (Partitioning 또는 archival)
- 분석 스냅샷 vs 운영 데이터 — 스냅샷이 오래된 경우 운영 데이터와 차이. 화면에 어느 시점인지 명시 필요
- AI 위험도 모델 변경 시 과거 데이터 비교 불가 — 모델 버전 관리 필요
