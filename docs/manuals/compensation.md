# 보상 매뉴얼 (Compensation)

> **최종 갱신**: 2026-05-12
> **담당**: 인사관리팀
> **상태**: Draft v1 (CEO 리뷰 전)

---

## 1. 개요

### 1.1 무엇을 다루는가

CTR HR Hub의 보상 모듈은 다음을 통합 관리합니다.

- **급여 밴드 (Salary Band)** — 회사·직급·직군 조합별 최소·중간·최대 급여 범위
- **보상 이력 (Compensation History)** — 모든 보상 변경의 시점별 감사 가능한 이력 (정기·승진·시장 조정 등 10종 유형)
- **인상률 매트릭스 (Salary Adjustment Matrix)** — 평가 등급 × 급여 밴드 위치 조합의 인상률 추천 (예: 탁월 × LOW = +12%)
- **비정기 보상 변경 (Off-Cycle)** — 평가 사이클과 별개로 진행하는 보상 변경 신청 + 결재
- **보상 통지서 (Compensation Letter)** — 보상 변경 시 직원에게 발송되는 PDF 통지서
- **보상 시뮬레이션** — 가상 시나리오 또는 일괄 적용 사전 검증
- **보상 분석** — 회사·부서·직급별 인건비·compa-ratio·분포 KPI
- **AI 보상 추천** — 과거 데이터 기반 적정 인상률 추천 (선택)
- **평가 → 보상 자동 연계** — 평가 사이클 결과를 받아 인상률 자동 산정 → 인사담당자 확정 → 급여 회차 반영

### 1.2 도입 의도

- **일관된 보상 정책** — 회사·직급·직군별 급여 밴드를 시점 기반으로 명문화하여 무원칙 보상 차단
- **평가 결과 활용** — 단순 등급 통보가 아닌 매트릭스로 자동 인상률 산정 → 평가 → 보상 → 급여 자동 연계
- **공정성 + 시장 경쟁력** — Compa-ratio(시장 중간값 대비 비율) 추적으로 시장가치 대비 적정성 검증
- **감사 가능성** — 모든 변경에 사유·결재자·시행일 기록, 정기/예외 구분 명시
- **다국적 법인 대응** — 13개 법인 각각의 통화·급여 밴드·시장 차이 반영 (법인 전환 시 도착 법인 밴드 재조정)
- **사일런트 누수 방지** — 매니저 무단 변경 차단, 모든 변경은 결재 또는 정기 사이클 거침

### 1.3 성과·급여·인사관리 매뉴얼과의 관계

| 항목 | 본 매뉴얼 (보상) | 관련 매뉴얼 |
|------|----------------|-----------|
| 직원의 현재 기본급 | 본 매뉴얼이 SSOT (CompensationHistory 최신값) | 급여 매뉴얼이 참조 (월 급여 계산 시) |
| 평가 등급 → 인상률 | 본 매뉴얼이 매트릭스 적용 | 성과 매뉴얼 §3.9에서 진입 |
| 급여 밴드 | 본 매뉴얼이 SSOT | 채용 공고·인사관리 법인전환에서 참조 |
| 보상 변경 결재 | 본 매뉴얼이 신청 | 결재 매뉴얼 §2.1 비정기 급여 조정 |
| 인상률 → 다음 급여 회차 반영 | 본 매뉴얼이 확정 → 급여로 이관 | 급여 매뉴얼 §3.7 수동 조정 + §4.3 Off-cycle |
| 법인 전환 시 급여 밴드 재조정 | 본 매뉴얼 정책 | 인사관리 매뉴얼 §3.5 |

## 2. 보상 구성 요소

### 2.1 보상 변경 유형 (CompensationChangeType) 10종

| 유형 | 의미 | 트리거 |
|------|------|--------|
| 신규 입사 (HIRE) | 입사 시 최초 보상 | 채용 매뉴얼 §3.9 입사 전환 |
| 정기 인상 (ANNUAL_INCREASE) | 연간·반기 평가 기반 정기 인상 | 평가 사이클 종료 (성과 매뉴얼 §3.9) |
| 승진 (PROMOTION) | 직급 상승 동반 인상 | 인사관리 매뉴얼 §3.2 PROMOTION 발령 |
| 시장 조정 (MARKET_ADJUSTMENT) | 시장 가치 변동 반영 | 시장 조사 결과 |
| 강등 보상 변경 (DEMOTION_COMP) | 직급 하향 동반 변경 | 인사관리 DEMOTION 발령 |
| 법인 전환 보상 변경 (TRANSFER_COMP) | 법인 이동 시 도착 법인 밴드 재조정 | 인사관리 §3.5 COMPANY_TRANSFER |
| 유지 보상 (RETENTION) | 핵심 인재 이탈 방지 | 인사담당자 판단 |
| 역할 변경 (ROLE_CHANGE) | 동일 직급 내 역할 변경 | 부서이동·직위 변경 |
| 주식 보상 (EQUITY) | 스톡옵션·RSU 등 | 별도 정책 |
| 기타 (OTHER) | 위 외 사유 | 자유 사유 입력 |

### 2.2 급여 밴드 (Salary Band)

| 속성 | 설명 |
|------|------|
| 회사 | 적용 법인 (13개 법인) |
| 직급 | 적용 직급 (L1/L2/전문리더/경영리더 등) |
| 직군 | 직군(관리·생산·전문 등) — 선택 적용 |
| 통화 | 회사별 통화 (KRW/CNY/USD/MXN/VND/RUB/PLN) |
| 최소·중간·최대 (min/mid/max) | 해당 직급의 적정 급여 구간 |
| 시행일·종료일 | 시점별 밴드 (매년 또는 분기별 갱신) |

**Compa-Ratio**: 직원 기본급 ÷ 밴드 중간값 (mid). 1.0이 표준, 0.85 미만 = 시장 대비 낮음, 1.20 초과 = 시장 대비 높음 (지표).

### 2.3 인상률 매트릭스 (Salary Adjustment Matrix)

평가 등급 × 급여 밴드 위치 조합의 인상률 추천.

| 등급 \ 밴드 위치 | LOW (하위 1/3) | MID (중간 1/3) | HIGH (상위 1/3) |
|-----------------|:-------------:|:-------------:|:--------------:|
| **탁월 (O)** | 가장 높은 인상 | 중간 인상 | 낮은 인상 (이미 상위) |
| **우수 (E)** | 높은 인상 | 표준 인상 | 낮은 인상 |
| **보통 (M)** | 표준 인상 | 낮은 인상 | 동결 또는 낮음 |
| **미흡 (S)** | 동결 또는 낮음 | 동결 | 동결 |

**원칙**: 동일 등급이라도 밴드 위치 낮으면 더 높은 인상으로 빠르게 적정 수준 도달, 밴드 상위는 이미 높으므로 낮은 인상으로 분포 조정.

각 셀마다 **최소·권장·최대 인상률** 정의. 매트릭스는 회사·평가 사이클별로 설정 가능.

> **데이터 흐름 주의**: 인상률 매트릭스의 등급 키(`gradeKey`)는 코드 내부 enum과 정합 유지가 필요합니다. 성과평가 매뉴얼 §9 #14의 PerformanceGrade enum 마이그레이션(`E/M_PLUS/M/B` → `O/E/M/S`)이 운영 DB에 적용되면 본 매트릭스도 함께 갱신되어야 합니다.

### 2.4 비정기 보상 변경 (Off-Cycle)

평가 사이클과 별개로 진행하는 보상 변경.

| 사유 카테고리 | 의미 |
|--------------|------|
| 승진 (PROMOTION) | 승진 발령 동반 |
| 유지 (RETENTION) | 핵심 인재 이탈 방지 |
| 시장 조정 (MARKET_ADJUSTMENT) | 시장 시세 변동 |
| 역할 변경 (ROLE_CHANGE) | 부서·직위 이동 |
| 주식 (EQUITY) | 주식 보상 변경 |
| 기타 (OTHER) | 자유 사유 |

**5단계 상태**: 작성 중 / 결재 대기 / 승인 / 반려 / 취소

**신청자 유형**: 팀장(MANAGER) 또는 인사담당자(HR) 직접 등록

### 2.5 보상 통지서 (Compensation Letter)

보상 변경 시 직원에게 발송되는 PDF.

- **버전 관리**: 정정·재발행 시 버전 증가, 이전 버전은 무효화 표시
- **상태**: 생성 / 발송 완료 / 실패
- **발송 채널**: 이메일 (선택 — 발송 전 인사담당자 검토 가능)
- **저장**: 회사 파일 저장소에 영구 보관

### 2.6 보상 사이클 (평가 사이클 연계)

보상 사이클은 평가 사이클의 7-8단계로 자연스럽게 진입합니다 (성과 매뉴얼 §3.1).

```
평가 사이클: ... → CALIBRATION → FINALIZED → [COMP_REVIEW] → [COMP_COMPLETED] → CLOSED
                                                ↑                    ↑
                                                보상 매뉴얼 진입        급여 매뉴얼 연동
```

- **COMP_REVIEW**: 매트릭스 기반 인상률 산정 + 인사담당자 검토·조정
- **COMP_COMPLETED**: 결재 완료 → 다음 급여 회차에 자동 반영

## 3. 핵심 정책

### 3.1 정기 보상 사이클 (평가 → 보상)

평가 사이클 종료 후 자동 진입.

```
[평가 사이클 FINALIZED]
       │
       ↓
[COMP_REVIEW 진입]
       │
       ├── 직원별 인상률 자동 산정 (매트릭스 적용)
       │     · 등급 (성과·역량 종합) × 밴드 위치 → 권장 인상률
       │     · 밴드 초과 시 예외 마킹
       │
       ↓
[인사담당자 검토·조정]
       │     · 권장값 그대로 적용 또는 개별 조정
       │     · 조정 시 사유 입력 (감사 대상)
       │     · 부서별·법인별 인건비 시뮬레이션
       │
       ↓
[결재 (정기 사이클은 일괄 결재 또는 미결재 — 회사 정책)]
       │
       ↓
[COMP_COMPLETED]
       │
       ├── CompensationHistory 일괄 생성 (시행일 = 사이클 정의)
       ├── CompensationLetter 일괄 생성 + 발송 옵션
       └── 다음 급여 회차에 자동 반영
```

### 3.2 비정기 보상 변경 (Off-Cycle) 워크플로

```
[작성] (팀장 또는 인사담당자)
       │
       ↓
[결재 상신]
       │
       ↓
[결재 진행] ── 직속 팀장 → 인사담당자 (결재 매뉴얼 §2.1 비정기 급여 조정)
       │     · 셀프 결재 차단 (직속 팀장이 본인 신청 결재 불가 — 결재 매뉴얼 §3.3)
       │     · 밴드 초과 시 예외 처리 + 추가 검토
       ├── 승인 ──→ CompensationHistory 생성 (시행일 적용)
       │              │
       │              ├── 통지서 PDF 자동 생성
       │              └── 다음 급여 회차에 반영 (또는 미래 시행일 예약)
       │
       └── 반려 ──→ 사유와 함께 신청자에게 회신 → 수정 후 재신청 가능
```

### 3.3 신규 입사 보상 (HIRE)

채용 매뉴얼 §3.9 입사 전환 시 자동 생성.

- **이전 기본급**: 없음 (HIRE이므로 0)
- **새 기본급**: 채용 매뉴얼의 offeredSalary
- **시행일**: 입사일
- **밴드 검증**: 새 기본급이 도착 직급 밴드 범위 내인지 자동 검증
- **밴드 초과 시**: 예외 마킹 + 사유 자동 기록 ("채용 협상 결과")

### 3.4 승진 동반 보상 변경

인사관리 매뉴얼 §3.2 PROMOTION 발령과 동반.

- **트리거**: PROMOTION 발령 생성 시 시스템이 자동으로 Off-Cycle 신청 생성 (선택)
- **권장 인상률**: 직급 변경 시 표준 인상률 (회사 설정, 예: L1→L2 +15%)
- **승진 결재와 보상 결재 분리**: 발령 결재 → 보상 결재 (두 단계, 회사 정책에 따라 통합 가능)
- **추후 개선** 항목: 승진 → Off-Cycle 자동 트리거 (§9 #1)

### 3.5 시장 조정 보상

회사가 시장 시세 변동을 반영하여 일괄 또는 개별 조정.

- **일괄 조정**: 인사담당자가 직군·직급 단위로 일괄 인상률 적용
- **개별 조정**: Off-Cycle로 개별 직원 인상
- **사유**: "시장 조사 결과" 등 명시 (감사 대상)

### 3.6 법인 전환 보상 변경 (TRANSFER_COMP)

인사관리 매뉴얼 §3.5 정책 그대로 적용.

- **재조정 원칙**: 도착 법인의 직급별 급여 밴드에 맞춰 재산정
- **출발 법인 급여 참고**: 도착 법인 밴드 범위 내에서 결정
- **자동 생성**: COMPANY_TRANSFER 발령 시 시스템이 새 기본급 추천 → 인사담당자 확정

### 3.7 유지 보상 (RETENTION)

핵심 인재 이탈 방지 목적의 비정기 인상.

- **트리거**: 인사담당자 또는 본부장 판단
- **결재**: 일반 Off-Cycle 흐름 + 추가 검토 (이탈 위험 평가, 시장 시세 조사)
- **밴드 초과 빈번**: 시장 가치 높은 인재는 밴드 max 초과하기 쉬워 예외 처리 자주 발생 → 별도 추적

### 3.8 강등 보상 변경 (DEMOTION_COMP)

징계 또는 역량 부족으로 직급 하향 시.

- **인사관리 매뉴얼 §3.2 DEMOTION 발령**과 동반
- **감액 한도**: 한국 근로기준법상 임금 감액에는 제한이 있음 — 회사·노무사 정책 따름
- **결재**: 일반 Off-Cycle보다 엄격한 결재 (인사담당자 + 본부장 또는 대표)
- **추후 개선** 항목: 강등 보상에 별도 결재 흐름 (§9 #2)

### 3.9 미래 시행일 예약 (Scheduled)

보상 변경의 시행일이 미래인 경우.

- `isScheduled = true`로 마킹, `effectiveDate` 미래 날짜
- 시행일 도래 시 자동으로 다음 급여 회차에 반영
- 시행일 전까지는 직원의 "현재 기본급"에 미반영 (예약 상태)
- 운영 예: 연봉 인상은 4월 1일 시행이지만 3월 초에 확정 → 3월 급여에는 미반영, 4월 급여부터 반영

### 3.10 보상 통지 정책

보상 변경 후 직원에게 통지하는 절차.

- **자동 PDF 생성**: 변경 즉시 또는 인사담당자 수동 트리거
- **이메일 발송**: 회사 정책으로 자동 발송 또는 인사담당자 검토 후 수동 발송
- **직원 화면 노출**: 통지서 + 보상 이력 본인 조회 (마스킹 정책 없음, 즉시 공개)
- **정정 시**: 새 버전 생성, 이전 버전 무효화 표시 (감사 대상)

### 3.11 데이터 마스킹 — 보상 마스킹 정책

- **직원**: 본인 보상 정보만 조회 가능 (다른 직원 보상 절대 비공개)
- **팀장**: 본인 팀원 보상 조회 — **회사 정책에 따라 차등** (전체 조회 / 밴드 위치만 / 익명 분포만)
- **인사담당자**: 본인 법인 전체 보상 데이터
- **전사관리자**: 13개 법인 전체

> **데이터 흐름 주의**: 팀장의 팀원 보상 조회 권한은 회사·법인별 정책 차이가 큽니다. 현재 시스템은 팀장이 본인 팀원의 보상 변경 신청(Off-Cycle)을 위해 본인이 신청하는 직원의 현재 기본급은 조회 가능합니다. 그러나 신청과 무관한 팀원 보상 조회는 회사 정책에 따라 결정됩니다.

### 3.12 자동 알림

- **정기 사이클 진입**: 평가 FINALIZED → 인사담당자에게 보상 기획 시작 알림
- **인상률 매트릭스 적용 완료**: 인사담당자에게 검토 요청 알림
- **Off-Cycle 결재 도착**: 결재자(직속 팀장 → 인사담당자)에게 알림 (결재 매뉴얼 §4.3)
- **결재 결과**: 신청자에게 알림
- **보상 변경 확정**: 직원에게 통지서 발송 알림
- **미래 시행일 D-7**: 직원·인사담당자에게 시행 임박 알림 (선택)

- **개별 보상 정보**: 본인만 + 인사담당자·전사관리자
- **팀장의 팀원 보상 조회**: 회사 정책 (전체 / 밴드 위치만 / 익명 분포만 / 본인 신청 건만)
- **인건비 KPI**: 부서·법인 단위 집계는 인사담당자·전사관리자만
- **통지서 PDF**: 본인 + 인사담당자
- **결재 이력**: 신청자·결재자·인사담당자·전사관리자

> **데이터 흐름 주의**: 팀장의 팀원 보상 조회 권한은 회사별로 결정 필요한 정책입니다. 시스템 기본값은 "본인 신청 건만 조회" 보수적 정책이며, 회사 정책에 따라 확대 가능합니다 (§9 #5 참조).

## 7. 특수 상황 처리

### 7.1 밴드 초과 직원

- 현재 기본급이 직급 밴드 max 초과
- 정기 사이클에서 매트릭스 적용 시 자동 마킹 (예외 표시)
- 인사담당자가 검토 → 동결 / 소액 인상 / 별도 사유 입력 결정
- 빈번한 경우: 밴드 자체 재검토 (시장 시세 변동)

### 7.2 밴드 미달 직원

- 현재 기본급이 직급 밴드 min 미달
- 시스템이 우선 인상 후보로 표시
- 매트릭스 권장값이 가장 높음 (LOW × 등급 조합)
- 빠르게 밴드 진입 권장

### 7.3 평가 등급이 보통(M)인데 시장 가치 높은 직원

- 매트릭스 권장값은 표준 인상 또는 낮음
- 시장 시세 대비 낮으면 이탈 위험
- 인사담당자가 RETENTION 사유로 별도 인상 검토

### 7.4 휴직 중 직원의 정기 보상

- 휴직 활성 상태 직원도 정기 사이클 보상 대상에 자동 포함 (평가 결과가 있다면)
- 회사 정책으로 휴직 직원은 제외 가능 (수동)
- 휴직 후 복직 시 인상 반영 시점은 시행일 또는 복직일 (회사 정책)

### 7.5 사이클 중 입사한 직원

- 신규 입사자는 입사 시 HIRE 보상으로 처음 등록
- 첫 평가 사이클까지는 정기 인상 대상 아님 (회사 정책으로 비례 적용 가능)

### 7.6 법인 전환 직원

- 인사관리 매뉴얼 §3.5 정책: 도착 법인 급여 밴드 재조정
- TRANSFER_COMP 유형으로 자동 보상 변경 기록 생성
- 출발 법인 정기 사이클이 진행 중이면 출발 법인 사이클에 우선 참여 후 이동

### 7.7 정기 사이클 도중 Off-Cycle 진행

- 정기 사이클 COMP_REVIEW 중에 같은 직원에 대한 Off-Cycle 신청
- 시스템이 충돌 경고 + 인사담당자 판단
- 일반적으로 정기 사이클 완료 후 Off-Cycle 처리 권장

### 7.8 시행일이 지난 보상 변경

- Off-Cycle 결재가 시행일 이후 완료된 경우
- 시행일 = 결재 시점 다음 달 또는 결재일로 자동 조정
- 인사담당자 판단 (소급 적용 시 급여 모듈에서 소급 조정 — 급여 매뉴얼 §3.7)

### 7.9 보상 변경 정정 (Compensation Letter)

- 통지서 발송 후 오류 발견
- 인사담당자가 새 버전 생성 + 이전 무효화 + 사유 기록
- 직원에게 정정 통지서 재발송
- 감사 대상 (모든 버전 영구 보존)

### 7.10 강등 시 임금 감액 제약

- 한국 근로기준법상 임금 감액에는 제한 (불이익 변경 제한)
- 시스템이 자동 차단하지 않음 — 인사담당자가 노무사 검토 후 결재
- 회사·노무사 정책 따름

### 7.11 통화 환산이 필요한 보상 분석

- 13개 법인 각자 통화 (KRW/CNY/USD/MXN/VND/RUB/PLN)
- 본사 KPI 환산은 급여 매뉴얼 §3.11 ExchangeRate 사용
- Compa-ratio·밴드 비교는 같은 통화 내에서만 (해외 법인끼리 직접 비교 안 함)

## 8. 자주 묻는 질문

**Q. 내 연봉은 어디서 확인하나요?**
A. `보상 → 내 보상` 또는 `프로필 → 보상`에서 현재 기본급, 통화, 시행일, 보상 이력, 통지서를 조회·다운로드할 수 있습니다.

**Q. 평가 등급이 높은데 인상률이 생각보다 낮아요. 왜죠?**
A. 인상률은 등급 × 급여 밴드 위치 조합으로 결정됩니다. 등급이 높아도 현재 기본급이 밴드 상위(HIGH)에 있으면 인상률이 낮을 수 있습니다 (이미 충분히 높기 때문). 자세한 정책은 인사담당자에게 문의하세요.

**Q. 우리 회사의 평가 등급별 인상률 매트릭스는 누가 정하나요?**
A. 인사담당자가 매년 평가 사이클 시작 전에 매트릭스를 설정합니다. 회사·법인별로 다를 수 있고, 시장 시세·인건비 예산을 반영합니다.

**Q. 같은 등급인데 동료보다 인상률이 낮아요. 왜죠?**
A. 같은 등급도 현재 기본급의 밴드 위치(LOW/MID/HIGH)에 따라 인상률이 다릅니다. 밴드 하위에 있는 사람이 더 빠르게 적정 수준에 도달하도록 매트릭스가 설계되어 있습니다.

**Q. Compa-ratio가 뭔가요?**
A. 본인 기본급을 직급 급여 밴드의 중간값(mid)으로 나눈 비율입니다. 1.0이 표준, 0.85 미만은 시장 대비 낮음, 1.20 초과는 시장 대비 높음을 의미합니다.

**Q. 정기 사이클이 아닌데 인상받을 수 있나요?**
A. 가능합니다. 비정기(Off-Cycle) 보상 변경 신청을 통해 진행됩니다. 일반적으로 직속 팀장이 신청 → 인사담당자가 결재하는 2단계 흐름입니다.

**Q. 승진하면 자동으로 연봉이 오르나요?**
A. 현재 시스템은 승진 발령과 보상 변경을 분리하여 처리합니다. 승진 발령 후 별도 Off-Cycle 보상 변경 신청 또는 인사담당자 수동 처리가 필요합니다. 자동 연계는 **추후 개선** 항목입니다 (§9 #1).

**Q. 보상 변경이 4월 1일 시행인데 3월 급여에 반영되나요?**
A. 아닙니다. 미래 시행일(isScheduled)로 예약된 변경은 시행일 도래 후 다음 급여 회차부터 반영됩니다. 4월 1일 시행이면 4월 급여부터 새 금액이 적용됩니다.

**Q. 팀장이 내 연봉을 다 보나요?**
A. 회사 정책에 따라 다릅니다. 시스템 기본값은 팀장이 Off-Cycle 신청을 위해 본인이 신청하는 팀원의 현재 기본급만 조회 가능합니다. 일반적인 팀원 보상 조회는 회사·법인별 결정입니다.

**Q. 통지서를 못 받았어요. 어디서 다운로드하나요?**
A. `보상 → 내 보상`에서 통지서 목록을 확인할 수 있습니다. 이메일 발송 실패 시에도 시스템에는 PDF가 생성되어 있습니다.

**Q. 통지서에 오류가 있어요. 어떻게 정정하나요?**
A. 인사담당자에게 문의하세요. 새 버전이 생성되고 이전 버전은 무효화 표시됩니다. 정정 통지서가 발송됩니다.

**Q. 법인 전환 시 연봉이 깎였어요. 왜죠?**
A. 인사관리 매뉴얼 §3.5에 따라 법인 전환 시 도착 법인의 급여 밴드에 맞춰 재조정됩니다. 출발 법인보다 낮은 시장의 법인으로 이동하면 감액될 수 있습니다. 근속은 통산 유지됩니다 (퇴직금·연차 기준).

**Q. 강등되면 연봉이 자동으로 깎이나요?**
A. 자동으로 깎이지 않습니다. 강등 발령 후 별도 보상 변경 절차가 필요하며, 한국 근로기준법상 임금 감액 제한이 있어 신중한 검토가 필요합니다 (§7.10 참조).

**Q. 매트릭스의 등급 키(O/E/M/S)가 평가 등급과 다르게 보여요. 왜죠?**
A. ADR `2026-04-06`에 따라 4등급이 O/E/M/S로 확정되었으나, 운영 DB에 일부 기존 데이터(E/M_PLUS/M/B)가 마이그레이션 대기 중입니다 (§9 #6). 코드 schema는 이미 O/E/M/S 적용되어 있으며, 시스템이 점진적으로 정합화될 예정입니다.

## 9. 알려진 제약

| # | 항목 | 상태 |
|---|------|------|
| 1 | 승진 발령 → 보상 변경 자동 연계 부재 | 인사관리 PROMOTION 발령 시 자동으로 Off-Cycle 신청 생성 안 함. 인사담당자 수동 처리. **추후 개선** — 발령 시 Off-Cycle 자동 트리거 |

| 4 | 보상 시뮬레이션 다년 시나리오 부재 | 단일 시점 시뮬레이션만 지원. 3년·5년 인상 시나리오 미지원. **추후 개선** — 다년 시나리오 (급여 매뉴얼 §9 #11과 통합) |
| 5 | 팀장의 팀원 보상 조회 정책 회사 결정 필요 | 시스템 기본값은 보수적(본인 신청 건만). 회사별 확대 정책은 수동 설정. **추후 개선** — 회사 정책 옵션 + 감사 로그 |
| 6 | PerformanceGrade enum 마이그레이션 (인상률 매트릭스 영향) | 운영 DB의 SalaryAdjustmentMatrix.gradeKey가 일부 E/M_PLUS/M/B 잔존 가능. **추후 개선** — 성과 매뉴얼 §9 #14 마이그레이션과 함께 일괄 처리 |
| 7 | 외부 시장 데이터 임포트 자동 연동 부재 | Mercer·Aon·Willis Towers Watson 등 시장 데이터 자동 연동 없음. 인사담당자 수동 입력. **추후 개선** — 주요 시장 조사 데이터 API 연동 |
| 8 | 통지서 다국어 부분 지원 | 통지서 PDF가 한국어 중심. 영문·중국어 등 다국어 부분. **추후 개선** — 회사·직원 locale 기반 다국어 통지서 (급여 매뉴얼 §9 #5와 통합) |
| 9 | 정기 사이클·Off-Cycle 충돌 자동 차단 미흡 | 같은 직원에 대한 정기·비정기 동시 진행 시 경고만, 차단 안 함. **추후 개선** — 충돌 자동 차단 또는 우선순위 정책 |
| 10 | AI 보상 추천 정확도 검증 부재 | AI 추천 기능은 있으나 추천 근거·정확도 분석 데이터 부재. **추후 개선** — AI 추천 vs 실제 결정 비교 분석 |
| 11 | 임금 감액 한도 자동 검증 부재 | 강등·정정 시 한국 근로기준법 감액 제한 자동 검증 없음. 인사담당자가 노무사 수동 검토. **추후 개선** — 감액 한도 자동 검증 |
| 12 | 보상 통지서 이메일 발송 실패 처리 단순화 | 발송 실패 시 인사담당자 수동 재발송. 자동 재시도 없음. **추후 개선** — 자동 재시도 + 다중 채널 (Teams·SMS 등) |
| 13 | 인건비 예산 관리 부재 | 인상률 시뮬레이션에서 예산 한도 표시는 있으나 실제 예산 관리 시스템과 연동 없음. **추후 개선** — 예산 시스템 연동 또는 자체 예산 모듈 |

## 10. 부록: 기술 메모

> 본문이 정책 설명, 이 섹션은 개발/QA 참고용. HR은 건너뛰어도 무방합니다.

### 10.1 데이터 모델

핵심 모델 (모두 `prisma/schema.prisma`):

- `SalaryBand` — 회사·직급·직군별 급여 밴드 (시점 기반, currency, min/mid/max)
- `CompensationHistory` — 보상 변경 이력 (CompensationChangeType 10종, isScheduled 미래 시행, compaRatio, isException + exceptionReason)
- `CompensationLetter` — 통지서 PDF (version, s3Key, status: GENERATED/SENT/FAILED, invalidatedAt)
- `OffCycleCompRequest` — 비정기 보상 변경 (5-state OffCycleRequestStatus, OffCycleReasonCategory 6종, OffCycleInitiatorType 2종, triggerEventType 자동 트리거 맥락)
- `OffCycleApprovalStep` — Off-Cycle 결재 단계
- `SalaryAdjustmentMatrix` — 인상률 매트릭스 (emsBlock 또는 gradeKey × comparatioBand, merit min/max/recommended %)
- `CompensationSetting` — 회사별 보상 설정 (payComponents·salaryBands·raiseMatrix·bonusRules JSON, currency)

**핵심 enum**:
- `CompensationChangeType`: HIRE / ANNUAL_INCREASE / PROMOTION / MARKET_ADJUSTMENT / DEMOTION_COMP / TRANSFER_COMP / RETENTION / ROLE_CHANGE / EQUITY / OTHER (10종)
- `OffCycleRequestStatus`: DRAFT / PENDING_APPROVAL / APPROVED / REJECTED / CANCELLED
- `OffCycleReasonCategory`: PROMOTION / RETENTION / MARKET_ADJUSTMENT / ROLE_CHANGE / EQUITY / OTHER
- `OffCycleInitiatorType`: MANAGER / HR
- `CompensationLetterStatus`: GENERATED / SENT / FAILED
- `PayrollApprovalStepStatus`: PENDING / APPROVED / REJECTED (OffCycleApprovalStep도 동일 enum 재사용)

### 10.5 알려진 위험 코드 경로

- `SalaryAdjustmentMatrix.gradeKey` enum 정합 — 코드의 PerformanceGrade(O/E/M/S)와 매트릭스 gradeKey 일치 필수. 운영 DB 잔존 데이터(E/M_PLUS/M/B) 마이그레이션 시 매트릭스 함께 갱신 필요 (§9 #6, 성과 §9 #14)
- `CompensationHistory.compaRatio` 재계산 — 직원의 직급·밴드 변경 시 compaRatio가 자동 재계산되어야 하나 현재 트리거가 부분 구현. 일관성 확인 필요
- `isScheduled` 미래 시행일 자동 반영 — cron으로 시행일 도래 시 다음 급여 회차에 반영. cron 실패 시 silent miss 가능. 모니터링 필요
- Off-Cycle 결재의 셀프 결재 차단 — 결재 매뉴얼 §3.3 셀프 결재 가드. `OffCycleCompRequest.initiatorId` ≠ 1단계 결재자 확인 필수
- `CompensationLetter.s3Key` 영구 보관 — 정정 시 이전 버전 무효화 표시만, 삭제 안 함. 저장 비용 누적 가능 (감사 정책으로 의도된 동작)
- 매트릭스 누락 셀 처리 — 등급 × 밴드 위치 조합 중 매트릭스 누락 셀(예: O × HIGH 미정의) 시 fallback 정책 명확화 필요
- 통화 변환 누락 — 해외 법인 보상을 본사 KPI에 환산 시 ExchangeRate 누락 월 fallback 없음 (급여 §10.5와 동일 위험)

### 10.6 추후 개선 항목

**자동화·연계**:
1. 승진 발령 → Off-Cycle 자동 트리거 (§9 #1)
2. 강등 보상 변경 별도 결재 흐름 + 노무사 게이트 (§9 #2, #11)
3. 정기 사이클 부서장 결재 옵션 (§9 #3)
4. 정기·Off-Cycle 충돌 자동 차단 (§9 #9)

**분석·시뮬레이션**:
5. 다년 시나리오 시뮬레이션 (§9 #4, 급여 §9 #11과 통합)
6. AI 보상 추천 정확도 분석 (§9 #10)
