# CTR HR Hub — GP#1 휴가 파이프라인 설계서 v1.1 (최종)

> **문서 목적:** GP#1 휴가 파이프라인 최종 설계안. 인사팀/법인 HR/경영진 합의 및 개발 에이전트 구현 지시용.
> **작성일:** 2026-03-10
> **버전:** v1.1 Final (CTO/CEO 아키텍처 결정사항 3건 반영 + v1.1b 코드스캔 검증: 구현 초과분 부록 C 추가)

## 1. 왜 휴가 파이프라인을 설계하는가

현재 CTR의 휴가 모듈은 **개별 기능으로는 구현되어 있지만, 하나의 파이프라인으로 연결되어 있지 않습니다.**

GP#1의 목표는 이 모듈들을 **Workday 수준의 end-to-end 파이프라인**으로 연결하여:

- 직원이 휴가를 신청하면 **잔여일수 실시간 반영 → 승인 → 급여 연동까지 자동으로 흐르게** 만들고
- 매니저가 **팀 부재 상황을 한눈에 파악**한 후 승인/반려를 판단할 수 있으며
- HR이 **법인별 휴가 정책을 유연하게 설정**하고, 전사 휴가 사용 현황을 분석할 수 있는 구조를 완성하는 것입니다.

### 왜 Leave Pipeline이 Golden Path #1인가?

| 이유 | 설명 |
|------|------|
| 전 직원이 매일 사용 | 가장 높은 사용 빈도. 직원 경험의 첫인상을 결정 |
| 즉각 반응이 중요 | 잔여일수 변화를 실시간으로 보여줘야 신뢰 형성 |
| 다법인 복잡도 | 6개국 × 11개 이상 휴가 유형 × 법인별 노동법 차이 |
| 급여 직결 | 유급/무급 처리가 급여 계산에 바로 영향 |
| 승인 워크플로 | 가장 기본적인 승인 패턴 — 다른 모듈의 승인 기반이 됨 |

## 2. 5단계 파이프라인 요약

```
STEP 1        STEP 2        STEP 3          STEP 4          STEP 5
정책 설정  → 잔여일수 부여 → 휴가 신청    → 승인 처리    → 급여 연동
(HR 설정)    (연초/월별)    (직원→매니저)  (매니저→직원)   (PayrollRun)
                                                           ↓
                                                      근태 기록 반영
```

| 단계 | 주체 | 주기 |
|------|------|:---:|
| STEP 1 정책 설정 | HR | 연 1회 (법인 설정 시) |
| STEP 2 잔여일수 부여 | 시스템 (자동) | 연초 일괄 또는 월별 발생 |
| STEP 3 휴가 신청 | 직원 | 수시 |
| STEP 4 승인 처리 | 매니저 → 알림 → 직원 | 수시 |
| STEP 5 급여 연동 | 시스템 (자동) | 급여 처리 시 |

## 3. 🚨 핵심 아키텍처 원칙 (4건)

> **이 원칙들은 전체 파이프라인에 공통 적용됩니다.**

### 원칙 1: 이중 잔액 모델 (Dual Balance)

```
LeaveBalance (정책 테이블 — 정적 잔액)
  totalDays = 15.0       ← 연간 부여 일수
  usedDays = 7.0         ← 승인 확정된 사용 일수
  remaining = 8.0        ← 확정 잔여 (= total - used)

LeaveRequest (신청 테이블 — 동적 잔액 계산)
  PENDING 건들의 합산 = 2.0  ← 신청했지만 아직 미승인

직원이 보는 잔여일수:
  실제 사용 가능 = remaining - PENDING 합산 = 6.0일
```

**왜 이중 모델인가:**

- `LeaveBalance`는 확정된 사실만 기록 (승인 완료된 것만 반영)
- `LeaveRequest` PENDING 건은 별도로 합산하여 "사용 가능 잔여"를 계산
- 두 테이블은 **의도적으로 동기화하지 않음** — 각각의 역할이 다름
- Workday도 동일한 패턴: "Balance" ≠ "Available Balance"

### 원칙 2: Optimistic UI

```
직원이 "신청" 클릭
    ↓
즉시 UI에 반영: 잔여일수 감소 + 신청 목록에 PENDING 표시
    ↓
서버 응답 대기 (백그라운드)
    ↓
성공 → 확정 / 실패 → 롤백 + 에러 메시지
```

- 직원 체감: "클릭하면 바로 반영된다"
- 네트워크 지연이 있어도 UX가 끊기지 않음
- 공장 현장(제조직)의 열악한 네트워크 환경에서 특히 중요

### 원칙 3: 마이너스 연차 (Negative Balance) 허용

```
잔여일수가 0일인 직원
    ↓
법인 설정에서 allowNegativeBalance = true이면
    ↓
최대 마이너스 한도(예: -3일)까지 연차 "가불" 허용
    ↓
이듬해 연차 부여 시 자동 차감 (빚 상환)
```

**왜 마이너스 연차인가:**

- 예기치 못한 상황(장례, 질병)에서 잔여 0일이라고 차단하면 직원 경험 최악
- 무급휴가로 전환하려면 HR 예외 처리가 필요 → HR 부담 증가
- 시스템이 자동으로 빚을 관리하고, 다음해에 알아서 차감하면 모든 당사자가 편함
- 법인별 ON/OFF + 한도 설정으로 통제 가능

### 원칙 4: 결재 병목 방지 — 대결(Delegation) 시스템

```
매니저가 출장/휴가로 자리를 비울 때
    ↓
시스템에서 "승인 권한 위임(Delegation)" 설정
    ↓
위임 기간 동안 수임자가 해당 팀의 휴가 승인/반려 처리
    ↓
위임 기간 종료 → 자동 회수
```

**왜 대결 시스템인가:**

- 매니저 1명이 2주 출장 가면, 팀원 휴가 신청이 2주간 PENDING에 묶임
- "HR이 대리 승인"은 매니저가 20명이면 HR이 모든 팀 상황을 파악해야 함 — 비현실적
- 매니저 본인이 위임 대상자를 지정하면, 권한과 책임이 명확
- `ApprovalDelegation` 모델: 위임자, 수임자, 시작일, 종료일, 위임 범위

## 4. STEP 1 — 정책 설정

HR이 법인별 휴가 정책을 설정합니다.

### 글로벌 + 법인 오버라이드 패턴

```
글로벌 기본값 (companyId = NULL)
    ↓
법인별 오버라이드 (companyId = 'CTR-KR' 등)
    ↓
직원에게 적용되는 최종값 = 법인 오버라이드 || 글로벌 기본값
```

이 패턴은 급여(GP#3), 성과(GP#4)와 동일합니다.

### 정책 설정 항목

| 설정 | 설명 | 예시 |
|------|------|------|
| 휴가 유형 | 연차/병가/경조사/출산 등 | KR: 11개 유형 |
| 연간 부여 일수 | 유형별 기본 일수 | 연차: 15일 (1년차) |
| 부여 방식 | 연초 일괄(Front-loaded) / 월별 발생(Accrual) | KR: 입사일 기준 발생 |
| 이월 정책 | 이월 가능 여부 + 이월 상한 일수 | 최대 10일 이월 |
| 최소 신청 단위 | 종일 / 반차 / 시간 단위 | 연차: 반차 가능 |
| 유급/무급 | 급여 차감 여부 | 연차: 유급, 무급휴가: 무급 |
| 근속 연동 | 근속년수에 따른 가산일 | KR: 3년마다 +1일 |
| 증빙 필요 여부 | 특정 유형은 증빙 첨부 필수 | 병가 3일 이상: 진단서 |
| **마이너스 연차** | 허용 여부 + 최대 마이너스 한도 | KR: ON, -3일 한도 |
| 사전 신청 기간 | 최소 N일 전 신청 규칙 | 연차: 3일 전 |
| 연속 휴가 상한 | 최대 연속 사용 일수 | 10일 |

### 법정 휴가 자동 세팅

법인 생성 시 `lib/labor/{country}.ts`의 `leave_types`를 기반으로 법정 휴가가 자동 생성됩니다. HR은 법정 기준 이상으로만 커스터마이즈 가능합니다 (법정 기준 미만으로 설정 불가).

## 5. 휴가 유형 체계

### CTR 글로벌 표준 휴가 유형 (11개)

| 코드 | 한글 | 유급/무급 | 설명 |
|:---:|------|:---:|------|
| ANNUAL | 연차 | 유급 | 법정 유급 휴가. 근속 연동 |
| SICK | 병가 | 법인별 | 질병/부상 사유. 3일 이상 시 증빙 필수 |
| MATERNITY | 출산휴가 | 유급 | 법정 출산 전후 휴가 |
| PATERNITY | 배우자출산 | 유급 | 배우자 출산 시 |
| FAMILY_CARE | 가족돌봄 | 무급 | 가족 간호/돌봄 |
| BEREAVEMENT | 경조사 | 유급 | 결혼/상 등 (관계별 일수 차등) |
| WEDDING | 결혼 | 유급 | 본인 결혼 |
| MENSTRUAL | 생리 | 유급 | KR 법정 (월 1일) |
| PTO | 유급시간외 | 유급 | 포괄적 유급 휴가 (US 등) |
| PERSONAL | 개인사유 | 무급 | 기타 개인 사유 |
| OTHER | 기타 | 설정 | HR이 정의하는 추가 유형 |

### 법인별 차이

- **KR:** 연차 15일(1년차) + 근속 가산 + 생리휴가 + 경조사 상세 구분
- **US:** PTO 통합형 (연차/병가 구분 없이 15~20일 PTO)
- **CN:** 연차 5~15일 (근속 연동) + 법정 공휴일 다수
- **VN:** 연차 12일 + 병가 30일(사회보험 연동)
- **RU:** 연차 28일(법정 최소) + 병가 무제한(의료 증빙)
- **MX:** 연차 12일(1년차) + 근속 가산 + 크리스마스 보너스 연동

## 6. STEP 2 — 잔여일수 부여

### 부여 방식 (2가지)

| 방식 | 설명 | 적용 법인 |
|------|------|----------|
| **Front-loaded** | 연초(또는 입사일 기준) 일괄 부여 | US, MX |
| **Monthly Accrual** | 매월 1일 발생분 가산 | KR (입사일 기준), CN, VN, RU |

### 연초 일괄 부여 (Front-loaded)

```
매년 1/1 또는 회계연도 시작일
    ↓
시스템 자동 실행:
  1. 이전 년도 잔여 이월 처리 (carry_over 정책 적용)
  2. 마이너스 잔액 상환: 이월 후 잔여에서 마이너스분 자동 차감
  3. 신규 연도 총 일수 부여 (근속 가산 반영)
  4. LeaveBalance 레코드 생성/갱신
```

### 월별 발생 (Monthly Accrual)

```
매월 1일
    ↓
해당 국가 정책에 따라:
  연간 15일 ÷ 12개월 = 월 1.25일 발생
    ↓
LeaveBalance.totalDays += 1.25
```

### 입사 첫해 비례 부여

중도 입사자는 **잔여 월수에 비례**하여 부여합니다.

| 입사월 | 비례 일수 (연간 15일 기준) |
|:---:|:---:|
| 1월 | 15.0일 |
| 4월 | 11.25일 |
| 7월 | 7.5일 |
| 10월 | 3.75일 |

### 마이너스 잔액 상환 (연초 자동)

```
전년도 잔여: -2.0일 (마이너스 연차 사용)
이월 가능분: 3.0일
    ↓
이월 후 잔여: 3.0 - 2.0 = 1.0일 (자동 상환)
    ↓
신규 부여: 15.0일
    ↓
올해 총 잔여: 1.0 + 15.0 = 16.0일
```

## 7. STEP 3 — 휴가 신청

### 플로우

```
직원: 휴가 신청 폼 작성
  ├ 유형 선택 → 잔여일수 실시간 표시
  ├ 날짜 선택 (시작~종료) → 공휴일/주말 자동 제외
  ├ 반차 선택: 오전/오후 (해당 유형이 반차 허용 시)
  ├ 사유 입력 (선택/필수 — 유형별 설정)
  └ 증빙 첨부 (필요 시)
    ↓
시스템 검증:
  ├ 잔여일수 >= 신청일수? (마이너스 허용 시: 잔여 >= 마이너스 한도?)
  ├ 중복 신청 없는지?
  ├ 법인별 제한 규칙 충족? (사전 신청 기간, 연속 상한 등)
  └ 증빙 필수 유형인데 미첨부?
    ↓
PENDING 상태로 생성 → 매니저에게 알림 (또는 Delegation 수임자에게)
```

### 신청 규칙

| 규칙 | 내용 |
|------|------|
| 잔여 부족 시 (마이너스 미허용) | 신청 불가 (버튼 비활성화) |
| 잔여 부족 시 (마이너스 허용) | 한도 내이면 신청 가능 + ⚠️ 경고 표시: "마이너스 연차 N일 사용됩니다" |
| 마이너스 한도 초과 | 신청 불가 (버튼 비활성화) |
| 중복 신청 | 동일 날짜에 이미 신청/승인 건이 있으면 차단 |
| 사전 신청 기간 | 법인별 설정 가능 (예: 최소 3일 전) |
| 연속 휴가 상한 | 법인별 설정 가능 (예: 최대 연속 10일) |
| 팀 동시 부재 제한 | 설정 가능: 같은 팀에서 동시에 N명 이상 휴가 불가 (Soft Warning) |

### 🚨 팀 부재 컨텍스트

직원이 신청 폼에서 날짜를 선택하면, **해당 기간의 팀 부재 현황**이 자동 표시됩니다.

```
📅 2026-03-15 (월) ~ 2026-03-17 (수)

⚠️ 팀 부재 현황:
  • 김대리 — 연차 (3/15~3/16)
  • 박과장 — 출장 (3/14~3/18)
  
🟡 이 기간 팀 잔여 인원: 5명 중 2명
```

매니저가 아닌 **직원 본인에게도** 이 정보를 보여주는 이유:
- 직원이 알아서 날짜를 조정하므로 불필요한 반려가 줄어듦
- Workday의 "Team Absence Calendar" 기능과 동일

## 8. STEP 4 — 승인 처리

### 승인 플로우

```
매니저 승인함에서 대기 건 확인 (또는 Delegation 수임자)
    ↓
참고 정보 표시:
  ├ 직원 잔여일수 (유형별)
  ├ 팀 부재 현황 (캘린더)
  ├ 대체 인력 가용성
  ├ 해당 직원의 최근 휴가 이력
  └ ⚠️ 마이너스 연차 경고 (해당 시)
    ↓
승인 또는 반려
  ├ 승인 → APPROVED
  │   ├ LeaveBalance.usedDays += 사용일수
  │   ├ 근태 기록에 휴가일 자동 반영
  │   ├ 직원에게 승인 알림
  │   └ LEAVE_APPROVED 이벤트 발행
  │
  └ 반려 → REJECTED
      ├ 반려 사유 필수 입력
      ├ LeaveBalance 변동 없음
      ├ 직원에게 반려 알림 (사유 포함)
      └ LEAVE_REJECTED 이벤트 발행
```

### 🚨 대결(Delegation) 시스템

매니저가 출장/휴가 등으로 자리를 비울 때, 승인 권한을 위임합니다.

```
매니저가 부재 전 시스템에서 Delegation 설정
    ↓
ApprovalDelegation {
  delegatorId     — 위임자 (원래 매니저)
  delegateeId     — 수임자 (권한 받는 사람)
  startDate       — 위임 시작일
  endDate         — 위임 종료일
  scope           — 위임 범위 (LEAVE / ALL)
}
    ↓
위임 기간 동안:
  - 수임자가 해당 팀의 승인함에 접근 가능
  - 수임자의 승인/반려에 "위임 승인(Delegated)" 태그 표시
  - 원래 매니저에게도 처리 결과 알림
    ↓
위임 기간 종료 → 권한 자동 회수
```

| 규칙 | 내용 |
|------|------|
| 수임자 범위 | 같은 팀 시니어, 상위 매니저, 또는 HR |
| 위임 표시 | 승인 기록에 "위임 승인 by {수임자명}" 태그 |
| 동시 위임 | 1명의 매니저가 동시에 1명에게만 위임 가능 |
| 위임 중 원래 매니저 | 원래 매니저도 여전히 승인 가능 (권한 회수가 아닌 추가) |
| 미설정 시 | 위임 미설정 상태에서 매니저 장기 부재 → PENDING 3일 초과 Nudge → HR에게 에스컬레이션 |

### 승인 후 취소

| 상태 | 취소 가능 여부 | 처리 |
|------|:---:|------|
| PENDING | ✓ 직원이 직접 취소 | 잔여일수 변동 없음 (아직 차감 전) |
| APPROVED (휴가 시작 전) | ✓ 직원이 취소 요청 | usedDays 차감분 복구 |
| APPROVED (휴가 시작 후) | △ HR만 취소 가능 | usedDays 부분 복구 (실제 사용일 제외) |

### 벌크 승인

매니저(또는 Delegation 수임자)가 여러 건을 한 번에 승인할 수 있습니다.

```
승인 대기함
  ☑ 김대리 — 연차 3/15~3/16 (2일)
  ☑ 이사원 — 병가 3/18 (1일)
  ☐ 박과장 — 연차 3/20~3/25 (4일)  ← 기간이 길어 개별 검토
    ↓
"선택 건 일괄 승인" 클릭 → 2건 동시 처리
```

## 9. STEP 5 — 급여 연동

### 급여 반영 방식

```
승인된 휴가 (LeaveRequest.status = APPROVED)
    ↓
PayrollRun 생성 시 자동 조회:
  - 해당 급여 기간 내 승인된 휴가 목록
  - 유급 휴가 → 급여 차감 없음
  - 무급 휴가 → 통상임금 고정 단가 차감
  - 반차 → 0.5일 차감
    ↓
급여 명세서에 "휴가 공제" 항목 표시
```

### 🚨 무급휴가 공제 공식의 표준화 (통상임금 고정 단가)

**문제:** 무급휴가 사용 시 해당 월의 근무일수(20일 vs 22일)에 따라 차감 금액이 달라지는 불공정성.

**해결:** 달력 일수와 무관하게 법정 소정근로시간 기준의 고정 단가를 적용합니다.

```
KR 법인 공제 공식:
  1일 차감액 = (월 통상임금 ÷ 209시간) × 8시간

예시 (월 통상임금 300만원):
  1일 차감액 = (3,000,000 ÷ 209) × 8 = 약 114,833원

→ 2월이든 3월이든 1일 무급휴가 차감액이 동일
```

| 법인 | 공제 기준 | 공식 |
|------|----------|------|
| **KR** | 통상임금 ÷ 209h × 8h | 법정 소정근로시간 기준 고정 단가 |
| **US** | Hourly Rate × 8h | 시급 기반 (exempt은 무급 불가) |
| **CN** | 月工资 ÷ 21.75일 | 법정 월 평균 근무일 기준 |
| **RU** | 중평균임금 ÷ 근무일수 | 최근 12개월 평균 기준 |
| **VN** | 월급 ÷ 26일 | 법정 기준 |
| **MX** | 일급 기준 | 일 통합임금(SDI) 기준 |

> 각 법인의 공제 로직은 `lib/labor/{country}.ts`에서 국가별 분기 처리합니다.

### 유급/무급 처리

| 유형 | 급여 영향 |
|------|----------|
| 유급 휴가 (연차, 경조사 등) | 급여 차감 없음. 정상 근무일로 간주 |
| 무급 휴가 | 통상임금 고정 단가 × 무급 일수 차감 |
| 마이너스 연차 사용분 | 유급 처리 (빚은 다음해 잔여일수에서 차감, 급여에서 안 깎음) |
| 병가 (법인별) | KR: 유급 (3일까지) / 이후 무급. 국가별 상이 |

## 10. 상태 머신

### LeaveRequest 상태

```
PENDING → APPROVED → (자동 소멸)
    │         │
    │         └→ CANCELLED (직원 취소 or HR 취소)
    │
    └→ REJECTED (매니저 반려)
    │
    └→ CANCELLED (직원 직접 취소)
```

| 상태 | 설명 | 전환 가능 |
|------|------|----------|
| PENDING | 신청됨, 승인 대기 | → APPROVED, REJECTED, CANCELLED |
| APPROVED | 승인 완료 | → CANCELLED (시작 전: 직원, 시작 후: HR만) |
| REJECTED | 반려됨 | 최종 상태 (재신청은 새 건으로) |
| CANCELLED | 취소됨 | 최종 상태 |

### 잔액 영향 매트릭스

| 전환 | usedDays 변동 | 설명 |
|------|:---:|------|
| PENDING → APPROVED | +N일 | 확정 사용 |
| PENDING → REJECTED | 0 | 변동 없음 |
| PENDING → CANCELLED | 0 | 변동 없음 |
| APPROVED → CANCELLED | -N일 | 복구 |

## 11. 이벤트 체인

| 이벤트 | 트리거 | 자동 처리 |
|--------|--------|----------|
| LEAVE_REQUESTED | 직원이 휴가 신청 | 매니저(또는 Delegation 수임자)에게 승인 요청 알림 |
| LEAVE_APPROVED | 매니저/수임자 승인 | 직원 알림 + LeaveBalance 차감 + 근태 반영 |
| LEAVE_REJECTED | 매니저/수임자 반려 | 직원 알림 (반려 사유 포함) |
| LEAVE_CANCELLED | 직원/HR 취소 | 관련자 알림 + LeaveBalance 복구 (APPROVED 취소 시) |
| LEAVE_BALANCE_GRANTED | 연초/월별 잔액 부여 | LeaveBalance 갱신 + 마이너스 잔액 상환 |
| LEAVE_BALANCE_CARRIED_OVER | 연말 이월 처리 | 이월 상한 적용 후 잔여 이월 |
| LEAVE_PENDING_REMINDER | PENDING 상태 3일 경과 | 매니저에게 재촉 Nudge |
| LEAVE_NEGATIVE_BALANCE_USED | 마이너스 연차 사용 | HR에게 알림 (모니터링용) |
| DELEGATION_STARTED | 승인 권한 위임 시작 | 수임자에게 위임 시작 알림 |
| DELEGATION_ENDED | 승인 권한 위임 종료 | 양측에게 위임 종료 알림 |

### Nudge 규칙

| 규칙 | 조건 | 대상 | 간격 | 최대 횟수 |
|------|------|------|:---:|:---:|
| 승인 대기 독촉 | PENDING 3일+ | 매니저 (또는 수임자) | 2일 | 3회 |

| Delegation 미설정 경고 | 매니저 휴가 신청 + Delegation 미설정 | 해당 매니저 | 즉시 1회 | 1회 |

## 13. 엣지 케이스

| # | 상황 | 처리 방안 |
|:---:|------|----------|
| 1 | 휴가 중 **퇴사** | 미사용 연차 정산: 잔여일수 × 1일 통상임금 지급. 마이너스 잔액은 최종 급여에서 공제 |
| 2 | 승인 후 **날짜 변경** | 기존 건 취소 + 새 건 신청 (수정이 아닌 재신청) |
| 3 | **공휴일이 휴가 기간에 포함** | 공휴일은 자동 제외 (근무일만 차감) |
| 4 | **연말/연초 경계 신청** (12/30~1/2) | 각 연도별 잔여일수에서 분할 차감 |
| 5 | **병가 3일 이상 증빙 미제출** | PENDING 유지 + 증빙 제출 독촉 알림 |
| 6 | **팀 동시 부재 초과** | Soft Warning (매니저에게 경고만, 신청 차단 안 함) |
| 7 | **법인 간 전출** 시 잔여일수 | 기존 법인 잔여 정산 → 신규 법인 정책으로 재부여 (비례) |
| 8 | **입사 첫해 이월** | 비례 부여분에 대해서는 이월 허용 여부 법인별 설정 |
| 9 | **반차 + 반차 = 1일?** | 같은 날 오전반차+오후반차 신청 시 1일 종일 휴가로 합산 |
| 10 | **대리 승인자(수임자)의 부재** | 수임자도 부재 시 → PENDING Nudge → HR 에스컬레이션 |
| 11 | **마이너스 연차 사용 후 퇴사** | 미상환 마이너스분 × 통상임금 1일 단가를 최종 급여에서 공제 |
| 12 | **마이너스 한도 변경** | 기존 사용분은 유지. 한도 축소 시 추가 마이너스만 차단 |

## 16. Workday와의 설계 비교 (CTR 특화)

CTR이 의도적으로 Workday와 다르게 설계한 항목과 그 근거입니다.

| 항목 | Workday 방식 | CTR 설계 | 근거 |
|------|-------------|---------|------|
| 잔액 모델 | 단일 Balance 테이블 | 이중 모델 (Balance + Request 분리) | PENDING 건의 "예약" 잔액을 별도 계산하여 실시간 UX 제공 |
| 신청 시 팀 부재 표시 | 매니저만 볼 수 있음 | 직원에게도 표시 | 직원이 알아서 날짜 조정 → 불필요한 반려 감소 |
| 취소 정책 | 승인 후 취소 = HR 전용 | 시작 전이면 직원 직접 취소 가능 | 직원 자율성 + HR 부담 감소 |
| 이월 처리 | 자동 + 수동 혼합 | 시스템 자동 (이월 상한 적용) | HR 수작업 제거 |
| 마이너스 연차 | 미지원 (잔여 0 = 차단) | 법인별 마이너스 한도 허용 | 직원 유연성 + HR 예외 처리 부담 감소 |
| 승인 위임 | Manager Delegation 기능 | 동일 + 미설정 시 Nudge 경고 | 위임 미설정으로 인한 병목 사전 방지 |
| 무급 공제 | 월 근무일수 ÷ 일할 | 통상임금 고정 단가 (209h) | 월별 근무일수 차이로 인한 불공정 제거 |
| Optimistic UI | 없음 (서버 응답 대기) | Optimistic 즉시 반영 | 모바일 환경 + 공장 현장 UX 중요 |

## 17. 피드백 요청 사항

### 인사팀 (전사)

1. 11개 휴가 유형이 현재 운영 중인 유형과 일치하는가요? 추가/삭제 필요한 유형?
2. 이월 상한(예: 10일)이 법인별로 적절한가요?
3. 팀 동시 부재 제한 기능 — 필요한 법인이 있나요? 최대 몇 명?
4. 사전 신청 기간 (최소 N일 전 신청) — 법인별로 다르게 설정하고 싶은가요?
5. Nudge "연말 소진 유도" — 11월부터 시작이 적절한가요?

### 법인 HR

6. 법인별 병가 유급/무급 처리 기준이 위 표와 맞나요?
7. 법인 간 전출 시 잔여일수 처리 — "정산 후 재부여" 방식이 적합한가요?
8. 반차 허용 여부 — RU, MX에서도 반차가 필요한가요?
9. 근속 가산 기준이 법인별로 다른 경우 구체적으로 알려주세요.
10. 마이너스 연차 — 법인별로 ON/OFF 및 한도를 어떻게 설정하고 싶으신가요?

### 경영진

11. 전사 연차 사용률 목표가 있나요? (예: 70% 이상 사용 유도)
12. 연차 미사용 보상 정책 — 미사용분 금전 보상이 법인별로 다른가요?
13. 임원(Executive)의 휴가 승인 체계 — 대표 직접 승인? 자율?
14. 마이너스 연차 최대 한도 — 글로벌 기본값을 몇 일로 설정할까요?

## 부록 A: 용어 정리

| 용어 | 설명 |
|------|------|
| 이중 잔액 모델 | LeaveBalance(확정 잔액) + LeaveRequest PENDING(예약 잔액) 분리 운영 |
| Front-loaded | 연초에 연간 일수를 일괄 부여하는 방식 |
| Monthly Accrual | 매월 일정 일수를 발생시키는 방식 |
| Carry-over | 미사용 잔여일수를 다음 연도로 이월 |
| Negative Balance | 잔여일수 0 이하로도 신청 가능한 마이너스 연차 제도 |
| Optimistic UI | 서버 응답 전에 UI를 먼저 반영하는 패턴 |
| Delegation | 매니저 부재 시 승인 권한을 다른 사람에게 위임 |
| 통상임금 고정 단가 | (월 통상임금 ÷ 209h) × 8h — 월별 근무일수와 무관한 고정 공제 단가 |
| 팀 부재 컨텍스트 | 휴가 신청 시 팀원 부재 현황을 함께 표시 |
| Nudge | 마감/기한 임박 시 자동 발송되는 재촉 알림 |
| CompanySelector | 법인 선택 드롭다운 (HR 뷰 전용) |
| Golden Path | 모듈들을 end-to-end 파이프라인으로 연결하는 프로젝트 |

## 부록 B: 전체 결정사항 요약

| # | 항목 | 결정 |
|---|------|------|
| 1 | 파이프라인 | 5단계 (정책설정→잔액부여→신청→승인→급여연동) |
| 2 | 잔액 모델 | 이중 모델 (Balance + Request 분리, 의도적 동기화 안 함) |
| 3 | UI 반영 | Optimistic UI (신청 즉시 잔여일수 반영) |
| 4 | 팀 부재 표시 | 직원에게도 표시 (신청 폼 내) |
| 5 | 정책 패턴 | 글로벌 기본값 + 법인별 오버라이드 (companyId=NULL = 글로벌) |
| 6 | 휴가 유형 | 글로벌 11개 표준 + HR 추가 가능 |
| 7 | 부여 방식 | Front-loaded / Monthly Accrual 법인별 선택 |
| 8 | 이월 | 시스템 자동 처리 (이월 상한 적용) |
| 9 | 취소 정책 | 시작 전: 직원 직접 취소 / 시작 후: HR만 |
| 10 | 승인 구조 | 직속 매니저 1단계 + Delegation 위임 체계 |
| 11 | 벌크 승인 | 매니저 승인함에서 다중 선택 일괄 승인 |
| 12 | 반차 | 법인별 설정 (KR/CN/VN: 가능, RU/MX: 불가) |
| 13 | 급여 연동 | PayrollRun 생성 시 승인 휴가 자동 조회 |
| 14 | 무급 공제 | 통상임금 고정 단가 (KR: 209h 기준, 국가별 분기) |
| 15 | 마이너스 연차 | 법인별 ON/OFF + 최대 마이너스 한도 설정 |
| 16 | Delegation | 매니저 부재 시 승인 권한 위임 (ApprovalDelegation 모델) |
| 17 | Nudge | 승인 대기 독촉 / 연말 소진 유도 / Delegation 미설정 경고 |

### GP#1 구현 상태: ✅ 100% 완료 + 기획 초과 구현

**API 라우트: 18개** (기획서 기준 약 12개 → 6개 추가)

| 추가 API | 설명 |
|----------|------|
| `POST /api/v1/leave/bulk-grant` | 부서/직급 필터 → 다중 선택 → 일괄 부여 |
| `GET /api/v1/leave/year-balances` | 연도별 잔액 이력 조회 |
| `GET/POST /api/v1/leave/accrual-rules` | 발생 규칙 CRUD |
| `GET/POST /api/v1/leave/type-defs` | 휴가 유형 정의 CRUD |
| `GET /api/v1/leave/admin/stats` | HR 관리자 대시보드 KPI (사용률, 마이너스, 소진 예측) |

**LeaveTypeDef 추가 필드** (기획서 미기술):

| 필드 | 설명 |
|------|------|
| `allowHalfDay` | 반차 허용 여부 (법인별 ON/OFF) |
| `requiresProof` | 증빙 첨부 필수 여부 |
| `maxConsecutiveDays` | 최대 연속 사용 일수 |
| `displayOrder` | UI 정렬 순서 |

> 위 필드들은 기획서 섹션 4(STEP 1 정책 설정)의 정책 항목과 정합성 있음. 기획서에서 "설정 가능"으로 기술한 항목들이 실제 모델 필드로 구현된 것.
